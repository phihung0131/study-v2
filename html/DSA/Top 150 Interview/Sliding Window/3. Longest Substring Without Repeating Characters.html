3. Longest Substring Without Repeating Characters

Medium Hash Table Sliding Window

Problem Description

Given a string s, find the length of the longest substring without repeating characters.

Example 1:

Input: s = "abcabcbb"

Output: 3 (The answer is "abc", with the length of 3)

Approach 1: Brute Force

The "Verify All" Strategy

Intuition

Check every single possible substring. For each substring, use a helper method to verify if all characters are unique. This is simple but computationally expensive because we re-check the same characters over and over.

Java Implementation (Brute Force)

public int lengthOfLongestSubstring(String inputString) {
    int maxLength = 0;
    int n = inputString.length();

    // Try every starting index
    for (int startIdx = 0; startIdx < n; startIdx++) {
        // Try every ending index
        for (int endIdx = startIdx + 1; endIdx <= n; endIdx++) {
            if (allUnique(inputString, startIdx, endIdx)) {
                maxLength = Math.max(maxLength, endIdx - startIdx);
            }
        }
    }
    return maxLength;
}

private boolean allUnique(String s, int start, int end) {
    Set<Character> set = new HashSet<>();
    for (int i = start; i < end; i++) {
        if (set.contains(s.charAt(i))) return false;
        set.add(s.charAt(i));
    }
    return true;
}

Complexity Analysis

Time: O(n³) - We iterate through O(n²) substrings, and for each, we iterate O(n) characters to check uniqueness.

Space: O(min(n, m)) - Where m is the size of the character set.

Pros & Cons

✅ Very simple to verify small strings.

❌ Extremely slow for long strings (n > 1,000).

Approach 2: Sliding Window (HashSet)

The "Dynamic Window" Strategy

Intuition

Instead of re-checking everything, we use two pointers to define a window. We move the right pointer to add characters. If we hit a duplicate, we move the left pointer forward, removing characters from our set until the duplicate is gone. This "slides" the window across the string.

Java Implementation (HashSet)

public int lengthOfLongestSubstring(String inputString) {
    int n = inputString.length();
    Set<Character> uniqueChars = new HashSet<>();
    int maxLength = 0, leftPointer = 0, rightPointer = 0;

    while (rightPointer < n) {
        if (!uniqueChars.contains(inputString.charAt(rightPointer))) {
            // Character is unique, expand the window
            uniqueChars.add(inputString.charAt(rightPointer));
            rightPointer++;
            maxLength = Math.max(maxLength, rightPointer - leftPointer);
        } else {
            // Duplicate found, shrink from the left
            uniqueChars.remove(inputString.charAt(leftPointer));
            leftPointer++;
        }
    }
    return maxLength;
}

Complexity Analysis

Time: O(2n) = O(n) - In the worst case, each character is visited twice (once by right, once by left).

Space: O(min(n, m)) - Size of the HashSet.

Pros & Cons

✅ Highly efficient for most strings.

❌ Still potentially moves the left pointer many times incrementally.

Approach 3: Optimized Sliding Window (HashMap)

The "Fast Jump" Strategy

Intuition

Instead of shrinking the window one character at a time, we store the last seen index of each character in a Map (or an array). When we find a duplicate, we jump the left pointer directly to `index + 1` of the previous occurrence. This is the fastest possible way.

public int lengthOfLongestSubstring(String s) {
    int n = s.length(), maxLength = 0;
    // Map to store the LAST seen index + 1 for each character
    Map<Character, Integer> lastSeenMap = new HashMap<>(); 

    for (int right = 0, left = 0; right < n; right++) {
        char currentChar = s.charAt(right);
        
        if (lastSeenMap.containsKey(currentChar)) {
            // Jump left to the position after the duplicate
            // Must take Math.max to avoid moving left pointer backwards
            left = Math.max(lastSeenMap.get(currentChar), left);
        }

        maxLength = Math.max(maxLength, right - left + 1);
        // Store next valid starting point
        lastSeenMap.put(currentChar, right + 1);
    }
    return maxLength;
}

Complexity Analysis

Time: O(n) - Only the right pointer moves from 0 to n. Left pointer jumps.

Space: O(min(m, n)) - Size of the Map.

Pros & Cons

✅ Absolutely optimal performance.

❌ Logic is slightly more subtle (Math.max is crucial).

Comparison

Approach Time Space Efficiency Note
Brute Force O(n³) O(m) Too slow for production.
Sliding Window (Set) O(2n) O(m) Standard solution.
Optimized Map O(n) O(m) Fastest. One pass only.

Key Takeaways

The "Window" Habit

Any "Longest/Shortest Subarray" problem on strings/arrays is usually a Sliding Window candidate.

Set vs Map

A HashSet tracks existence. A HashMap tracks "where did I see this before?". Use the Map to skip work.

Why Math.max?

Crucial for optimized window! When jumping `left`, you must ensure you don't jump backward to a duplicate that was already outside the current window.
