30. Substring with Concatenation of All Words

Hard Hash Table Sliding Window

Problem Description

You are given a string s and an array of strings words. All strings in words are of the same length.

Find all starting indices of substring(s) in s that is a concatenation of each word in words exactly once and without any intervening characters.

Example:

Input: s = "barfoothefoobarman", words = ["foo","bar"]

Output: [0, 9]

Explanation: "barfoo" starts at 0, "foobar" starts at 9.

Approach 1: Check Every Index

Simple but repetitive string slicing.

Intuition

We know the total length of the required concatenation is numWords * wordLength. We can simply iterate through every possible start index in s, extract a substring of that length, and check if it can be decomposed into exactly the words we have in our list.

Java Implementation (Naive)

public List<Integer> findSubstring(String s, String[] words) {
    List<Integer> result = new ArrayList<>();
    int wordCount = words.length;
    int wordLen = words[0].length();
    int totalLen = wordCount * wordLen;

    Map<String, Integer> wordFreq = new HashMap<>();
    for (String w : words) wordFreq.put(w, wordFreq.getOrDefault(w, 0) + 1);

    // Iterate through every possible start index
    for (int startIdx = 0; startIdx <= s.length() - totalLen; startIdx++) {
        Map<String, Integer> seenWords = new HashMap<>();
        int matchCount = 0;
        
        // Check words one by one in the current window
        for (int j = 0; j < wordCount; j++) {
            int currentWordStart = startIdx + j * wordLen;
            String word = s.substring(currentWordStart, currentWordStart + wordLen);
            
            if (wordFreq.containsKey(word)) {
                seenWords.put(word, seenWords.getOrDefault(word, 0) + 1);
                if (seenWords.get(word) > wordFreq.get(word)) break;
                matchCount++;
            } else {
                break;
            }
        }
        
        if (matchCount == wordCount) result.add(startIdx);
    }
    return result;
}

Complexity Analysis

Time: O(N * M * L) where N is string length, M is number of words, L is word length. We check N indices, each doing M substring operations.

Space: O(M) - Frequency maps store M unique words.

Pros & Cons

✅ Straightforward logic, easy to debug.

❌ High constant factor; repetitive substring operations.

Approach 2: Sliding Window (The Offset Trick)

Optimal O(N) performance by jumping word-by-word.

Intuition

Since all words have length L, a valid concatenation must start at some index i. If we shift i by L, we are effectively just removing one word from the left and adding one to the right. We can run L separate sliding windows (starting at offsets 0, 1, ..., L-1) to cover all possible starting positions efficiently.

Java Implementation (Optimized)

public List<Integer> findSubstring(String s, String[] words) {
    List<Integer> res = new ArrayList<>();
    int wordLen = words[0].length(), wordCount = words.length();
    Map<String, Integer> counts = new HashMap<>();
    for (String w : words) counts.put(w, counts.getOrDefault(w, 0) + 1);

    // Run L separate sliding windows
    for (int offset = 0; offset < wordLen; offset++) {
        Map<String, Integer> seen = new HashMap<>();
        int leftPointer = offset, count = 0;
        
        for (int rightPointer = offset; rightPointer <= s.length() - wordLen; rightPointer += wordLen) {
            String word = s.substring(rightPointer, rightPointer + wordLen);
            if (counts.containsKey(word)) {
                seen.put(word, seen.getOrDefault(word, 0) + 1);
                count++;
                // If we have too many instances of 'word', shrink from left
                while (seen.get(word) > counts.get(word)) {
                    String leftWord = s.substring(leftPointer, leftPointer + wordLen);
                    seen.put(leftWord, seen.get(leftWord) - 1);
                    count--;
                    leftPointer += wordLen;
                }
                if (count == wordCount) res.add(leftPointer);
            } else {
                // Not a word we want, reset window
                seen.clear();
                count = 0;
                leftPointer = rightPointer + wordLen;
            }
        }
    }
    return res;
}

Complexity Analysis

Time: O(N). Each offset loop processes a disjoint set of characters. Each character is visited constant number of times.

Space: O(M). Hash map storing word frequencies.

Pros & Cons

✅ Mathematically optimal time complexity.

❌ More complex implementation; logic for resets is delicate.

Efficiency Comparison

Approach Time Space When to use?
Naive Brute Force O(N * M * L) O(M) Interviews where time is limited or constraints are small.
Optimized Sliding Window O(N) O(M) Production and competitive programming.

Key Takeaways for "Concatenation" Problems

  • Fixed Length is Key: If words are not equal length, the sliding window "offset trick" doesn't work.
  • Frequency Maps: Essential for tracking counts of duplicate words (e.g., words = ["a", "a"]).
  • Substrings: String slicing in Java (substring) creates new objects. For heavy optimization, consider working with character arrays or hashes.

💡 Interview Tip

When you see "all same length", think: Jumping Sliding Window. Instead of moving 1 char at a time, move L chars at a time to drastically reduce the search space.
