76. Minimum Window Substring

Hard Hash Table Sliding Window

Problem Description

Given two strings s and t of lengths m and n respectively, return the minimum window substring of s such that every character in t (including duplicates) is included in the window.

If there is no such substring, return the empty string "". The test cases will be generated such that the answer is unique.

Example:

Input: s = "ADOBECODEBANC", t = "ABC"

Output: "BANC"

Explanation: The minimum window "BANC" includes 'A', 'B', and 'C' from string t.

Approach 1: Brute Force

Generating all possible windows.

Intuition

The simplest approach is to check every possible substring of s. For each substring, we check if it contains all characters of t with the required frequencies. We keep track of the smallest such substring found.

Target (t) "ABC"
Min Found ""
Ready to scan all substrings...

Java Implementation (Brute Force)

public String minWindow(String s, String t) {
    String minWindow = "";
    int minLength = Integer.MAX_VALUE;

    // Check every possible start and end index
    for (int startIdx = 0; startIdx < s.length(); startIdx++) {
        for (int endIdx = startIdx; endIdx < s.length(); endIdx++) {
            String sub = s.substring(startIdx, endIdx + 1);
            
            if (containsAll(sub, t)) {
                if (sub.length() < minLength) {
                    minLength = sub.length();
                    minWindow = sub;
                }
            }
        }
    }
    return minWindow;
}

private boolean containsAll(String sub, String t) {
    Map<Character, Integer> counts = new HashMap<>();
    for (char c : t.toCharArray()) counts.put(c, counts.getOrDefault(c, 0) + 1);
    
    for (char c : sub.toCharArray()) {
        if (counts.containsKey(c)) {
            counts.put(c, counts.get(c) - 1);
        }
    }
    
    for (int count : counts.values()) {
        if (count > 0) return false;
    }
    return true;
}

Complexity Analysis

Time: O(N³). We generate O(N²) substrings and for each, we scan it in O(N) time.

Space: O(K) where K is the size of the character set in string t.

Pros & Cons

✅ Mathematically exhaustive; easy to reason about.

❌ Time Limit Exceeded (TLE) for even small inputs in LeetCode.

Approach 2: Sliding Window (Optimal)

Expanding and contracting to find the sweet spot.

Intuition

We use two pointers, leftPointer and rightPointer. First, we move rightPointer to expand the window until it contains all characters from t. Once the window is "valid", we move leftPointer to shrink the window as much as possible while keeping it valid. This "contracting" phase helps us find the minimal length.

Current Window Freq {}
Min Length ∞
Ready to slide...

Java Implementation (Sliding Window)

public String minWindow(String s, String t) {
    if (s.length() == 0 || t.length() == 0) return "";

    // Dictionary which keeps a count of all the unique characters in t.
    Map<Character, Integer> targetFreq = new HashMap<>();
    for (char c : t.toCharArray()) targetFreq.put(c, targetFreq.getOrDefault(c, 0) + 1);

    int requiredMatches = targetFreq.size();
    int leftPointer = 0, rightPointer = 0, formedMatches = 0;

    // Dictionary which keeps a count of all the unique characters in the current window.
    Map<Character, Integer> windowFreq = new HashMap<>();

    // ans list of the form {window length, left, right}
    int[] ans = {-1, 0, 0};

    while (rightPointer < s.length()) {
        char c = s.charAt(rightPointer);
        windowFreq.put(c, windowFreq.getOrDefault(c, 0) + 1);

        // If the frequency of current char matches desired freq in t, increment match count.
        if (targetFreq.containsKey(c) && windowFreq.get(c).intValue() == targetFreq.get(c).intValue()) {
            formedMatches++;
        }

        // Try and contract the window till the point where it ceases to be 'desirable'.
        while (leftPointer <= rightPointer && formedMatches == requiredMatches) {
            c = s.charAt(leftPointer);

            // Save the smallest window until now.
            if (ans[0] == -1 || rightPointer - leftPointer + 1 < ans[0]) {
                ans[0] = rightPointer - leftPointer + 1;
                ans[1] = leftPointer;
                ans[2] = rightPointer;
            }

            // The character at the position pointed by the `leftPointer` is no longer a part of the window.
            windowFreq.put(c, windowFreq.get(c) - 1);
            if (targetFreq.containsKey(c) && windowFreq.get(c) < targetFreq.get(c)) {
                formedMatches--;
            }

            leftPointer++;
        }

        rightPointer++;
    }

    return ans[0] == -1 ? "" : s.substring(ans[1], ans[2] + 1);
}

Complexity Analysis

Time: O(N + M). We traverse string s once and t once. Each character in s is processed at most twice.

Space: O(K). Map size is proportional to the number of unique characters.

Pros & Cons

✅ Optimal performance; scales to very large strings.

❌ Harder to implement; requires keeping track of frequency logic and pointer state.

Efficiency Comparison

Approach Time Space Summary
Naive Brute Force O(N³) O(1) Simple but unusable for large data.
Sliding Window O(N) O(K) Standard algorithm for substring window problems.

Key Takeaways: Sliding Window Patterns

  • Expand until Valid: Move the right boundary until the condition (e.g., "contains all chars") is satisfied.
  • Shrink to Optimize: Move the left boundary as long as the condition remains satisfied.
  • State Management: Use counters (`formedMatches`) and target counts (`requiredMatches`) to avoid repeated map scanning inside the loop.
  • Characters vs Strings: In Java, `Map` is easy to explain, but `int[128]` (frequency array) is faster.
"Find the smallest window that contains everything. Think of it as a caterpillar: stretching its head forward, then pulling its tail behind it."
