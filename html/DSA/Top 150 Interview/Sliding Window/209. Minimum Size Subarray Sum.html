209. Minimum Size Subarray Sum

Medium Array Sliding Window

Problem Description

Given an array of positive integers nums and a positive integer target, return the minimal length of a subarray whose sum is greater than or equal to target. If there is no such subarray, return 0 instead.

Example 1:

Input: target = 7, nums = [2, 3, 1, 2, 4, 3]

Output: 2 (Subarray [4, 3] has length 2)

Approach 1: Brute Force

The "Check Everything" Strategy

Intuition

The simplest way is to check every single possible subarray within the input. For every starting index, we iterate through all possible ending indices, calculate the sum, and check if it meets the target. We keep track of the minimum length found.

Java Implementation (Brute Force)

public int minSubArrayLen(int target, int[] nums) {
    int n = nums.length;
    int minLength = Integer.MAX_VALUE;

    // Iterate through every possible starting point
    for (int startIdx = 0; startIdx < n; startIdx++) {
        int currentSum = 0;
        // Iterate through every possible ending point
        for (int endIdx = startIdx; endIdx < n; endIdx++) {
            currentSum += nums[endIdx];
            // Check if current subarray sum satisfies target
            if (currentSum >= target) {
                minLength = Math.min(minLength, endIdx - startIdx + 1);
                break; // Optimizing slightly: stop early if target met
            }
        }
    }

    return (minLength == Integer.MAX_VALUE) ? 0 : minLength;
}

Complexity Analysis

Time: O(n²) - Two nested loops. In the worst case, we check all pairs.

Space: O(1) - No extra data structures used.

Pros & Cons

✅ Very easy to implement and understand.

❌ Inefficient for large arrays (n > 10,000).

Approach 2: Prefix Sums + Binary Search

The "Math-Heavy" Strategy

Intuition

Since all numbers are positive, the Prefix Sums array will be strictly increasing. We can calculate `sums[i]` (sum from 0 to i). Then, for each `sums[i]`, we need to find the smallest index `j` such that `sums[j] - sums[i-1] >= target`. Since `sums` is sorted, we can use Binary Search to find `j` in O(log n) time.

Java Implementation (Binary Search)

public int minSubArrayLen(int target, int[] nums) {
    int n = nums.length;
    if (n == 0) return 0;
    int minLength = Integer.MAX_VALUE;
    int[] sums = new int[n + 1];
    
    // Precompute Prefix Sums: O(n)
    for (int i = 1; i <= n; i++) sums[i] = sums[i - 1] + nums[i - 1];

    // For each i, find j such that sums[j] >= sums[i-1] + target
    for (int i = 1; i <= n; i++) {
        int toFind = target + sums[i - 1];
        int bound = Arrays.binarySearch(sums, toFind);
        // binarySearch returns -(insertionPoint) - 1 if not found
        if (bound < 0) bound = -bound - 1;
        if (bound <= n) {
            minLength = Math.min(minLength, bound - (i - 1));
        }
    }
    return (minLength == Integer.MAX_VALUE) ? 0 : minLength;
}

Complexity Analysis

Time: O(n log n) - We iterate `n` times and perform a binary search inside each iteration.

Space: O(n) - Storing the prefix sums array.

Pros & Cons

✅ Good complexity, uses interesting properties of sorted data.

❌ Requires O(n) extra space. More complex to code correctly.

Approach 3: Sliding Window

The "Caterpillar" Strategy (Most Efficient)

Intuition

Imagine a window that stretches as we move the right pointer to add elements. Once the sum hits the target, we try to shrink the window from the left to find the absolute minimum size that still satisfies the condition. This ensures we never process the same element twice with the same pointer.

Java Implementation (Sliding Window)

public int minSubArrayLen(int target, int[] nums) {
    int leftPointer = 0;
    int runningSum = 0;
    int minLength = Integer.MAX_VALUE;

    // rightPointer expands the window
    for (int rightPointer = 0; rightPointer < nums.length; rightPointer++) {
        runningSum += nums[rightPointer];

        // Shrink window as much as possible while sum >= target
        while (runningSum >= target) {
            minLength = Math.min(minLength, rightPointer - leftPointer + 1);
            runningSum -= nums[leftPointer];
            leftPointer++; // Shift left boundary right
        }
    }

    return (minLength == Integer.MAX_VALUE) ? 0 : minLength;
}

Complexity Analysis

Time: O(n) - Each element is visited at most twice (once by right, once by left pointer).

Space: O(1) - No extra space proportional to input size.

Pros & Cons

✅ Optimal performance in both time and space.

❌ Logic is trickier (requires careful handling of pointer boundaries).

Comparison

Approach Time Space Key Characteristic
Brute Force O(n²) O(1) Simple nested iteration. Slow for large N.
Binary Search O(n log n) O(n) Relies on sorted prefix sums property.
Sliding Window O(n) O(1) Optimal. Single pass with two pointers.

Cheat Sheet: When to use Sliding Window?

  • ✔ Problem asks for Minimum/Maximum Subarray.
  • ✔ The data is linear (Array or String).

💡 Quick Hint

If the array contains negative numbers, standard Sliding Window doesn't work because adding an element could decrease the sum. In that case, use Prefix Sums + HashMap.
