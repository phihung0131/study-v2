Longest Common Prefix

LeetCode #14 | Difficulty: Easy | String, Array

🔍 Problem Summary

Write a function to find the longest common prefix string amongst an array of strings. If there is no common prefix, return an empty string "".

Example: strs = ["flower", "flow", "flight"] → Output: "fl"

Approach 1: Horizontal Scanning

Intuition: "One String at a Time"

We start by assuming the first string is the common prefix. Then we compare it with the second string and "trim" it until it matches. We take that result and compare it with the third, and so on.

Java Implementation

public String longestCommonPrefix(String[] strs) {
    if (strs == null || strs.length == 0) return "";
    
    // Start with the first string as the potential prefix
    String prefix = strs[0];
    
    for (int i = 1; i < strs.length; i++) {
        // While the current string doesn't start with 'prefix'
        while (strs[i].indexOf(prefix) != 0) {
            // Trim the prefix by one character from the end
            prefix = prefix.substring(0, prefix.length() - 1);
            if (prefix.isEmpty()) return "";
        }
    }
    return prefix;
}

Complexity

Time: O(S), where S is the sum of all characters in all strings.

Space: O(1). We only use a string variable.

Trade-off

Easy to implement, but if the mismatch happens at the very start of a long array, we waste time processing earlier strings.

Approach 2: Vertical Scanning (Optimal for early exits)

Intuition: "One Character at a Time"

We compare the first character of all strings. If they all match, we move to the second character. As soon as we find a mismatch or reach the end of any string, the process stops.

Java Implementation

public String longestCommonPrefix(String[] strs) {
    if (strs == null || strs.length == 0) return "";
    
    // Iterate through each character of the first string
    for (int charIdx = 0; charIdx < strs[0].length(); charIdx++) {
        char currentChar = strs[0].charAt(charIdx);
        
        // Compare this character with the same index in all other strings
        for (int strIdx = 1; strIdx < strs.length; strIdx++) {
            // Mismatch found OR current string is too short
            if (charIdx == strs[strIdx].length() || 
                strs[strIdx].charAt(charIdx) != currentChar) {
                return strs[0].substring(0, charIdx);
            }
        }
    }
    return strs[0];
}

Complexity

Time: O(S). In the best case, it exits very early (e.g., first chars mismatch).

Space: O(1).

Efficiency

This is generally preferred in interviews because it handles early-mismatch scenarios much more gracefully than Horizontal scanning.

Approach 3: Sorting Trick

Intuition: "Compare the Extremes"

If we sort the array lexicographically, the most different strings will be at the very first and very last positions. The common prefix of the entire array must be the common prefix of just the first and last sorted strings.

Java Implementation

public String longestCommonPrefix(String[] strs) {
    Arrays.sort(strs);
    String first = strs[0];
    String last = strs[strs.length - 1];
    
    int index = 0;
    while (index < first.length() && index < last.length()) {
        if (first.charAt(index) == last.charAt(index)) {
            index++;
        } else {
            break;
        }
    }
    return first.substring(0, index);
}

Complexity

Time: O(N log N * M), where N is number of strings and M is avg length (sorting cost).

Space: O(1) or O(log N) depending on sort implementation.

Verdict

Elegant code, but theoretically slower due to sorting overhead. Best when you want to write the fewest lines of code.

⚡ Recall Cheat Sheet

  • • Horizontal Scanning: Compare prefix with strs[i]. Trim prefix. Loop.
  • • Vertical Scanning: Compare strs[0][j] with strs[i][j]. Mismatch? Return.
  • • Sorted Advantage: Common prefix of a sorted array is just the overlap of the min and max strings.
