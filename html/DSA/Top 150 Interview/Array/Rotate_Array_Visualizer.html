Rotate Array

LeetCode #189 | Difficulty: Medium | Arrays, Two Pointers

📝 Problem Statement

Given an integer array nums, rotate the array to the right by k steps, where k is non-negative.

Example: nums = [1, 2, 3, 4, 5, 6, 7], k = 3 → [5, 6, 7, 1, 2, 3, 4]

Approach 1: Rotate One-by-One (Brute Force)

Intuition

The simplest way is to perform k separate rotations. In each rotation, we save the last element and shift every other element one position to the right. This is intuitive but involves a lot of redundant movement.

Click "Next Step" to start rotating. (k = 3)
Current Rotation: 0 / 3

Java Implementation

public void rotate(int[] nums, int k) {
    int n = nums.length;
    // Reduce k if it's larger than array length
    k %= n;

    for (int i = 0; i < k; i++) {
        // 1. Save the very last element
        int lastElement = nums[n - 1];
        
        // 2. Shift every element one step to the right
        for (int j = n - 1; j > 0; j--) {
            nums[j] = nums[j - 1];
        }
        
        // 3. Put the saved element at the front
        nums[0] = lastElement;
    }
}

Complexity

Time: O(n * k). We iterate n elements for k rotations.

Space: O(1). No extra array used.

Trade-off

Very slow for large k and n. If k ≈ n, it becomes O(n²).

Approach 2: Extra Array (Memory Trade-off)

Intuition

We can calculate the new position of every element directly using the formula (oldIndex + k) % n. By using an extra array, we can place each element in its correct final spot in a single pass.

Mapping elements to new positions.
Original Array (nums)
New Array (temp)

Java Code

public void rotate(int[] nums, int k) {
    int n = nums.length;
    int[] temp = new int[n];
    
    for (int i = 0; i < n; i++) {
        // Calculate new index using modulo
        int newIndex = (i + k) % n;
        temp[newIndex] = nums[i];
    }
    
    // Copy back to original array
    System.arraycopy(temp, 0, nums, 0, n);
}

Complexity

Time: O(n). One pass to fill temp, one pass to copy back.

Space: O(n). Requires a full copy of the array.

Verdict

Faster than Brute Force, but fails the "in-place" requirement if space is a constraint.

Approach 3: Triple Reverse (Optimal)

Intuition: "The Mirror Magic"

This is the most elegant solution. It involves three simple steps:
1. Reverse the entire array.
2. Reverse the first k elements.
3. Reverse the remaining n-k elements.

Ready for the 3-Step Reverse technique.
1. Reverse All 2. Reverse [0...k-1] 3. Reverse [k...n-1]

Java Code

public void rotate(int[] nums, int k) {
    int n = nums.length;
    k %= n;
    
    // 1. Reverse entire array
    reverse(nums, 0, n - 1);
    
    // 2. Reverse first k elements
    reverse(nums, 0, k - 1);
    
    // 3. Reverse the rest
    reverse(nums, k, n - 1);
}

private void reverse(int[] nums, int start, int end) {
    while (start < end) {
        int tempValue = nums[start];
        nums[start] = nums[end];
        nums[end] = tempValue;
        start++;
        end--;
    }
}

Complexity

Time: O(n). Each element is moved at most twice.

Space: O(1). Entirely in-place.

Key takeaway

The "Reverse Trick" is a common pattern for rotation problems in interviews. Memorize this sequence!

Comparison

Approach Time Space In-Place?
Brute Force O(n * k) O(1) ✅ Yes
Extra Array O(n) O(n) ❌ No
Triple Reverse O(n) O(1) ✅ Yes

⚡ Quick Recall Cheat Sheet

The Reverse Logic

  • • Reverse(0, n-1): Turns [1,2,3,4,5] into [5,4,3,2,1].
  • • Reverse(0, k-1): Puts rotated elements in correct relative order.
  • • Reverse(k, n-1): Puts non-rotated elements in correct relative order.

Edge Cases

  • • k = 0: Array stays the same.
  • • k > n: Use k % n.
  • • n = 1: No change possible.
