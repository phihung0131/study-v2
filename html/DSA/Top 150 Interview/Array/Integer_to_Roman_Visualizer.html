Integer to Roman

LeetCode #12 | Difficulty: Medium | Math, String, Greedy

🔢 Problem Context

Seven symbols represent Roman numerals: I, V, X, L, C, D, M. Roman numerals are usually written largest to smallest from left to right. To represent 4 or 9, we use subtraction: 4 is IV, 9 is IX, 40 is XL, etc.

Goal: Convert an integer between 1 and 3999 into a Roman numeral string.

APPROACH 1 | GREEDY SUBTRACTION

Intuition: "The Largest Piece First"

Roman numerals follow a greedy pattern. We want to use the largest possible symbol value that is less than or equal to our current number. We subtract that value and repeat until the number becomes zero.

Java Implementation

public String intToRoman(int num) {
    // Parallel arrays containing symbols and their integer values
    int[] values = {1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1};
    String[] symbols = {"M", "CM", "D", "CD", "C", "XC", "L", "XL", "X", "IX", "V", "IV", "I"};
    
    StringBuilder romanResult = new StringBuilder();
    
    // Iterate through the values from largest to smallest
    for (int i = 0; i < values.length && num > 0; i++) {
        // While current value can fit into the remaining number
        while (num >= values[i]) {
            num -= values[i];
            romanResult.append(symbols[i]);
        }
    }
    
    return romanResult.toString();
}

Complexity

Time: O(1). Although there is a loop, the number of symbols is fixed and the maximum input is 3999, so the loop runs a constant number of times.

Space: O(1). We only use a fixed amount of extra space for the arrays.

Pros & Cons

Pros: Elegant, easy to explain in interviews, very memory efficient.

Cons: Requires maintaining two parallel arrays manually.

APPROACH 2 | PLACE VALUE MAPPING

Intuition: "Decomposition Buffet"

Every Roman numeral is just a combination of thousands, hundreds, tens, and ones. Since each digit (0-9) has a unique fixed Roman representation for its position, we can pre-calculate all 40 possibilities and just "pick" the right one.

Java Implementation

public String intToRoman(int num) {
    // Arrays representing 0-9 for each decimal place
    String[] thousands = {"", "M", "MM", "MMM"};
    String[] hundreds = {"", "C", "CC", "CCC", "CD", "D", "DC", "DCC", "DCCC", "CM"};
    String[] tens = {"", "X", "XX", "XXX", "XL", "L", "LX", "LXX", "LXXX", "XC"};
    String[] ones = {"", "I", "II", "III", "IV", "V", "VI", "VII", "VIII", "IX"};
    
    // Extract each place value and concatenate
    return thousands[num / 1000] + 
           hundreds[(num % 1000) / 100] + 
           tens[(num % 100) / 10] + 
           ones[num % 10];
}

Complexity

Time: O(1). Only 4 array lookups are performed regardless of input.

Space: O(1). Pre-calculated strings take fixed memory.

The "Hack"

This is the "cheat code" version. It's essentially a lookup table. It's the fastest approach but requires more manual typing to set up the arrays.

⚡ Interview Cheat Sheet

The Greedy Rule

Always handle the subtraction symbols as their own separate values:

  • • CM = 900, CD = 400
  • • XC = 90, XL = 40
  • • IX = 9, IV = 4

Range Check

Input is always 1 to 3999. Why? Because Romans didn't have a standard way to write 4000 or larger (they used bars over symbols), and they had no symbol for 0.
