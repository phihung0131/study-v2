Zigzag Conversion

LeetCode #6 | Difficulty: Medium | String Manipulation

📝 Problem Statement

The string "PAYPALISHIRING" is written in a zigzag pattern on a given number of rows like this:

P . . . A . . . H . . . N
A . P . L . S . I . I . G
Y . . . I . . . R . . . .

And then read line by line: "PAHNAPLSIIGYIR".

Goal: Write the code that will take a string and make this conversion given a number of rows.

Approach 1: Row-by-Row Simulation

Intuition: "The String Elevator"

Imagine numRows buckets. We iterate through the string and place each character in a bucket. We move "down" the buckets until we reach the bottom, then reverse direction and move "up" until we reach the top. Finally, we concatenate all buckets.

Java Implementation

public String convert(String s, int numRows) {
    if (numRows == 1 || s.length() <= numRows) return s;

    // 1. Initialize buckets for each row
    List<StringBuilder> rows = new ArrayList<>();
    for (int i = 0; i < Math.min(numRows, s.length()); i++) {
        rows.add(new StringBuilder());
    }

    int currentRow = 0;
    boolean goingDown = false;

    // 2. Traverse string and simulate the zigzag direction
    for (char c : s.toCharArray()) {
        rows.get(currentRow).append(c);
        
        // Flip direction at the top or bottom row
        if (currentRow == 0 || currentRow == numRows - 1) {
            goingDown = !goingDown;
        }
        
        currentRow += goingDown ? 1 : -1;
    }

    // 3. Combine rows into a single string
    StringBuilder result = new StringBuilder();
    for (StringBuilder row : rows) result.append(row);
    return result.toString();
}

Complexity

Time: O(n). Every character is processed once.

Space: O(n). Storing characters in row-specific buckets.

Verdict

The most intuitive way to solve the problem. Easy to debug and explain in an interview.

Approach 2: Mathematical Jumps

Intuition: "The Cycle Formula"

Instead of simulating, we can calculate which character goes into which row. The pattern repeats every cycle = 2 * numRows - 2 steps. Row 0 and Row n-1 have one character per cycle. Middle rows have two.

Java Implementation

public String convert(String s, int numRows) {
    if (numRows == 1) return s;

    StringBuilder result = new StringBuilder();
    int n = s.length();
    int cycleLen = 2 * numRows - 2;

    for (int i = 0; i < numRows; i++) {
        for (int j = 0; j + i < n; j += cycleLen) {
            // First character in cycle
            result.append(s.charAt(j + i));
            
            // Middle rows have a second character in the cycle
            if (i != 0 && i != numRows - 1 && j + cycleLen - i < n) {
                result.append(s.charAt(j + cycleLen - i));
            }
        }
    }
    return result.toString();
}

Complexity

Time: O(n). Each index is visited exactly once.

Space: O(1) (excluding output). No extra lists or builders used.

Aha! Moment

For a middle row `i`, the two indices in a cycle starting at `j` are `j+i` and `j+cycleLen-i`. This is purely mathematical construction!

⚡ Cheat Sheet

  • • Elevator Pattern: Use a flag goingDown to bounce between 0 and numRows-1.
  • • Cycle Formula: 2 * numRows - 2.
  • • In-between indices: Only exist for rows that are not 0 or numRows-1.
  • • Safety First: Always handle numRows == 1 as an early return.

🧩 Edge Cases

  • numRows = 1: Zigzag is just the straight string.
  • String length < numRows: Just the string.
  • numRows = 2: Even indices in row 0, odd in row 1.
