Length of Last Word

LeetCode #58 | Difficulty: Easy | String Manipulation

📝 Problem Statement

Given a string s consisting of words and spaces, return the length of the last word in the string. A word is a maximal substring consisting of non-space characters only.

Example: Input s = "Hello World " → Output: 5 ("World")

APPROACH 1 | BUILT-IN STRING API

Intuition: "The Split Strategy"

Java's String.split() method automatically identifies word boundaries. By splitting the string by spaces, we get an array of all words. We simply need to access the last element in that array.

Java Implementation

public int lengthOfLastWord(String s) {
    // 1. Split string by spaces into an array
    // Regex " +" handles one or more spaces
    String[] words = s.split(" +");
    
    // 2. If no words exist, return 0
    if (words.length == 0) return 0;
    
    // 3. Get the last word and return its length
    return words[words.length - 1].length();
}

Complexity

Time: O(N). We must traverse the entire string to identify splits.

Space: O(N). We create a new array containing copies of the words.

Pros & Cons

Pros: Highly readable, simple logic.

Cons: Inefficient memory usage due to new array/string objects.

APPROACH 2 | OPTIMAL REVERSE SCAN

Intuition: "The Backwards Leap"

Why process the whole string if we only care about the end? We can iterate backwards:
1. Skip all trailing spaces.
2. Once we hit a character, start counting until we find the next space or the start of the string.

Java Implementation (Optimal)

public int lengthOfLastWord(String s) {
    int lastWordLength = 0;
    int currentIndex = s.length() - 1;

    // 1. Skip trailing spaces
    while (currentIndex >= 0 && s.charAt(currentIndex) == ' ') {
        currentIndex--;
    }

    // 2. Count characters of the last word
    while (currentIndex >= 0 && s.charAt(currentIndex) != ' ') {
        lastWordLength++;
        currentIndex--;
    }

    return lastWordLength;
}

Efficiency

Time: O(N). Worst case we scan the whole string once.

Space: O(1). No extra data structures or copies.

Why it's better?

We only look at the necessary characters. If a string is 1 million characters long with a 5-letter word at the end, we only perform ~10 operations, whereas split() would process the whole thing.

⚡ Memory Hooks & Tips

Interview Key Phrases

  • 1. "Two-Loop Logic": One while loop to skip spaces, one while loop to count.
  • 2. "In-place processing": Emphasize avoiding string copies to show seniority.

Common Pitfalls

  • • Forgetting currentIndex >= 0 check inside the while loops.
  • • Handling strings that contain only spaces.
  • • Not accounting for single-word strings without any spaces.
