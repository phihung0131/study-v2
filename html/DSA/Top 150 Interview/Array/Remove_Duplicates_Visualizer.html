Remove Duplicates from Sorted Array

LeetCode #26 | Difficulty: Easy | In-place Modification

Problem Overview

Given an integer array nums sorted in non-decreasing order, remove the duplicates in-place such that each unique element appears only once. The relative order of the elements should be kept the same.

Constraint: You must do this by modifying the input array in-place with O(1) extra memory.

1

Approach: Shifting (Brute Force)

Intuition

Scan the array from left to right. Every time we find a duplicate (current == next), delete it by shifting all elements to the right of it one position to the left. Repeat until no duplicates remain.

Java Implementation

public int removeDuplicates(int[] nums) {
    int currentSize = nums.length;
    int currentIndex = 0;

    while (currentIndex < currentSize - 1) {
        // If current is same as next, it's a duplicate
        if (nums[currentIndex] == nums[currentIndex + 1]) {
            // Shift everything left to "delete" the duplicate
            for (int k = currentIndex + 1; k < currentSize - 1; k++) {
                nums[k] = nums[k + 1];
            }
            currentSize--; // Reduce effectively visible size
        } else {
            // Only move forward if no duplicate was found at this index
            currentIndex++;
        }
    }
    return currentSize;
}

Complexity

Time: O(n²). For every duplicate, we potentially shift the rest of the array.

Space: O(1). In-place modification.

Why it's inefficient?

The nested loop for shifting is redundant because the array is sorted. We are moving the same data multiple times.

2

Approach: Two Pointers (Optimal)

Intuition

Since the array is sorted, duplicates are always adjacent. We keep a writeIndex (Slow) representing the last unique element found, and a readIndex (Fast) to find new unique elements. When readIndex finds a value different from writeIndex, we advance writeIndex and copy the value.

Java Implementation

public int removeDuplicates(int[] nums) {
    if (nums.length == 0) return 0;
    
    // writePointer points to the last uniquely placed element
    int lastUniqueIndex = 0;
    
    // readPointer scans the array starting from the second element
    for (int currentIndex = 1; currentIndex < nums.length; currentIndex++) {
        // If current element is different from the last unique one...
        if (nums[currentIndex] != nums[lastUniqueIndex]) {
            // Move lastUniqueIndex forward and update value
            lastUniqueIndex++;
            nums[lastUniqueIndex] = nums[currentIndex];
        }
    }
    
    // Number of unique elements is index + 1
    return lastUniqueIndex + 1;
}

Complexity

Time: O(n). Single pass through the array.

Space: O(1). No extra data structures used.

Pros

This is the optimal solution. It leverages the "sorted" property to skip unnecessary work and performs the minimum number of writes possible.

Performance Comparison

Approach Time Space Efficiency
Shifting (Brute Force) O(n²) O(1) Low
Two Pointers (Optimal) O(n) O(1) High

When to use?

Use this whenever you need to process a sorted array and perform in-place updates or removals.
