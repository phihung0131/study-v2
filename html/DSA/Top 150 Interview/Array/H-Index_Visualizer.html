H-Index

LeetCode #274 | Difficulty: Medium | Sorting, Bucket Sort

📚 Problem Definition

The H-Index is a metric that attempts to measure both the productivity and citation impact of a researcher.

A scientist has an index h if h of their n papers have at least h citations each, and the other n − h papers have no more than h citations each.

Example: citations = [3, 0, 6, 1, 5]

Output: 3 (3 papers have at least 3 citations: 3, 6, 5).

Approach 1: Sorting ($O(n \log n)$)

Intuition

If we sort the papers by citations in descending order, the H-Index is simply the point where the number of papers (index + 1) is no longer less than or equal to the citation count.

Java Implementation

public int hIndex(int[] citations) {
    // Sort citations in ascending order
    Arrays.sort(citations);
    int n = citations.length;
    
    // Check from right to left (largest to smallest)
    for (int i = 0; i < n; i++) {
        // Remaining papers count: n - i
        int remainingPapers = n - i;
        if (citations[i] >= remainingPapers) {
            return remainingPapers;
        }
    }
    return 0;
}

Complexity

Time: O(n \log n) due to sorting.

Space: O(1) (ignoring sort stack space).

Note

While sorting is clean, we can achieve better performance by noticing that H-Index can never exceed the total number of papers n.

Approach 2: Bucket Sort ($O(n)$)

Intuition: "The Bucket Strategy"

Since the H-Index is capped at n, we can use an array of size n+1 to count how many papers have 0, 1, ..., n citations. Any paper with more than n citations is treated as having exactly n citations for the calculation. We then sum counts from right to left until the sum is $\geq$ the current bucket index.

Click "Next Step" to distribute citations into buckets.

Java Implementation (Optimal)

public int hIndex(int[] citations) {
    int n = citations.length;
    int[] buckets = new int[n + 1];
    
    // 1. Fill buckets
    for (int c : citations) {
        if (c >= n) {
            buckets[n]++;
        } else {
            buckets[c]++;
        }
    }
    
    // 2. Accumulate from right to left
    int totalPapersCount = 0;
    for (int hCandidate = n; hCandidate >= 0; hCandidate--) {
        totalPapersCount += buckets[hCandidate];
        if (totalPapersCount >= hCandidate) {
            return hCandidate;
        }
    }
    return 0;
}

Complexity

Time: O(n). Two linear passes.

Space: O(n). Extra array of size n+1.

Key Insight

By capping citations at n, we turn an unbounded sorting problem into a bounded counting problem. This is a classic "trade memory for time" scenario.

Strategy Comparison

Approach Time Space Ideal For
Sorting O(n \log n) O(1) Low memory environments
Bucket Sort O(n) O(n) Large datasets (Fastest)

⚡ Interview Cheat Sheet

  • • H-Index Pattern: A value h where at least h items satisfy a condition.
  • • Sorting Hook: Sort descending $\rightarrow$ if cit[i] >= i + 1, keep going.
  • • Bucket Hook: If you see $N$ as a bound, think Counting/Bucket sort.
  • • Common Mistake: Forgetting that citations can be $> n$. Always cap them at $n$ for the bucket.

🧩 Edge Cases

  • [0]: H-Index is 0.
  • [10, 8]: H-Index is 2 (2 papers have $\geq$ 2 citations).
  • [0, 0, 0]: H-Index is 0.
  • [100]: H-Index is 1.
