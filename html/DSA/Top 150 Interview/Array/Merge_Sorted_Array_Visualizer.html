Merge Sorted Array

LeetCode #88 | Difficulty: Easy | Array, Two Pointers, Sorting

Problem Summary

You are given two integer arrays nums1 and nums2, sorted in non-decreasing order, and two integers m and n, representing the number of elements in nums1 and nums2 respectively.

Merge nums2 into nums1 as a single sorted array.

Crucial Constraint:

The final sorted array should not be returned by the function, but instead be stored inside the array nums1. nums1 has a length of m + n, where the first m elements denote the elements that should be merged, and the last n elements are set to 0 and should be ignored.

Approach 1: Merge and Sort (Brute Force)

Intuition

The simplest way to solve this is to follow the instructions literally: 1. Fill the empty slots at the end of nums1 with the elements from nums2. 2. Sort the entire nums1 array using a built-in sorting algorithm. While this works, it doesn't take advantage of the fact that the input arrays are already sorted.

Java Implementation

class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        // 1. Copy nums2 into the end of nums1
        for (int i = 0; i < n; i++) {
            nums1[m + i] = nums2[i];
        }

        // 2. Sort the entire array
        // Time Complexity: O((m+n) log(m+n))
        Arrays.sort(nums1);
    }
}

Complexity Analysis

  • Time Complexity: O((m + n) log(m + n)) due to the sorting step.
  • Space Complexity: O(log(m + n)) or O(1) depending on the sort implementation (e.g., QuickSort space).

Pros & Cons

  • Pros: Extremely simple to write; uses built-in library methods.
  • Cons: Slowest performance; ignores the "sorted" property of inputs.
Approach 2: Three Pointers (Optimal - Reverse Order)

Intuition

If we try to merge from the start, we would have to shift elements in nums1 to make room, which is inefficient O(n^2).

The Insight: Start from the end! Since the back of nums1 is empty (zeros), we can compare the largest elements from both arrays and place the winner at the very end of nums1. This way, we never overwrite data we haven't processed yet.

Java Implementation

class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        // Set pointers at the last elements to be compared
        int firstPointer = m - 1;
        int secondPointer = n - 1;
        
        // Set pointer at the last index of the merged result
        int writePointer = m + n - 1;
        
        // Iterate as long as there are elements in both arrays
        while (firstPointer >= 0 && secondPointer >= 0) {
            // Compare the elements and place the larger one at the back
            if (nums1[firstPointer] > nums2[secondPointer]) {
                nums1[writePointer] = nums1[firstPointer];
                firstPointer--;
            } else {
                nums1[writePointer] = nums2[secondPointer];
                secondPointer--;
            }
            writePointer--;
        }
        
        // If there are leftover elements in nums2, copy them
        // (Leftovers in nums1 are already in place)
        while (secondPointer >= 0) {
            nums1[writePointer] = nums2[secondPointer];
            secondPointer--;
            writePointer--;
        }
    }
}

Complexity Analysis

  • Time Complexity: O(m + n). We iterate through each element once.
  • Space Complexity: O(1). We modify the array in-place without extra space.

Pros & Cons

  • Pros: Most efficient possible solution. No extra memory.
  • Cons: Slightly harder to implement than sorting.

Efficiency Comparison

Approach Time Space Difficulty
Merge and Sort O((m+n) log(m+n)) O(1) / O(log n) Beginner
Two Pointers (Optimal) O(m+n) O(1) Intermediate

💡 Key Takeaways

  • PATTERN When merging two sorted arrays into one that has extra space at the end, work backwards.
  • TIP Always check if one pointer finishes before the other. Leftover nums2 elements must be copied.
  • TIP Leftover nums1 elements are already sorted and in their correct final positions.

⚠️ Edge Cases

  • n = 0: nums2 is empty. nums1 is already correctly merged.
  • m = 0: nums1 has no valid elements. Just copy all of nums2 into nums1.
  • Duplicate Values: Arrays with identical numbers should be handled by the >= or <= logic.
  • Single Element Arrays: m=1, n=0 or m=0, n=1.
