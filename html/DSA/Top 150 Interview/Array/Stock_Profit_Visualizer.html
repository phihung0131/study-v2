Best Time to Buy and Sell Stock

LeetCode #121 | Difficulty: Easy | Array, Dynamic Programming

📊 Problem Summary

You are given an array prices where prices[i] is the price of a given stock on the i-th day. You want to maximize your profit by choosing a single day to buy one stock and choosing a different day in the future to sell that stock.

Constraint: You cannot sell a stock before you buy one.

Approach 1: Brute Force (Nested Loops)

Intuition

To find the maximum profit, we can check every possible pair of (Buy Day, Sell Day). For each day i (buying day), we check every day j > i (selling day) and calculate the profit. We keep track of the highest profit found so far.

Java Implementation

public int maxProfit(int[] prices) {
    int maxProfitFound = 0;
    
    // i is the buying day
    for (int i = 0; i < prices.length - 1; i++) {
        // j is the selling day (must be after buying day)
        for (int j = i + 1; j < prices.length; j++) {
            int currentProfit = prices[j] - prices[i];
            
            // Update global max if we found a better deal
            if (currentProfit > maxProfitFound) {
                maxProfitFound = currentProfit;
            }
        }
    }
    return maxProfitFound;
}

Complexity Analysis

Time Complexity: O(n²). We use nested loops to check all possible pairs (n*(n-1)/2).

Space Complexity: O(1). We only store the max profit variable.

Pros & Cons

Cons: Extremely inefficient for large input sizes (e.g., 100,000 prices). It will likely result in a Time Limit Exceeded (TLE) error on LeetCode.

Approach 2: One Pass (Greedy/DP)

Intuition

We only care about the lowest price we've seen before the current day. As we iterate through the array once:
1. Keep track of the minimum price seen so far.
2. For every new price, calculate the profit if we sold today using that minimum price.
3. Update the maximum profit accordingly.

Java Implementation

public int maxProfit(int[] prices) {
    if (prices == null || prices.length == 0) return 0;
    
    // Initialize minPrice to a very large value
    int minPriceSeenSoFar = Integer.MAX_VALUE;
    int maxProfitAchieved = 0;
    
    for (int i = 0; i < prices.length; i++) {
        int currentDayPrice = prices[i];
        
        // If today's price is the lowest we've seen, update minPrice
        if (currentDayPrice < minPriceSeenSoFar) {
            minPriceSeenSoFar = currentDayPrice;
        } 
        // Otherwise, see if selling today gives us a better profit
        else if (currentDayPrice - minPriceSeenSoFar > maxProfitAchieved) {
            maxProfitAchieved = currentDayPrice - minPriceSeenSoFar;
        }
    }
    
    return maxProfitAchieved;
}

Complexity Analysis

Time Complexity: O(n). We only traverse the prices array once.

Space Complexity: O(1). We use a constant amount of extra space (two variables).

Key Insight

By keeping track of the minimum price, we effectively "compress" all previous days into a single state, allowing us to make an optimal decision for the current day instantly.

⚡ Cheat Sheet

  • • Pattern: Minimum Tracker.
  • • Keyword: Profit = High - Low (where Low comes first).
  • • Logic: If price < min, update min. Else, calculate price - min and update maxProfit.

⚠️ Edge Cases

  • Decreasing Prices: [7, 6, 4, 3, 1]. Result: 0 (No profit possible).
  • Single Day: [5]. Result: 0 (Cannot buy and sell same day).
  • All Same: [3, 3, 3]. Result: 0.
