Jump Game II

LeetCode #45 | Difficulty: Medium | Minimum Jumps Strategy

📋 Problem Statement

You are given a 0-indexed array of integers nums of length n. You are initially positioned at nums[0]. Each element nums[i] represents the maximum length of a forward jump from index i.

Goal: Return the minimum number of jumps to reach the last index.

Note: You can assume that you can always reach the last index.

Approach 1: Recursive Backtracking

Intuition

To find the minimum jumps, we can try every possible jump from our current index. From each landing spot, we recursively ask: "What's the minimum jumps to reach the end from here?" We then add 1 to that result and keep the minimum across all possible landing spots.

Java Implementation

public int jump(int[] nums) {
    return findMinJumps(nums, 0);
}

private int findMinJumps(int[] nums, int currentIndex) {
    // Base case: if we reached the last index, no more jumps needed
    if (currentIndex >= nums.length - 1) {
        return 0;
    }

    int minJumps = 10001; // Large value representing infinity
    int maxJumpLength = nums[currentIndex];

    // Try all possible jump lengths from 1 to maxJumpLength
    for (int jumpLength = 1; jumpLength <= maxJumpLength; jumpLength++) {
        int jumpsFromNextPosition = findMinJumps(nums, currentIndex + jumpLength);
        if (jumpsFromNextPosition != 10001) {
            minJumps = Math.min(minJumps, 1 + jumpsFromNextPosition);
        }
    }
    return minJumps;
}

Complexity

  • Time: O(N!) or O(2ⁿ). The number of paths explored grows exponentially.
  • Space: O(N). Recursive call stack depth.

Problem

This solution recalculates the same subproblems repeatedly. For example, index 3 might be reached via [0->3] or [0->1->3], but the cost from 3 to end is calculated twice.

Approach 2: DP (Tabulation)

Intuition

We can store the minimum jumps needed to reach every index in a table `dp`. We start from index 0 (cost is 0). For each index, we look at all indices it can jump to and update their minimum jump counts if we've found a shorter path.

Java Implementation

public int jump(int[] nums) {
    int n = nums.length;
    int[] minJumpsToReach = new int[n];
    Arrays.fill(minJumpsToReach, Integer.MAX_VALUE);
    
    // Start at index 0 requires 0 jumps
    minJumpsToReach[0] = 0;
    
    for (int current = 0; current < n; current++) {
        int maxJump = nums[current];
        
        // Update all reachable indices from 'current'
        for (int nextIdx = current + 1; nextIdx <= current + maxJump && nextIdx < n; nextIdx++) {
            if (minJumpsToReach[current] != Integer.MAX_VALUE) {
                minJumpsToReach[nextIdx] = Math.min(minJumpsToReach[nextIdx], minJumpsToReach[current] + 1);
            }
        }
    }
    return minJumpsToReach[n - 1];
}

Complexity

  • Time: O(N²). We use a nested loop to update reachable neighbors.
  • Space: O(N). Array to store min jumps for each position.

Efficiency

Much better than recursion, but still slow for very large arrays (e.g., N=100,000).

Approach 3: Greedy Range Exploration

Intuition: "The Farthest Reach"

Imagine jumping as expanding your "range." At each step, we look at all indices in our current jump range and find the farthest index any of them can jump to. Once we exhaust our current jump range, we MUST take another jump to enter the new expanded range.

Java Implementation (Optimal)

public int jump(int[] nums) {
    int totalJumps = 0;
    int currentJumpEnd = 0;
    int farthestReachableIndex = 0;

    // Iterate until the second-to-last index
    for (int currentIndex = 0; currentIndex < nums.length - 1; currentIndex++) {
        // Update the farthest index reachable from current scan
        int possibleReach = currentIndex + nums[currentIndex];
        farthestReachableIndex = Math.min(farthestReachableIndex, possibleReach); // Wait, logically it's max:
        farthestReachableIndex = Math.max(farthestReachableIndex, possibleReach);

        // If we have reached the end of our current jump range
        if (currentIndex == currentJumpEnd) {
            // We MUST jump to the farthest point we found
            totalJumps++;
            currentJumpEnd = farthestReachableIndex;
        }
    }
    return totalJumps;
}

Complexity Analysis

  • Time: O(N). Single pass through the array.
  • Space: O(1). Constant auxiliary variables.

The "Aha!" Moment

This is essentially a Breadth-First Search (BFS) implemented greedily. Each jump moves you to the next "level" of reachable indices. `currentJumpEnd` represents the current level's boundary.

Efficiency Comparison

⚡ Interview Cheat Sheet

  • • Pattern: Greedy Level-by-Level exploration.
  • • Goal: Find min jumps, not "is reachable".
  • • Logic: Iterate through the array, updating the farthest reach. If you reach the current end, increment jumps and update end to the new farthest.
  • • Boundary: Stop iterating at nums.length - 1 because if you reach the last index, you don't need another jump.

🧩 Edge Cases

  • [0]: Jumps = 0. Already at the end.
  • [2, 3, 1, 1, 4]: Jumps = 2. Optimal path 0 -> 1 -> 4.
  • Single Element: Array with size 1 always returns 0.
  • Maximum Jumps: `nums[i]` being very large (e.g., 1,000,000) doesn't break the Greedy logic.
