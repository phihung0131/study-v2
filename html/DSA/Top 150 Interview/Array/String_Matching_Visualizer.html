Find the Index of First Occurrence

LeetCode #28 | Difficulty: Easy | Two Pointers, String Matching

📝 Problem Statement

Given two strings needle and haystack, return the index of the first occurrence of needle in haystack, or -1 if needle is not part of haystack.

Example: haystack = "sadbutsad", needle = "sad" → Output: 0

Approach 1: Naive Brute Force

Intuition: "Slide and Check"

Iterate through the haystack. At each position, check if the needle matches character by character. If a mismatch occurs, "reset" and move the needle window just one step forward in the haystack.

Java Implementation

public int strStr(String haystack, String needle) {
    int haystackLen = haystack.length();
    int needleLen = needle.length();

    // Iterate until there's not enough space left for the needle
    for (int start = 0; start <= haystackLen - needleLen; start++) {
        int j = 0;
        // Check every character of the needle
        while (j < needleLen && haystack.charAt(start + j) == needle.charAt(j)) {
            j++;
        }
        // If we matched the full needle length, we found the first occurrence
        if (j == needleLen) {
            return start;
        }
    }
    return -1;
}

Complexity

Time: O(n * m). In the worst case (e.g., haystack="aaaaaa", needle="aaab"), we check almost every combination.

Space: O(1). No extra memory structures used.

Verdict

The baseline solution. It works for small strings but fails to optimize for patterns within the strings.

Approach 2: Java API (Substring)

Intuition: "Block Match"

Instead of manual character loops, we use Java's substring method to extract a block of the haystack and compare it directly to the needle. This is often cleaner to write but performs similar work under the hood.

Java Implementation

public int strStr(String haystack, String needle) {
    int n = haystack.length();
    int m = needle.length();

    for (int i = 0; i <= n - m; i++) {
        // Compare needle with the haystack segment from i to i + m
        if (haystack.substring(i, i + m).equals(needle)) {
            return i;
        }
    }
    return -1;
}

Complexity

Time: O(n * m). Substring creation and comparison both take linear time.

Space: O(m). substring() creates a new string object in memory.

Note

This approach is readable but "heavier" on memory allocation. In high-performance systems, the character-by-character Brute Force is actually slightly better.

Approach 3: KMP (Knuth-Morris-Pratt)

Intuition: "The Intelligent Skip"

KMP avoids redundant work by observing the needle itself. We precompute an "LPS" (Longest Proper Prefix which is also a Suffix) array. If we mismatch at character 5, and we know characters 0-2 match 3-5, we don't need to re-check 0-2!

Java Implementation (KMP)

public int strStr(String haystack, String needle) {
    if (needle.isEmpty()) return 0;
    
    // 1. Build LPS (Longest Proper Prefix which is also a Suffix)
    int[] lps = buildLPS(needle);
    
    int i = 0; // pointer for haystack
    int j = 0; // pointer for needle
    
    while (i < haystack.length()) {
        if (haystack.charAt(i) == needle.charAt(j)) {
            i++; j++;
            if (j == needle.length()) return i - j; // Found match
        } else {
            if (j != 0) {
                j = lps[j - 1]; // Smart skip using precomputed LPS
            } else {
                i++;
            }
        }
    }
    return -1;
}

private int[] buildLPS(String pattern) {
    int[] lps = new int[pattern.length()];
    int len = 0;
    int i = 1;
    while (i < pattern.length()) {
        if (pattern.charAt(i) == pattern.charAt(len)) {
            lps[i++] = ++len;
        } else if (len != 0) {
            len = lps[len - 1];
        } else {
            lps[i++] = 0;
        }
    }
    return lps;
}

Complexity

Time: O(n + m). We only ever move the haystack pointer forward.

Space: O(m). Extra array to store the LPS mapping.

Interview Pro Tip

Mentioning KMP shows high algorithmic maturity. Even if you don't implement it fully, explaining how "LPS lets us skip indices by leveraging pattern symmetry" is a massive win.

⚡ Recall Cheat Sheet

  • • Sliding Window: Basic approach. Shift 1 by 1.
  • • Pattern Matching: If needle repeats (e.g. "abab"), use KMP.
  • • KMP Logic: Mismatch at index $j$ in needle? Jump to lps[j-1].
  • • LPS Array: Tells you where the needle could "re-start" its match without going back to the beginning.

🧩 Edge Cases

  • Needle > Haystack: Impossible to match, return -1.
  • Empty Needle: By LeetCode convention, return 0.
  • All characters same: Needle="aaa", Haystack="aaaa".
  • Mismatch at the end: Needle="abc", Haystack="abx".
