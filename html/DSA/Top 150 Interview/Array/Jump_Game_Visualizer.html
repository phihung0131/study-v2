Jump Game

LeetCode #55 | Difficulty: Medium

📝 Problem Summary

You are given an integer array nums. You are initially positioned at the array's first index, and each element in the array represents your maximum jump length at that position.

Return true if you can reach the last index, or false otherwise.

Example: [2, 3, 1, 1, 4] -> True (Jump 1 step from index 0 to 1, then 3 steps to the end).

Approach 1: Backtracking (Brute Force)

Intuition

The most naive way is to try every possible jump from the current position. If we are at index i and nums[i] = 3, we recursively try jumping 1 step, 2 steps, and 3 steps. This explores a massive decision tree.

Java Implementation

public boolean canJump(int[] nums) {
    return canJumpFromIndex(0, nums);
}

private boolean canJumpFromIndex(int position, int[] nums) {
    // Base case: if we reached the last index
    if (position == nums.length - 1) {
        return true;
    }

    // Calculate the furthest jump possible
    int furthestJump = Math.min(position + nums[position], nums.length - 1);

    // Try every jump distance from 1 to furthestJump
    for (int nextPosition = position + 1; nextPosition <= furthestJump; nextPosition++) {
        if (canJumpFromIndex(nextPosition, nums)) {
            return true;
        }
    }

    return false;
}

Complexity

Time: O(2ⁿ). In the worst case, every index can jump to every subsequent index.

Space: O(n). Recursion stack depth.

Pros & Cons

Pros: Correct logical flow. Cons: Extremely slow; redundant work solving the same index multiple times.

Approach 2: DP (Bottom-Up)

Intuition

Instead of recursion, we can use a Memoization table. We work backwards from the last index. If an index i can reach any index that is already marked as REACHABLE, then index i is also REACHABLE.

Java Implementation

public boolean canJump(int[] nums) {
    Boolean[] memo = new Boolean[nums.length];
    memo[nums.length - 1] = true; // The end is reachable from the end

    for (int i = nums.length - 2; i >= 0; i--) {
        int furthestJump = Math.min(i + nums[i], nums.length - 1);
        for (int j = i + 1; j <= furthestJump; j++) {
            if (memo[j] != null && memo[j]) {
                memo[i] = true;
                break;
            }
        }
    }
    return memo[0] != null && memo[0];
}

Complexity

Time: O(n²). For each index, we look ahead up to n steps.

Space: O(n). To store the memo table.

Efficiency

Greatly improved! We eliminate redundant recursive calls by saving results. However, we still have nested loops.

Approach 3: Greedy (Goal Post Shifting)

Intuition: "The Shifting Target"

Instead of a table, we only need to keep track of the left-most position that can reach the end. Let's call this our Goal Post. Initially, the Goal Post is at the last index. If we find an index that can jump to the Goal Post, that index becomes our new Goal Post.

Java Implementation

public boolean canJump(int[] nums) {
    // Start the goal at the very last index
    int lastReachablePosition = nums.length - 1;

    // Iterate backwards from right to left
    for (int i = nums.length - 1; i >= 0; i--) {
        // If current index + max jump >= current goal post
        if (i + nums[i] >= lastReachablePosition) {
            // Shift the goal post to our current index
            lastReachablePosition = i;
        }
    }

    // If the goal post reached the start (index 0), success!
    return lastReachablePosition == 0;
}

Complexity Analysis

Time: O(n). One single pass through the array.

Space: O(1). Only one integer variable needed.

Key Insight

"Can I get to the current 'safe' spot from here?" If yes, then I am now the new 'safe' spot. This eliminates the need for any nested loops or extra memory.

Efficiency Comparison

Approach Time Space Verdict
Backtracking O(2ⁿ) O(n) TLE for n > 20
Dynamic Programming O(n²) O(n) Solid Improvement
Greedy O(n) O(1) Optimal

⚡ Interview Cheat Sheet

  • • Pattern: Shifting Goal Post (Greedy).
  • • Keyword: "Farthest Reach" or "Last Reachable Index".
  • • Logic: Work backwards. If you can jump to a known good spot, your current spot is now the closest good spot.
  • • Complexity Tip: Always mention the O(n) Greedy solution after explaining why O(2ⁿ) Backtracking is bad.

🧩 Edge Cases

  • [0]: True (Already at the end).
  • [3, 0, 0, 0]: True (Can reach the end exactly).
  • [1, 0, 1, 0]: False (Stuck at the second zero).
  • Huge Numbers: [1000, 0, 0...]. Greedy handles large jump values perfectly.
