54. Spiral Matrix

Given an m x n matrix, return all elements of the matrix in spiral order.

🌀 Problem Essence

Imagine you are walking along the edges of a garden, turning clockwise every time you hit a wall or a path you've already walked. You start at the top-left and move right until you can't anymore, then down, then left, then up, repeating the cycle.

Key Challenge: Knowing exactly when to turn and when to stop to avoid infinite loops or array out-of-bounds errors.

Approach 1: Simulation with Visited Matrix

The intuitive approach: move in a direction until you hit a boundary or a visited cell, then rotate 90° clockwise.

Java Implementation (Visited Matrix)

public List<Integer> spiralOrder(int[][] matrix) {
    List<Integer> result = new ArrayList<>();
    if (matrix.length == 0) return result;

    int rows = matrix.length, cols = matrix[0].length;
    boolean[][] visited = new boolean[rows][cols];
    
    // Right, Down, Left, Up
    int[] dRow = {0, 1, 0, -1};
    int[] dCol = {1, 0, -1, 0};
    
    int currentRow = 0, currentCol = 0, directionIdx = 0;

    for (int i = 0; i < rows * cols; i++) {
        result.add(matrix[currentRow][currentCol]);
        visited[currentRow][currentCol] = true;
        
        // Predict next step
        int nextRow = currentRow + dRow[directionIdx];
        int nextCol = currentCol + dCol[directionIdx];
        
        // Turn if out of bounds OR already visited
        if (nextRow >= 0 && nextRow < rows && nextCol >= 0 && nextCol < cols && !visited[nextRow][nextCol]) {
            currentRow = nextRow;
            currentCol = nextCol;
        } else {
            directionIdx = (directionIdx + 1) % 4;
            currentRow += dRow[directionIdx];
            currentCol += dCol[directionIdx];
        }
    }
    return result;
}

Complexity Analysis

Time Complexity: O(m * n) - We visit every single cell in the matrix exactly once.

Space Complexity: O(m * n) - We maintain a boolean visited matrix of the same dimensions as the input.

Pros & Cons

Pros: Very intuitive, follows the physical movement naturally.

Cons: Uses extra memory for the visited matrix, which isn't strictly necessary.

Approach 2: Optimal Boundary Shrinking

The "Space-Efficient" way: maintain four pointers (top, bottom, left, right) and shrink the search space after each side is completed.

Java Implementation (Boundary Shrink)

public List<Integer> spiralOrder(int[][] matrix) {
    List<Integer> result = new ArrayList<>();
    int topRow = 0;
    int bottomRow = matrix.length - 1;
    int leftCol = 0;
    int rightCol = matrix[0].length - 1;

    while (topRow <= bottomRow && leftCol <= rightCol) {
        // 1. Traverse Right
        for (int j = leftCol; j <= rightCol; j++) 
            result.add(matrix[topRow][j]);
        topRow++;

        // 2. Traverse Down
        for (int i = topRow; i <= bottomRow; i++) 
            result.add(matrix[i][rightCol]);
        rightCol--;

        if (topRow <= bottomRow) {
            // 3. Traverse Left
            for (int j = rightCol; j >= leftCol; j--) 
                result.add(matrix[bottomRow][j]);
            bottomRow--;
        }

        if (leftCol <= rightCol) {
            // 4. Traverse Up
            for (int i = bottomRow; i >= topRow; i--) 
                result.add(matrix[i][leftCol]);
            leftCol++;
        }
    }
    return result;
}

Complexity Analysis

Time Complexity: O(m * n) - Constant work per cell.

Space Complexity: O(1) - Excluding the result list, we only use 4 integer variables for boundaries.

The "Edge" Protection

The crucial part of this algorithm is the if checks before traversing Left and Up. Since we modify topRow and rightCol inside the while loop, the condition might change mid-iteration.

💡 Interview Cheat Sheet

Direction Cycling

In a spiral, the sequence is always Right → Down → Left → Up. Use direction arrays: `dr = {0, 1, 0, -1}` and `dc = {1, 0, -1, 0}`.

Why Boundary Shrink?

It's the optimal answer. By eliminating the visited matrix, you reduce auxiliary space from O(N) to O(1).

Termination Condition

The loop continues as long as `left <= right` and `top <= bottom`. Always re-check these conditions before going back left or up.

Edge Cases to Consider

Single Row Matrix (1 x n) Single Column Matrix (m x 1) Single Element (1 x 1) Rectangle (Wide or Tall)
