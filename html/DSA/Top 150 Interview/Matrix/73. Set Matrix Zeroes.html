73. Set Matrix Zeroes

Given an m x n integer matrix, if an element is 0, set its entire row and column to 0's. Do it in-place.

❗ Constraints & Challenges

The primary challenge isn't finding the zeroes, but updating the matrix without accidentally triggering new zeroes. If you set a row to zero as you see a zero, you'll eventually turn the whole matrix into zeroes!

Goal: Use minimal auxiliary space. A brute force solution uses O(MN), a better one uses O(M+N), and the optimal uses O(1) extra space.

Approach 1: Auxiliary Matrix (Brute Force)

Create a copy of the original matrix to keep track of where the original zeroes were.

Java Implementation

public void setZeroes(int[][] matrix) {
    int m = matrix.length;
    int n = matrix[0].length;
    // Create a full copy
    int[][] copy = new int[m][n];
    for (int i = 0; i < m; i++)
        copy[i] = matrix[i].clone();

    for (int r = 0; r < m; r++) {
        for (int c = 0; c < n; c++) {
            if (copy[r][c] == 0) {
                // Zero out row r in original
                for (int i = 0; i < n; i++) matrix[r][i] = 0;
                // Zero out col c in original
                for (int i = 0; i < m; i++) matrix[i][c] = 0;
            }
        }
    }
}

Complexity Analysis

Time: O(M * N) - We visit every cell.

Space: O(M * N) - We store an entire duplicate matrix.

Pros & Cons

Pros: Easiest to implement, no index confusion.

Cons: Extremely memory inefficient for large matrices.

Approach 2: Auxiliary Marker Arrays

Instead of a full matrix, use two simple arrays to track which rows and columns should be zeroed.

Java Implementation

public void setZeroes(int[][] matrix) {
    int m = matrix.length;
    int n = matrix[0].length;
    boolean[] rowMarkers = new boolean[m];
    boolean[] colMarkers = new boolean[n];

    // Step 1: Identify zeroes
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (matrix[i][j] == 0) {
                rowMarkers[i] = true;
                colMarkers[j] = true;
            }
        }
    }

    // Step 2: Zero out based on markers
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (rowMarkers[i] || colMarkers[j]) {
                matrix[i][j] = 0;
            }
        }
    }
}

Complexity Analysis

Time: O(M * N) - Two passes over the matrix.

Space: O(M + N) - We only need arrays for height and width.

Why this is better?

Instead of duplicating the whole matrix (M*N), we use linear space. For a 1000x1000 matrix, this is 2,000 markers versus 1,000,000 elements!

Approach 3: Optimal In-Place Markers

The clever trick: Use the first row and first column of the matrix itself as your marker arrays.

Animation: In-Place Logic

Java Implementation

public void setZeroes(int[][] matrix) {
    int m = matrix.length, n = matrix[0].length;
    boolean firstRowZero = false, firstColZero = false;

    // 1. Check if first row/col have zeroes
    for (int i = 0; i < m; i++) if (matrix[i][0] == 0) firstColZero = true;
    for (int j = 0; j < n; j++) if (matrix[0][j] == 0) firstRowZero = true;

    // 2. Use first row/col as markers for the rest
    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            if (matrix[i][j] == 0) {
                matrix[i][0] = 0;
                matrix[0][j] = 0;
            }
        }
    }

    // 3. Zero out based on markers
    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            if (matrix[i][0] == 0 || matrix[0][j] == 0) matrix[i][j] = 0;
        }
    }

    // 4. Finally handle first row/col markers
    if (firstColZero) for (int i = 0; i < m; i++) matrix[i][0] = 0;
    if (firstRowZero) for (int j = 0; j < n; j++) matrix[0][j] = 0;
}

Complexity Analysis

Time: O(M * N)

Space: O(1) - Truly in-place, no extra storage.

Why handle first row/col first?

Since we use the first row/col as markers, their original values get overwritten. We must remember if they originally had zeroes before we start using them as a "scratchpad".

💡 Interview Cheat Sheet

Identify the Pitfall

"I can't update row-by-row because I'll create 'fake' zeroes that propagate throughout the matrix."

Space Optimization

Start with O(M*N), then suggest O(M+N). If they ask for O(1), use the first row/column trick.

The "Overlap" Element

Matrix[0][0] is shared by the first row and first column. Handle these status flags separately to avoid bugs.

Edge Cases to Consider

Matrix with NO zeroes Matrix with ALL zeroes 1-row or 1-column matrix Zero in matrix[0][0]
