36. Valid Sudoku

Determine if a 9 x 9 Sudoku board is valid. Only the filled cells need to be validated according to the three rules of Sudoku.

📝 Problem Rules

  • 1. Each row must contain the digits 1-9 without repetition.
  • 2. Each column must contain the digits 1-9 without repetition.
  • 3. Each of the nine 3 x 3 sub-boxes of the grid must contain the digits 1-9 without repetition.

Note: A Sudoku board (partially filled) could be valid but is not necessarily solvable. Only the filled cells need to be validated.

Approach 1: Three Pass Iteration (HashSet)

The most intuitive way: check each row, then each column, then each 3x3 box independently using sets.

Java Implementation

public boolean isValidSudoku(char[][] board) {
    // Check Rows
    for (int row = 0; row < 9; row++) {
        Set<Character> rowSet = new HashSet<>();
        for (int col = 0; col < 9; col++) {
            if (!checkValid(board[row][col], rowSet)) return false;
        }
    }

    // Check Columns
    for (int col = 0; col < 9; col++) {
        Set<Character> colSet = new HashSet<>();
        for (int row = 0; row < 9; row++) {
            if (!checkValid(board[row][col], colSet)) return false;
        }
    }

    // Check 3x3 Boxes
    for (int box = 0; box < 9; box++) {
        Set<Character> boxSet = new HashSet<>();
        for (int i = 0; i < 3; i++) {
            for (int j = 0; j < 3; j++) {
                int r = (box / 3) * 3 + i;
                int c = (box % 3) * 3 + j;
                if (!checkValid(board[r][c], boxSet)) return false;
            }
        }
    }
    return true;
}

private boolean checkValid(char val, Set<Character> set) {
    if (val == '.') return true;
    if (set.contains(val)) return false;
    set.add(val);
    return true;
}

Complexity Analysis

Time Complexity: O(1) - Technically, the board is fixed at 81 cells (9x9). If we consider N as the dimension, it is O(N²).

Space Complexity: O(1) - The sets store at most 9 characters at a time.

Pros & Cons

Pros: Easy to debug, clear separation of logic.

Cons: Traverses the board three separate times; less efficient memory management with multiple Set objects.

Approach 2: Single Pass (Bitmasks / Arrays)

The "Pro" way: validate rows, columns, and sub-boxes in a single traversal using integer arrays as frequency trackers.

Java Implementation

public boolean isValidSudoku(char[][] board) {
    // We use a boolean array [9][9] to track if digit k has appeared in row/col/box i
    boolean[][] rows = new boolean[9][9];
    boolean[][] cols = new boolean[9][9];
    boolean[][] boxes = new boolean[9][9];

    for (int r = 0; r < 9; r++) {
        for (int c = 0; c < 9; c++) {
            if (board[r][c] == '.') continue;

            int num = board[r][c] - '1'; // convert '1'-'9' to 0-8 index
            int boxIndex = (r / 3) * 3 + (c / 3);

            if (rows[r][num] || cols[c][num] || boxes[boxIndex][num]) {
                return false; // Duplicate detected!
            }

            rows[r][num] = true;
            cols[c][num] = true;
            boxes[boxIndex][num] = true;
        }
    }
    return true;
}

Complexity Analysis

Time Complexity: O(N²) - Still O(81) = O(1). We visit each cell exactly once.

Space Complexity: O(N²) - Still O(3 * 81) = O(1). We pre-allocate tracking arrays.

Why this is faster?

1. Single pass: Only 81 iterations instead of 243.

2. Primitive arrays vs Objects: Boolean arrays are much faster and use less memory than `HashSet<Character>` which involves boxing and hashing.

Comparison Table

Approach Time Space Pros
Three Pass HashSet O(1) O(1) Simple, logic separated
Single Pass Array O(1) O(1) Fastest, efficient memory

💡 Interview Cheat Sheet

Box Indexing Formula

To convert (row, col) to a box index (0-8):
idx = (r / 3) * 3 + (c / 3)

The "Valid" nuance

A "Valid" board doesn't mean it's "Solvable". Don't try to solve it; just check for duplicate numbers in rows, cols, and boxes.

Optimal Pattern

Use 2D boolean arrays to store states. `row[i][num]` means digit `num` exists in row `i`. This is O(1) lookup.

Edge Cases to Consider

Empty Board (all '.') Board with only one number Board that looks valid but cannot be solved Duplicates in different boxes but same row
