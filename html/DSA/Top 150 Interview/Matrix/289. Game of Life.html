289. Game of Life

Implement Conway's Game of Life. The board updates simultaneously: every cell's next state depends on its current 8 neighbors.

📜 The 4 Golden Rules

1. Under-population: Live cell with < 2 live neighbors ➜ Dies.

2. Survival: Live cell with 2 or 3 live neighbors ➜ Lives.

3. Over-population: Live cell with > 3 live neighbors ➜ Dies.

4. Reproduction: Dead cell with exactly 3 live neighbors ➜ Becomes Alive.

Approach 1: Using an Extra Copy

The most straightforward method: read from a static copy of the board to calculate new states for the original board.

Java Implementation (O(MN) Space)

public void gameOfLife(int[][] board) {
    int rows = board.length;
    int cols = board[0].length;

    // Create a copy to store the current state
    int[][] copyBoard = new int[rows][cols];
    for (int r = 0; r < rows; r++) {
        for (int c = 0; c < cols; c++) {
            copyBoard[r][c] = board[r][c];
        }
    }

    // Iterate through board cells
    for (int r = 0; r < rows; r++) {
        for (int c = 0; c < cols; c++) {
            int liveNeighbors = countLiveNeighbors(copyBoard, r, c);

            // Apply Rules based on copyBoard state
            if (copyBoard[r][c] == 1 && (liveNeighbors < 2 || liveNeighbors > 3)) {
                board[r][c] = 0; // Dies
            } else if (copyBoard[r][c] == 0 && liveNeighbors == 3) {
                board[r][c] = 1; // Reproduces
            }
        }
    }
}

Complexity Analysis

Time Complexity: O(M * N) - We visit every cell once for copying and once for processing.

Space Complexity: O(M * N) - We allocate an entire duplicate grid.

Pros & Cons

Pros: Simple, readable, handles "simultaneous" updates easily.

Cons: Large memory footprint for high-resolution boards.

Approach 2: State Encoding (In-Place)

The expert way: use transition markers to store both the "current" and "next" states in a single integer cell.

Java Implementation (O(1) Space)

public void gameOfLife(int[][] board) {
    int rows = board.length, cols = board[0].length;

    // Pass 1: Encode states
    for (int r = 0; r < rows; r++) {
        for (int c = 0; c < cols; c++) {
            int liveNeighbors = countLiveNeighbors(board, r, c);

            // If cell is alive and dies, mark as -1
            if (board[r][c] == 1 && (liveNeighbors < 2 || liveNeighbors > 3)) {
                board[r][c] = -1;
            }
            // If cell is dead and lives, mark as 2
            if (board[r][c] == 0 && liveNeighbors == 3) {
                board[r][c] = 2;
            }
        }
    }

    // Pass 2: Finalize states
    for (int r = 0; r < rows; r++) {
        for (int c = 0; c < cols; c++) {
            if (board[r][c] > 0) board[r][c] = 1;
            else board[r][c] = 0;
        }
    }
}

private int countLiveNeighbors(int[][] board, int r, int c) {
    int count = 0;
    for (int i = r - 1; i <= r + 1; i++) {
        for (int j = c - 1; j <= c + 1; j++) {
            if (i == r && j == c) continue;
            if (i >= 0 && i < board.length && j >= 0 && j < board[0].length 
                && Math.abs(board[i][j]) == 1) {
                count++;
            }
        }
    }
    return count;
}

Complexity Analysis

Time Complexity: O(M * N) - Two full passes.

Space Complexity: O(1) - No auxiliary arrays. We use the bits/values of the input array itself.

The Magic of Math.abs()

By marking a dying cell as -1, its absolute value is still 1. This means future neighbor counts in the same pass still see it as "currently alive," preserving the simultaneous update logic!

💡 Interview Cheat Sheet

Simultaneous Update

"The key is that cell state changes must not influence other cell state changes in the same generation."

Neighborhood Definition

Don't forget the diagonal neighbors! There are 8 neighbors total. Use a nested loop -1 to 1 to check them cleanly.

Bit Manipulation Trick

You can use the 2nd bit to store the future state: board[i][j] |= (nextState << 1). Then shift right to get the final state.
