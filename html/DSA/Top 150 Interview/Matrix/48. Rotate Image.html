48. Rotate Image

Rotate an n x n 2D matrix 90 degrees clockwise in-place. You must modify the input matrix directly.

📐 Problem Essence

Rotating a matrix means the element at matrix[row][col] needs to move to matrix[col][n - 1 - row]. While simple with a second matrix, doing it "in-place" requires clever swapping strategies.

Approach 1: Brute Force (Auxiliary)

The simplest logic: create a new matrix and map each element to its new rotated position.

Java Implementation

public void rotate(int[][] matrix) {
    int n = matrix.length;
    int[][] rotated = new int[n][n];

    // Map each element to its new position
    for (int row = 0; row < n; row++) {
        for (int col = 0; col < n; col++) {
            rotated[col][n - 1 - row] = matrix[row][col];
        }
    }

    // Copy back to original matrix (to satisfy in-place requirement)
    for (int row = 0; row < n; row++) {
        for (int col = 0; col < n; col++) {
            matrix[row][col] = rotated[row][col];
        }
    }
}

Complexity Analysis

Time Complexity: O(N²) - We visit every element twice (mapping then copying).

Space Complexity: O(N²) - We create an entirely new matrix of the same size.

Pros & Cons

Pros: Extremely simple to understand and implement.

Cons: Fails the "in-place" requirement of the interview if we consider memory usage.

Approach 2: Mathematical - Transpose + Reverse

A brilliant two-step transformation: Flip the matrix diagonally (transpose), then flip each row horizontally.

Step 1: Transpose (Flip on Diagonal)

Java Implementation

public void rotate(int[][] matrix) {
    int n = matrix.length;

    // 1. Transpose: Swap matrix[i][j] with matrix[j][i]
    for (int row = 0; row < n; row++) {
        for (int col = row + 1; col < n; col++) {
            int temp = matrix[row][col];
            matrix[row][col] = matrix[col][row];
            matrix[col][row] = temp;
        }
    }

    // 2. Reverse: Flip each row horizontally
    for (int row = 0; row < n; row++) {
        for (int col = 0; col < n / 2; col++) {
            int temp = matrix[row][col];
            matrix[row][col] = matrix[row][n - 1 - col];
            matrix[row][n - 1 - col] = temp;
        }
    }
}

Complexity Analysis

Time Complexity: O(N²) - We visit cells twice (once for transpose, once for reverse).

Space Complexity: O(1) - Truly in-place. Only a single temp variable used for swaps.

The Intuition

A transpose converts rows to columns. However, after transpose, the columns are in the wrong order for a clockwise rotation. Reversing each row puts the columns in their final 90° clockwise positions.

Approach 3: Geometric - Four-Way Layer Swap

The "Single Pass" method: Rotate elements in groups of four, working from the outside layers inward.

4-Cell Cyclic Rotation

Java Implementation

public void rotate(int[][] matrix) {
    int n = matrix.length;
    // Process layer by layer
    for (int layer = 0; layer < n / 2; layer++) {
        int first = layer;
        int last = n - 1 - layer;
        for (int i = first; i < last; i++) {
            int offset = i - first;
            int top = matrix[first][i]; // save top

            // left -> top
            matrix[first][i] = matrix[last - offset][first];

            // bottom -> left
            matrix[last - offset][first] = matrix[last][last - offset];

            // right -> bottom
            matrix[last][last - offset] = matrix[i][last];

            // top -> right
            matrix[i][last] = top; // right <- saved top
        }
    }
}

Complexity Analysis

Time Complexity: O(N²) - We visit every element once. This is the fastest single-pass solution.

Space Complexity: O(1) - Only one top variable used for temporary storage.

Comparison

While Approach 2 is easier to explain and code, Approach 3 is technically more efficient as it performs the rotation in a single pass of the cells. It is harder to implement correctly due to complex indexing.

💡 Interview Cheat Sheet

Coordinate Swap Rule

90° Clockwise: (i, j) → (j, n-1-i)
90° Counter-Clockwise: (i, j) → (n-1-j, i)

Transpose Trick

Transpose is Swap(matrix[i][j], matrix[j][i]). Always start inner loop at j = i + 1 to only swap the top triangle!

In-Place Requirement

If the interviewer says "In-Place", avoid Approach 1. Mention Transpose + Reverse as it's the most mathematically elegant.
