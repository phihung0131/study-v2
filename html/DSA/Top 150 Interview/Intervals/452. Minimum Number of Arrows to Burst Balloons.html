452. Minimum Number of Arrows to Burst Balloons

Medium Greedy Sorting

Imagine balloons on a 1D wall. You shoot arrows vertically. One arrow bursts all balloons that overlap with its x-coordinate. What is the minimum number of arrows needed?

The Greedy Core

To minimize arrows, each arrow should burst as many balloons as possible. This happens when we shoot through the intersection of multiple intervals.

"Don't shoot a new arrow unless you absolutely have to."

1

Greedy: Sort by Start Point

Intuition

Sort by points[i][0]. As we iterate, we maintain a "common intersection" end point. If the next balloon starts after our current intersection ends, we must shoot a new arrow. If it overlaps, we shrink the intersection's end point to the minimum of current ends to ensure the arrow hits all balloons in that group.

Ready to process sorted balloons.
Arrows Used: 0
public int findMinArrowShots(int[][] points) {
    Arrays.sort(points, (a, b) -> Integer.compare(a[0], b[0]));
    int arrows = 1;
    int currentEnd = points[0][1];

    for (int i = 1; i < points.length; i++) {
        if (points[i][0] > currentEnd) {
            arrows++;
            currentEnd = points[i][1];
        } else {
            // Shrink intersection to hit both
            currentEnd = Math.min(currentEnd, points[i][1]);
        }
    }
    return arrows;
}
Time: O(N log N) for sort.
Space: O(log N) for sort stack.
2

Greedy: Sort by End Point (Optimal)

Intuition

This is the standard approach. Sort by points[i][1]. Always shoot the arrow at the very end of the current balloon. Because we are sorted by end point, this arrow has the "maximum reach" to catch subsequent balloons that start before this end point. We don't need to shrink anything here!

Ready.
Arrows Used: 0
public int findMinArrowShots(int[][] points) {
    // Sort by end point to maximize arrow reach
    Arrays.sort(points, (a, b) -> Integer.compare(a[1], b[1]));
    
    int arrows = 1;
    int arrowPos = points[0][1];

    for (int i = 1; i < points.length; i++) {
        // If next balloon starts after current arrow, shoot new arrow
        if (points[i][0] > arrowPos) {
            arrows++;
            arrowPos = points[i][1];
        }
    }
    return arrows;
}
3

Why not Brute Force?

A brute force approach would involve checking all possible permutations of arrows or trying to find the "Maximum Independent Set" (balloons that don't overlap at all). Without sorting, you'd spend O(N²) or worse trying to find groups of overlapping balloons.

Comparison Table

Approach Logic Time Complexity
Sort by Start Track shrinking intersection O(N log N) Moderate
Sort by End Always shoot at the boundary O(N log N) Optimal/Clean
Unsorted Greedy Misses overlaps, wrong result - Incorrect

💡 Pattern: Activity Selection

This problem is a variation of the Activity Selection Problem. Instead of picking the maximum non-overlapping tasks, we are picking the minimum arrows to cover all tasks. Sorting by End Time is almost always the "Magic Key" for these interval problems.

🎯 Edge Cases

  • • Integer Overflow: [[-2147483648, 2147483647]]. Use Integer.compare instead of a[1] - b[1].
  • • Single Balloon: Always requires exactly 1 arrow.
  • • Touch Point: [1,2] and [2,3]. They overlap at 2! One arrow bursts both.
