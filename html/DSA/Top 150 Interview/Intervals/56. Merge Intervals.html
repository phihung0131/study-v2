56. Merge Intervals

Medium Array Sorting

Given an array of intervals where intervals[i] = [start, end], merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the intervals in the input.

Problem Summary

Two intervals [a, b] and [c, d] overlap if c ≤ b and a ≤ d. When they overlap, they merge into a single interval: [min(a, c), max(b, d)].

Example:

Input: intervals = [[1,3],[2,6],[8,10],[15,18]] Output: [[1,6],[8,10],[15,18]]

Explanation: Since intervals [1,3] and [2,6] overlap, merge them into [1,6].

1

Approach: Brute Force (N² Merging)

Intuition

Without sorting, we must compare every interval with every other interval. If we find two that overlap, we merge them into one and restart the process (or keep track of which are merged) until no more overlaps exist. This is similar to finding connected components in a graph.

public int[][] merge(int[][] intervals) {
    boolean[] isMerged = new boolean[intervals.length];
    List<int[]> result = new ArrayList<>();

    for (int i = 0; i < intervals.length; i++) {
        if (isMerged[i]) continue;
        
        int[] current = intervals[i];
        boolean foundMerge;
        
        do {
            foundMerge = false;
            for (int j = i + 1; j < intervals.length; j++) {
                if (!isMerged[j] && overlaps(current, intervals[j])) {
                    current[0] = Math.min(current[0], intervals[j][0]);
                    current[1] = Math.max(current[1], intervals[j][1]);
                    isMerged[j] = true;
                    foundMerge = true;
                }
            }
        } while (foundMerge); // Repeat scan if something merged
        
        result.add(current);
    }
    return result.toArray(new int[0][]);
}

Complexity Analysis

  • Time: O(N²)

    In the worst case (unsorted input), we might need to perform multiple passes over the array to ensure all transitive overlaps are caught.

  • Space: O(N)

    To store the result and potentially a boolean[] to track merged states.

Pros & Cons

Cons: Inefficient for large datasets.

Pros: Conceptually simple if you view it as grouping overlapping ranges.

2

Approach: Sorting (The Optimal Way)

Intuition

By sorting intervals by their start times, we guarantee that if two intervals overlap, they must be adjacent in the sorted list (or one will overlap with a series of subsequent ones). This allows us to merge everything in a single linear pass.

public int[][] merge(int[][] intervals) {
    if (intervals.length <= 1) return intervals;

    // 1. Sort by start time: O(N log N)
    Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

    List<int[]> mergedList = new ArrayList<>();
    int[] lastInterval = intervals[0];
    mergedList.add(lastInterval);

    // 2. Linear scan: O(N)
    for (int[] current : intervals) {
        int lastEnd = lastInterval[1];
        int currentStart = current[0];
        int currentEnd = current[1];

        if (currentStart <= lastEnd) {
            // Overlap: Update the end of the last added interval
            lastInterval[1] = Math.max(lastEnd, currentEnd);
        } else {
            // No Overlap: Add current as the new "last" reference
            lastInterval = current;
            mergedList.add(lastInterval);
        }
    }

    return mergedList.toArray(new int[mergedList.size()][]);
}

Complexity Analysis

  • Time: O(N log N)

    The dominant operation is the initial sort. The subsequent scan is O(N).

  • Space: O(log N) or O(N)

    Space for sorting algorithm (recursion stack) plus space for the result array.

Pros & Cons

Pros: Optimal time complexity. Elegant code.

Cons: Requires O(N log N) time even if the array is "almost" sorted.

💡 Pattern Recognition

  • Sort First: Whenever you see "Intervals", the first thought should be: "Do I need to sort these by start or end time?"
  • Greedy Choice: By processing intervals in chronological order, we only ever need to look at the most recently merged interval.
  • Updating in Place: In Java, objects in a list share references. Updating lastInterval[1] automatically updates it inside the mergedList.

🚩 Tricky Edge Cases

  • Nested Intervals: [[1,10], [2,5]]. The result is [[1,10]]. Don't forget Math.max!
  • Exact Touch: [[1,2], [2,3]]. These do overlap.
  • Large Values: Watch for Integer overflow if you calculate lengths (though not usually an issue in this specific problem).
  • Single Interval: Ensure your code handles intervals.length == 1 without extra logic.
