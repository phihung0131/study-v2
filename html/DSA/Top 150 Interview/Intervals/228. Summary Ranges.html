228. Summary Ranges

Easy Array Two Pointers

Given a sorted unique integer array nums, return the smallest sorted list of ranges that cover all the numbers in the array exactly.

Problem Summary

We need to group consecutive numbers together. For example, if we have 0, 1, 2, they form the range "0->2". If a number stands alone, like 4, it is just "4".

Example:

Input: nums = [0,1,2,4,5,7] Output: ["0->2", "4->5", "7"]
1

Approach: The Two-Pointer Linear Scan

Intuition

Since the array is sorted and contains unique values, any consecutive numbers will have a difference of exactly 1. We can maintain a rangeStart pointer and iterate through the array. As soon as we find a gap (where nums[i+1] != nums[i] + 1), we close the current range and start a new one.

public List<String> summaryRanges(int[] nums) {
    List<String> resultList = new ArrayList<>();
    int n = nums.length;
    
    // Traverse the array using a "rangeStart" pointer
    for (int currentIdx = 0; currentIdx < n; currentIdx++) {
        int rangeStartValue = nums[currentIdx];
        
        // Keep moving currentIdx while the next element is consecutive
        while (currentIdx + 1 < n && nums[currentIdx + 1] == nums[currentIdx] + 1) {
            currentIdx++;
        }
        
        // Form the string based on whether it's a single value or a range
        if (rangeStartValue != nums[currentIdx]) {
            resultList.add(rangeStartValue + "->" + nums[currentIdx]);
        } else {
            resultList.add(String.valueOf(rangeStartValue));
        }
    }
    
    return resultList;
}

Step-by-Step Breakdown

  • 1. Initialize an empty resultList.
  • 2. Loop through the array using currentIdx.
  • 3. Store the value at currentIdx as the rangeStartValue.
  • 4. Increment currentIdx as long as nums[i+1] == nums[i] + 1.
  • 5. Compare rangeStartValue with the final nums[currentIdx] to decide the string format.

Complexity Analysis

Time Complexity

O(n) where n is the length of the input array. Each element is visited exactly once by the pointer.

Space Complexity

O(1) excluding the output list, as we only use a few integer variables for tracking pointers.

Efficiency Comparison

Approach Time Space Intuition
Brute Force / Sub-intervals O(n) O(1) Checking every number independently.
One Pass Two-Pointer O(n) O(1) Smart scanning by jumping to the end of each range.

💡 Key Takeaways

  • ✔ Sorted Input is Key: Always leverage the sort order for O(1) difference checks.
  • ✔ Pointer Management: A simple while loop inside a for loop can still be O(n) if the pointer only moves forward.
  • ✔ String Concatenation: In Java, + is okay here, but StringBuilder is faster for massive strings.

🚩 Edge Cases to Watch

  • 1 Empty Array: Ensure the method returns an empty list immediately.
  • 2 Single Element: Array like [1] should return ["1"].
  • 3 No Ranges: Array like [1, 3, 5, 7] should return ["1", "3", "5", "7"].
  • 4 All Consecutive: Array like [1, 2, 3] should return ["1->3"].
