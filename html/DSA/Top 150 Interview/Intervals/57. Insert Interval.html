57. Insert Interval

Medium Array Simulation

Insert a new interval into a sorted, non-overlapping array. Merge any overlaps that occur as a result of the insertion.

Problem Summary

We are given a list of intervals [[s1, e1], [s2, e2]...] sorted by start. We must insert [newStart, newEnd] such that the resulting list remains sorted and non-overlapping.

Target Example:

intervals = [[1,2],[3,5],[6,7],[8,10],[12,16]], newInterval = [4,8] Output: [[1,2],[3,10],[12,16]]
1

Approach: The Three-Phase Linear Scan

Intuition

Since the input is already sorted, we can process it in one pass by splitting the task into three logical phases:
1. Before: Add all intervals that end before the new interval starts.
2. During: Merge all intervals that overlap with the new interval.
3. After: Add all remaining intervals that start after the merged interval ends.

InsertInterval.java 
public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0, n = intervals.length;

    // Phase 1: Add intervals that end before newInterval starts
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.add(intervals[i++]);
    }

    // Phase 2: Merge overlapping intervals
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval);

    // Phase 3: Add remaining intervals
    while (i < n) {
        result.add(intervals[i++]);
    }

    return result.toArray(new int[result.size()][]);
}

Complexity

  • Time: O(n) - We traverse the array exactly once.
  • Space: O(n) - To store the result list.

Pros & Cons

Pros: Extremely simple to implement and understand.

Cons: Requires extra space for a new list if modifying in-place isn't allowed.

2

Approach: Binary Search for Boundaries

Intuition

If the input array is very large and the insertion only affects a small middle portion, we can use Binary Search to find exactly where the overlap begins and ends. While the merge still takes linear time for the affected elements, we avoid manually scanning "clean" portions of the array.

// Finding the 'start' index using binary search
int leftIdx = binarySearch(intervals, newInterval[0], true);
// Finding the 'end' index using binary search
int rightIdx = binarySearch(intervals, newInterval[1], false);

// Then merge everything between [leftIdx, rightIdx]
// Time: O(log N) search + O(K) merging where K is affected elements.

Why use Binary Search?

In systems with massive datasets where you might insert one interval into 1,000,000 existing ones, jumping to the overlap point in O(log N) is significantly faster than scanning the first 500,000 "safe" intervals.

💡 Strategy Summary

  • Rule of Thumb: If intervals[i].end < newInterval.start, it's safe.
  • Overlap Condition: intervals[i].start <= newInterval.end AND intervals[i].end >= newInterval.start.
  • In-Place: In Java, ArrayList is better than array for deletions/insertions.

⚠ Common Pitfalls

  • Missing the end: Forgetting to add the "After" intervals after merging.
  • Empty Input: Make sure to handle an empty input array by returning [newInterval].
  • End of Array: The merge might happen at the very last element; your while loops must check i < n.
