11. Container With Most Water

Find two lines that together with the x-axis form a container, such that the container contains the most water.

Medium Two Pointers Greedy

Problem Summary

Given an array of heights, imagine each element is a vertical line. You need to pick two lines i and j to form a tank. The amount of water is limited by the shorter line.

Area = min(height[i], height[j]) * (j - i)

Approach 1: Brute Force (All Pairs)

Check every single combination of two lines and calculate the area for each.

Input: [1, 8, 6, 2, 5, 4]
Max Area Found: 0

Java Implementation

public int maxArea(int[] height) {
    int maxAreaFound = 0;
    for (int left = 0; left < height.length; left++) {
        for (int right = left + 1; right < height.length; right++) {
            int currentHeight = Math.min(height[left], height[right]);
            int currentWidth = right - left;
            int currentArea = currentHeight * currentWidth;
            
            maxAreaFound = Math.max(maxAreaFound, currentArea);
        }
    }
    return maxAreaFound;
}

Complexity Analysis

Time: O(n²). We compare every pair of lines.

Space: O(1). Only one variable to store the max.

The Downside

This approach is too slow for large inputs (n = 10⁵). We need a way to skip useless pairs.

Approach 2: Two Pointers (Optimal)

Start wide and move inward. Since moving a pointer always reduces width, we only move the pointer with the shorter height to potentially gain a taller container.

Input: [1, 8, 6, 2, 5, 4, 8, 3, 7]
Optimal Max Area: 0

Java Implementation

public int maxArea(int[] height) {
    int leftWall = 0;
    int rightWall = height.length - 1;
    int maxAreaFound = 0;

    while (leftWall < rightWall) {
        int currentHeight = Math.min(height[leftWall], height[rightWall]);
        int currentWidth = rightWall - leftWall;
        int currentArea = currentHeight * currentWidth;

        maxAreaFound = Math.max(maxAreaFound, currentArea);

        // Move the pointer pointing to the shorter line
        if (height[leftWall] < height[rightWall]) {
            leftWall++;
        } else {
            rightWall--;
        }
    }
    return maxAreaFound;
}

Complexity Analysis

Time: O(n). We traverse the array only once.

Space: O(1). Constant extra space used.

Why it works?

By moving the shorter line, we give ourselves a chance to find a taller line that compensates for the lost width. If we moved the taller line, the height of the container could only stay the same or decrease, while the width always decreases.

Comparison Table

Approach Time Complexity Space Complexity Logic
Brute Force O(n²) O(1) Exhaustive search
Two Pointers O(n) O(1) Greedy inward squeeze

⚡ Quick Takeaways

  • • Area Formula: min(h1, h2) * (index2 - index1)
  • • Optimization Hint: Since width decreases as pointers move together, we must prioritize finding greater heights.
  • • The Rule: Always move the pointer that points to the shorter bar.
  • • Greedy Property: This logic ensures we don't miss the optimal solution by eliminating states that are strictly worse.

🧩 Edge Cases

  • • Minimal Input: Array of length 2 (only one possible container).
  • • Uniform Heights: All lines have same height (move either pointer).
  • • Ascending/Descending: All lines are in sorted order.
  • • Steep Differences: One very tall bar in middle, but tiny bars on ends.
