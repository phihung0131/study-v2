15. 3Sum

Given an integer array nums, return all unique triplets [nums[i], nums[j], nums[k]] such that their sum is 0.

Medium Two Pointers Sorting

The Logic

The challenge isn't just finding three numbers that add to zero, but finding unique triplets. Sorting the array first is the key trick—it allows us to easily skip duplicate values and use efficient pointer movement.

Approach 1: Brute Force (Nested Loops)

Check every possible combination of three different indices. Use a Set to store the triplets to ensure uniqueness.

Input: [-1, 0, 1, 2, -1, -4]
Unique Triplets Found: 0

Java Implementation

public List<List<Integer>> threeSum(int[] nums) {
    Set<List<Integer>> uniqueTriplets = new HashSet<>();
    for (int i = 0; i < nums.length; i++) {
        for (int j = i + 1; j < nums.length; j++) {
            for (int k = j + 1; k < nums.length; k++) {
                if (nums[i] + nums[j] + nums[k] == 0) {
                    List<Integer> triplet = Arrays.asList(nums[i], nums[j], nums[k]);
                    Collections.sort(triplet);
                    uniqueTriplets.add(triplet);
                }
            }
        }
    }
    return new ArrayList<>(uniqueTriplets);
}

Complexity Analysis

  • Time Complexity: O(n³) - Triple nested loops iterate over all combinations.
  • Space Complexity: O(n) - The Set stores unique triplets.

Cons

Extremely slow for arrays larger than 100 elements. The sorting inside the loops to handle uniqueness adds even more overhead.

Approach 2: Sorting + Two Pointers (Optimal)

Sort the array first. Fix one pointer i, then use two pointers (left and right) to find the remaining two numbers.

Sorted Input: [-4, -1, -1, 0, 1, 2]
Triplets Found: 0
Sum Target: 0
Current Sum: 0

Java Implementation

public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);
    List<List<Integer>> result = new ArrayList<>();

    for (int i = 0; i < nums.length - 2; i++) {
        // Skip duplicate values for the fixed pointer
        if (i > 0 && nums[i] == nums[i - 1]) continue;

        int leftPointer = i + 1;
        int rightPointer = nums.length - 1;

        while (leftPointer < rightPointer) {
            int sum = nums[i] + nums[leftPointer] + nums[rightPointer];

            if (sum == 0) {
                result.add(Arrays.asList(nums[i], nums[leftPointer], nums[rightPointer]));
                leftPointer++;
                rightPointer--;
                // Skip duplicates for left and right pointers
                while (leftPointer < rightPointer && nums[leftPointer] == nums[leftPointer - 1]) leftPointer++;
                while (leftPointer < rightPointer && nums[rightPointer] == nums[rightPointer + 1]) rightPointer--;
            } else if (sum < 0) {
                leftPointer++;
            } else {
                rightPointer--;
            }
        }
    }
    return result;
}

Complexity Analysis

  • Time Complexity: O(n²) - Outer loop runs n times, and the two-pointer inner scan runs n times.
  • Space Complexity: O(log n) to O(n) depending on the sorting implementation.

Pros

The standard optimal solution. Sorting allows us to skip duplicate values easily, ensuring our result set contains unique triplets without using a heavy Set.

Approach Comparison

Approach Time Space Duplicates Handle
Brute Force O(n³) O(n) HashSet of lists
Two Pointers O(n²) O(1)* Manual skip logic

*Excluding output list and sorting space.

⚡ Quick Cheat Sheet

  • • Sorted? If the problem asks for triplets/sums and uniqueness, sort first!
  • • Two Pointers: Use them whenever you need to find a sum in a sorted range.
  • • Skip Duplicates: if (i > 0 && nums[i] == nums[i-1]) continue;
  • • Target: 3Sum is essentially fixing one number and solving 2Sum for the rest.

🧩 Edge Cases

  • • Short Array: Length < 3 (Return empty list).
  • • All Zeros: `[0, 0, 0, 0]` should return `[[0, 0, 0]]`.
  • • No Triplets: No three numbers add to zero.
  • • Duplicates: Multiple occurrences of the same numbers (e.g. `[-1, -1, 0, 1, 1]`).
