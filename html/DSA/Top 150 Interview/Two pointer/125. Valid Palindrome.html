125. Valid Palindrome

A phrase is a palindrome if, after converting all uppercase letters into lowercase letters and removing all non-alphanumeric characters, it reads the same forward and backward.

Easy String Two Pointers

Approach 1: Filter and Reverse

The "Brute Force" way: Clean the string first, then compare it with its reversed version.

Java Implementation

public boolean isPalindrome(String inputString) {
    // Step 1: Filter and normalize
    StringBuilder filteredContent = new StringBuilder();
    for (char currentChar : inputString.toCharArray()) {
        if (Character.isLetterOrDigit(currentChar)) {
            filteredContent.append(Character.toLowerCase(currentChar));
        }
    }

    // Step 2: Compare with reversed version
    String original = filteredContent.toString();
    String reversed = filteredContent.reverse().toString();
    
    return original.equals(reversed);
}

Complexity Analysis

Time Complexity: O(n) where n is the length of the string. We traverse the string once to filter and once to reverse.

Space Complexity: O(n). We create a new StringBuilder and two strings to store filtered content.

Pros & Cons

Pros: Very easy to understand and write. Uses built-in Java utility methods.

Cons: High memory overhead due to multiple string copies. Inefficient for extremely large inputs.

Approach 2: Two-Pointer Technique (Optimal)

The "In-Place" way: Use two pointers moving inward to compare characters directly without extra storage.

Java Implementation

public boolean isPalindrome(String s) {
    int leftPointer = 0;
    int rightPointer = s.length() - 1;

    while (leftPointer < rightPointer) {
        // Skip non-alphanumeric from left
        while (leftPointer < rightPointer && !Character.isLetterOrDigit(s.charAt(leftPointer))) {
            leftPointer++;
        }
        // Skip non-alphanumeric from right
        while (leftPointer < rightPointer && !Character.isLetterOrDigit(s.charAt(rightPointer))) {
            rightPointer--;
        }

        // Compare lowercase versions
        if (Character.toLowerCase(s.charAt(leftPointer)) != Character.toLowerCase(s.charAt(rightPointer))) {
            return false;
        }

        leftPointer++;
        rightPointer--;
    }
    return true;
}

Complexity Analysis

Time Complexity: O(n). Even though we have nested loops, each character is visited at most once by either pointer.

Space Complexity: O(1). We only store two integer pointers. This is the optimal space solution.

Pros & Cons

Pros: Constant space. Extremely efficient. Stops immediately if a mismatch is found.

Cons: Slightly more complex while loop logic than the brute force approach.

Approach Comparison

Approach Time Complexity Space Complexity Verdict
Filter & Reverse O(n) O(n) Simpler logic, memory heavy
Two Pointers O(n) O(1) Gold Standard (Interview Ready)

💡 Key Takeaways

  • Character Validation: Use Character.isLetterOrDigit(c) to handle punctuation.
  • Normalization: Always convert to a uniform case (lowercase) before comparison.
  • Two Pointer Pattern: Standard way to check properties related to symmetry.

⚠️ Edge Cases to Mention

  • Empty String: Usually considered a palindrome (Technical result: "" is valid).
  • Strings with NO Alphanumeric: e.g., " " or ".,." become "", thus valid.
  • Single Character: Always a palindrome.
