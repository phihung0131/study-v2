167. Two Sum II - Input Array Is Sorted

Find two numbers in a 1-indexed, non-decreasingly sorted array that add up to a target. You must use O(1) extra space.

Medium Array Two Pointers

Approach 1: Brute Force (Nested Loops)

The most basic strategy: Check every possible pair of numbers. We ignore the "sorted" property entirely.

Input Array
target = 9

Java Implementation

public int[] twoSum(int[] numbers, int target) {
    for (int outerPointer = 0; outerPointer < numbers.length; outerPointer++) {
        for (int innerPointer = outerPointer + 1; innerPointer < numbers.length; innerPointer++) {
            int currentSum = numbers[outerPointer] + numbers[innerPointer];
            
            if (currentSum == target) {
                return new int[]{outerPointer + 1, innerPointer + 1};
            }
        }
    }
    return new int[]{-1, -1};
}

Complexity Analysis

  • Time Complexity: O(n²) - We use nested loops to visit every pair.
  • Space Complexity: O(1) - No extra space used besides pointers.

The Verdict

Extremely inefficient for large arrays. It fails to utilize the fact that the input is already sorted, which is a massive hint for optimization.

Approach 2: Binary Search

Better optimization: For each number x, search for target - x using Binary Search.

target = 22

Java Implementation

public int[] twoSum(int[] numbers, int target) {
    for (int i = 0; i < numbers.length; i++) {
        int complement = target - numbers[i];
        int low = i + 1, high = numbers.length - 1;
        
        while (low <= high) {
            int mid = low + (high - low) / 2;
            if (numbers[mid] == complement) return new int[]{i + 1, mid + 1};
            if (numbers[mid] < complement) low = mid + 1;
            else high = mid - 1;
        }
    }
    return new int[]{-1, -1};
}

Complexity Analysis

  • Time Complexity: O(n log n) - Loop through n elements and perform binary search on each.
  • Space Complexity: O(1) - No extra storage.

When to use?

Useful if you are already comfortable with Binary Search, but for this specific problem, we can do even better with a linear scan.

Approach 3: Two Pointers (Optimal)

The Gold Standard: Place one pointer at the start and one at the end. Move them inward based on the current sum.

target = 15

Java Implementation

public int[] twoSum(int[] numbers, int target) {
    int leftPointer = 0;
    int rightPointer = numbers.length - 1;

    while (leftPointer < rightPointer) {
        int currentSum = numbers[leftPointer] + numbers[rightPointer];

        if (currentSum == target) {
            return new int[]{leftPointer + 1, rightPointer + 1};
        } else if (currentSum < target) {
            leftPointer++; // Need a larger sum, move left right
        } else {
            rightPointer--; // Need a smaller sum, move right left
        }
    }
    return new int[]{-1, -1};
}

Complexity Analysis

  • Time Complexity: O(n) - We traverse the array exactly once.
  • Space Complexity: O(1) - Only two pointers are used.

Intuition

Because the array is sorted, increasing leftPointer is guaranteed to increase the sum, and decreasing rightPointer is guaranteed to decrease it. This allows us to narrow down the search space linearly.

Comparison Table

Approach Time Complexity Space Complexity Performance
Brute Force O(n²) O(1) Slowest
Binary Search O(n log n) O(1) Good
Two Pointers O(n) O(1) Optimal

⚡ Quick Cheat Sheet

  • • Sorted Input? Think Two Pointers or Binary Search immediately.
  • • Target Sum? Consider the complement (target - x).
  • • Two Sum II? Different from Two Sum I (which usually needs a HashMap for O(1) space).
  • • 1-Indexed? Always add +1 to your final indices.

🔍 Common Edge Cases

  • • Smallest Array: Array of size 2 (minimum allowed).
  • • Negative Numbers: Sorted order still applies (e.g., [-5, -2, 0, 3]).
  • • Duplicates: The input may contain duplicate numbers, handle normally.
  • • Target Mismatch: Problem usually guarantees exactly one solution.
