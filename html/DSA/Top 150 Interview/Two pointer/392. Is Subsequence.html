392. Is Subsequence

Given two strings s and t, return true if s is a subsequence of t, or false otherwise.

Easy Two Pointers Dynamic Programming

Approach 1: Greedy Two-Pointer Scan

The most efficient way for a single pair of strings. We maintain two pointers and try to find each character of s in t in order.

Java Solution

public boolean isSubsequence(String s, String t) {
    if (s.length() == 0) return true;

    int sPointer = 0;
    int tPointer = 0;

    while (tPointer < t.length()) {
        // If characters match, move the subsequence pointer
        if (s.charAt(sPointer) == t.charAt(tPointer)) {
            sPointer++;
        }
        
        // If all characters in s are found, we are done
        if (sPointer == s.length()) {
            return true;
        }

        // Always move the target pointer
        tPointer++;
    }

    return false;
}

Complexity Analysis

Time Complexity: O(T) where T is the length of the target string. We scan t at most once.

Space Complexity: O(1). We only use two integer pointers regardless of input size.

Pros & Cons

Pros: Optimal time and space for a single query. Very simple to implement.

Cons: If we need to check 1,000,000 different s strings against the same t, this becomes O(N * T).

Approach 2: Pre-computation + Binary Search

"What if there are lots of incoming S strings?"

We pre-index the positions of every character in t. For each character in s, we binary search for the first available position after our current index.

Java Solution (Pre-computed)

public boolean isSubsequence(String s, String t) {
    // 1. Pre-compute: Map characters to their list of indices
    List<Integer>[] charIndices = new ArrayList[26];
    for (int i = 0; i < t.length(); i++) {
        int charIdx = t.charAt(i) - 'a';
        if (charIndices[charIdx] == null) charIndices[charIdx] = new ArrayList<>();
        charIndices[charIdx].add(i);
    }

    int currentPosition = -1;
    for (char c : s.toCharArray()) {
        List<Integer> list = charIndices[c - 'a'];
        if (list == null) return false;

        // 2. Binary search for first index > currentPosition
        int nextIndex = Collections.binarySearch(list, currentPosition + 1);
        if (nextIndex < 0) nextIndex = -nextIndex - 1;

        if (nextIndex == list.size()) return false;
        currentPosition = list.get(nextIndex);
    }
    return true;
}

Complexity Analysis

Pre-computation: O(T) time and space.

Query Time: O(S * log T). For each char in s, we binary search in t's indices.

Why this is better?

If T is very large (10^6) and S is small (10), log T is only ~20. Checking a new s takes 10 * 20 = 200 operations vs 1,000,000 operations for the two-pointer scan!

Choosing the Right Strategy

Scenario Method Time (Single) Space
Single Check Two Pointers O(T) O(1)
Multiple S, Huge T Binary Search O(S log T) O(T)
DP Approach Matrix Matching O(S*T) O(S*T)

🚀 Interview Pro-Tips

  • • Always clarify if there's only one "s" or multiple.
  • • Mention the greedy nature: picking the first available 'a' is always optimal.
  • • Edge cases: If s.length() > t.length(), return false immediately.

🧩 Vocabulary

  • • Subsequence: Order matters, but adjacency does not.
  • • Substring: Both order and adjacency matter.
  • • Greedy: Making the best local choice at each step.
