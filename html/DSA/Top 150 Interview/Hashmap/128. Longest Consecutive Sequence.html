128. Longest Consecutive Sequence

Finding the longest streak of consecutive integers in an unsorted array.

Medium Hash Set Array

Problem Overview

Given an unsorted array of integers nums, return the length of the longest consecutive elements sequence.

Constraints: You must write an algorithm that runs in O(n) time.

Example: Input: [100, 4, 200, 1, 3, 2] → Output: 4 (Sequence: [1, 2, 3, 4])

Approach 1: Brute Force Search

For each number x in the array, try to find x+1, x+2, etc., by scanning the entire array for each increment.

Java Implementation

public int longestConsecutive(int[] nums) { int maxStreak = 0; for (int num : nums) { int currentNum = num; int currentStreak = 1; // Repeatedly check if the next number exists in the array while (arrayContains(nums, currentNum + 1)) { currentNum += 1; currentStreak += 1; } maxStreak = Math.max(maxStreak, currentStreak); } return maxStreak; } private boolean arrayContains(int[] arr, int target) { for (int x : arr) if (x == target) return true; return false; }

Complexity Analysis

Time Complexity: O(n³). Outer loop runs n times. The while loop can run n times. The arrayContains search is O(n).

Space Complexity: O(1). No extra data structures used.

Pros & Cons

Pros: No extra space, simple logic.

Cons: Extremely inefficient. Multiple redundant scans of the same values.

Approach 2: Sorting and Linear Scan

Sort the array first. Consecutive numbers will now be adjacent, allowing us to find streaks in a single pass.

Java Implementation

public int longestConsecutive(int[] nums) { if (nums.length == 0) return 0; Arrays.sort(nums); int maxStreak = 1; int currentStreak = 1; for (int i = 1; i < nums.length; i++) { if (nums[i] != nums[i - 1]) { if (nums[i] == nums[i - 1] + 1) { currentStreak += 1; } else { maxStreak = Math.max(maxStreak, currentStreak); currentStreak = 1; } } } return Math.max(maxStreak, currentStreak); }

Complexity Analysis

Time Complexity: O(n log n) due to sorting. The subsequent scan is O(n).

Space Complexity: O(1) or O(n) depending on the sorting implementation's recursion stack.

Key Detail

Must handle duplicates. If nums[i] == nums[i-1], we ignore it and continue without breaking the streak.

Approach 3: Hash Set (Optimal O(n))

Store all numbers in a Set for O(1) lookups. Only start building a sequence from a number that is the actual start of a sequence (i.e., num - 1 is not in the Set).

Java Implementation

public int longestConsecutive(int[] nums) { Set<Integer> numSet = new HashSet<>(); for (int num : nums) numSet.add(num); int maxStreak = 0; for (int num : numSet) { // Only check for sequence if 'num' is the start of a sequence if (!numSet.contains(num - 1)) { int currentNum = num; int currentStreak = 1; while (numSet.contains(currentNum + 1)) { currentNum += 1; currentStreak += 1; } maxStreak = Math.max(maxStreak, currentStreak); } } return maxStreak; }

Complexity Analysis

Time Complexity: O(n). Although there is a nested loop, each number is visited only a constant number of times (twice: once in the outer loop, and once inside the while loop across all iterations).

Space Complexity: O(n) to store all numbers in the Hash Set.

Why is it O(n)?

The while loop only triggers for the first element of a streak. Every element in the array is "processed" by the while loop at most once in the entire execution.

Interview Cheat Sheet

Key Pattern

  • • "Consecutive" problems usually benefit from Sorting or Sets.
  • • To reach O(n) in sequence problems, look for a way to identify the start of the sequence.
  • • Set.contains(x-1) is a common "is this the start?" check.

Edge Cases

  • • Empty Array: Handle separately (return 0).
  • • Single Element: Should return 1.
  • • Duplicate Elements: Sorting must ignore them; Set handles them automatically.
  • • Negative Values: Handled naturally by comparison/Set.
