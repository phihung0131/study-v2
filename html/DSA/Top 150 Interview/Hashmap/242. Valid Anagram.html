242. Valid Anagram

Checking if two strings contain the exact same characters in any order.

Easy Hash Table Sorting

Problem Overview

Given two strings s and t, return true if t is an anagram of s, and false otherwise.

Anagram Definition: A word or phrase formed by rearranging the letters of a different word or phrase, typically using all the original letters exactly once.

Approach 1: Search and Mark (Brute Force)

Iterate through every character in s. For each, search through t for a match. If found, mark that character in t as "used".

Java Implementation

public boolean isAnagram(String s, String t) { if (s.length() != t.length()) return false; // Use a boolean array to track used characters in string t boolean[] usedInT = new boolean[t.length()]; for (int i = 0; i < s.length(); i++) { char charS = s.charAt(i); boolean foundMatch = false; for (int j = 0; j < t.length(); j++) { if (!usedInT[j] && t.charAt(j) == charS) { usedInT[j] = true; // Consume the character foundMatch = true; break; } } if (!foundMatch) return false; } return true; }

Complexity Analysis

Time Complexity: O(N²) where N is string length. Nested loops result in quadratic time.

Space Complexity: O(N) for the boolean array tracking used characters.

Verdict

Very slow for long strings. The constant scanning of the second string is redundant if we use a more structured data format.

Approach 2: Sort and Compare

If two strings are anagrams, they will look identical after being sorted alphabetically. Sorting brings matching characters to the same index.

Java Implementation

public boolean isAnagram(String s, String t) { if (s.length() != t.length()) return false; // Convert to char arrays char[] sArray = s.toCharArray(); char[] tArray = t.toCharArray(); // Sort both arrays alphabetically Arrays.sort(sArray); Arrays.sort(tArray); // If anagrams, arrays must be identical return Arrays.equals(sArray, tArray); }

Complexity Analysis

Time Complexity: O(N log N) dominated by the sorting algorithm (Dual-Pivot Quicksort in Java).

Space Complexity: O(N) to store the char array copies of the strings.

Verdict

Very clean and readable. This is a solid solution for coding interviews unless O(N) is strictly required.

Approach 3: Frequency Count (Optimal)

Use an integer array of size 26 to count occurrences. Add for s and subtract for t. If all slots end at zero, it's a match.

T: "act"

Step 1: Count characters in S

Java Implementation

public boolean isAnagram(String s, String t) { if (s.length() != t.length()) return false; // 26 slots for 'a' through 'z' int[] counts = new int[26]; for (int i = 0; i < s.length(); i++) { counts[s.charAt(i) - 'a']++; // Increment for S counts[t.charAt(i) - 'a']--; // Decrement for T } // If anagram, every slot should be 0 for (int count : counts) { if (count != 0) return false; } return true; }

Complexity Analysis

Time Complexity: O(N). We iterate over strings once and the 26-slot array once.

Space Complexity: O(1). The array size is constant (26), regardless of input string length.

Pro Tip

This is the most efficient way to solve counting problems involving fixed character sets (like lowercase English letters).

Interview Cheat Sheet

Key Patterns

  • • "Anagram" = "Exact Character Frequency Match".
  • • Always start by checking if s.length() == t.length().
  • • Use int[26] for 26-lowercase problems.
  • • Use HashMap<Character, Integer> for Unicode/Generic problems.

Corner Cases

  • • Different Lengths: Immediate False.
  • • Case Sensitivity: Usually LeetCode says "lowercase only", but clarify in interviews.
  • • Empty Strings: Both empty = True.
  • • Special Characters: Treat as regular chars or ignore? (Clarify with interviewer).
