202. Happy Number

Detecting cycles in the sequence of digit-square sums.

Easy Hash Table Two Pointers

Problem Definition

Write an algorithm to determine if a number n is happy.

A happy number is defined by the following process:

  1. Starting with any positive integer, replace the number by the sum of the squares of its digits.
  2. Repeat the process until the number equals 1 (at which point it will stay 1).
  3. Otherwise, it loops endlessly in a cycle which does not include 1.

Approach 1: HashSet Tracking

The most straightforward method. We keep track of every sum we calculate. If we ever see the same sum twice, we've hit a cycle and the number isn't happy.

Java Implementation

public boolean isHappy(int number) { Set<Integer> seenNumbers = new HashSet<>(); while (number != 1) { if (seenNumbers.contains(number)) { return false; // Cycle detected! } seenNumbers.add(number); number = getNextSumOfSquares(number); } return true; } private int getNextSumOfSquares(int n) { int totalSum = 0; while (n > 0) { int digit = n % 10; totalSum += digit * digit; n /= 10; } return totalSum; }

Complexity Analysis

Time Complexity: O(log N). Calculating digit squares for a number N takes log N steps. The total number of steps in the sequence is small (even for 2^31-1, the next step is only 730).

Space Complexity: O(log N) to store seen values in the HashSet.

Pros & Cons

Pros: Intuitive and easy to debug. Directly implements the cycle detection rule.

Cons: Uses extra memory. Not the most space-efficient for competitive programming.

Approach 2: Floyd's Cycle-Finding Algorithm

The "Tortoise and Hare" trick. One pointer moves through the sequence at 1x speed, the other at 2x speed. If they collide at a number other than 1, there is a cycle.

Java Implementation

public boolean isHappy(int n) { int slowRunner = n; int fastRunner = getNext(n); while (fastRunner != 1 && slowRunner != fastRunner) { slowRunner = getNext(slowRunner); // 1 step fastRunner = getNext(getNext(fastRunner)); // 2 steps } return fastRunner == 1; } private int getNext(int n) { int totalSum = 0; while (n > 0) { int d = n % 10; totalSum += d * d; n /= 10; } return totalSum; }

Complexity Analysis

Time Complexity: O(log N). Similar to the HashSet method, but potentially more steps until the fast pointer catches the slow one.

Space Complexity: O(1). We only use two integer pointers. Memory Optimal.

Pros & Cons

Pros: Constant space. Extremely common interview pattern for cycle detection.

Cons: Can be slightly harder to visualize logic-wise compared to a HashSet.

Approach 3: Mathematical Cycle Observation

Mathematical proofs show that any number that isn't happy always eventually enters the specific cycle: 4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4. We can just check for these numbers!

Java Implementation

public boolean isHappy(int n) { // Any number that isn't happy will eventually hit 4 and stay in that cycle while (n != 1 && n != 4) { n = getNext(n); } return n == 1; } private int getNext(int n) { int sum = 0; while (n > 0) { int d = n % 10; sum += d * d; n /= 10; } return sum; }

Complexity Analysis

Time Complexity: O(log N). This is the fastest practical simulation because it stops as soon as it hits any part of the cycle.

Space Complexity: O(1). No auxiliary structures needed.

Critical Context

Note: In an interview, you should explain why you chose 4 (as the representative of the cycle). It shows deep mathematical preparation but is less "generic" than Floyd's algorithm.

Interview Mastery Cheat Sheet

Key Takeaways

  • • This problem is fundamentally about Cycle Detection in a sequence.
  • • The "Happy Number" sequence is guaranteed to either hit 1 or enter a loop.
  • • For any number N, the next sum drops quickly (9^2 * 10 digits = 810). This means the space searched is always finite and small.

Corner Cases

  • • n = 1: Already happy.
  • • n = Large Integer: log N handles the scale automatically.
  • • n = 7: A happy number that takes several steps (49, 97, 130, 10, 1).
  • • n = 4: The base cycle entry point.
