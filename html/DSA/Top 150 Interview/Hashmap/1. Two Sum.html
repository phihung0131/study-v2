1. Two Sum

Find two numbers that add up to a specific target.

Easy Array Hash Table

Problem Overview

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

Constraints:
  • Exactly one solution exists.
  • You may not use the same element twice.
  • Indices can be returned in any order.

Approach 1: Brute Force

Check every possible pair of numbers in the array. If their sum equals the target, return their indices.

Java Implementation

public int[] twoSum(int[] nums, int target) { for (int outerIndex = 0; outerIndex < nums.length; outerIndex++) { for (int innerIndex = outerIndex + 1; innerIndex < nums.length; innerIndex++) { if (nums[outerIndex] + nums[innerIndex] == target) { return new int[] { outerIndex, innerIndex }; } } } return new int[] {}; // Should not happen per constraints }

Complexity Analysis

Time Complexity: O(n²). For each element, we loop through the rest of the array.

Space Complexity: O(1). We only use constant extra space.

Pros & Cons

Pros: No extra memory needed. Extremely simple to implement.

Cons: Extremely slow for large datasets. Inefficient use of information.

Approach 2: Two-Pass Hash Map

First, build a dictionary mapping values to indices. Then, iterate again to see if each element's complement (target - num) exists in the map.

Java Implementation

public int[] twoSum(int[] nums, int target) { Map<Integer, Integer> numToIndexMap = new HashMap<>(); // Pass 1: Build the map for (int i = 0; i < nums.length; i++) { numToIndexMap.put(nums[i], i); } // Pass 2: Find the complement for (int i = 0; i < nums.length; i++) { int complement = target - nums[i]; if (numToIndexMap.containsKey(complement) && numToIndexMap.get(complement) != i) { return new int[] { i, numToIndexMap.get(complement) }; } } return new int[] {}; }

Complexity Analysis

Time Complexity: O(n). We traverse the list twice, but Hash Map lookups are O(1).

Space Complexity: O(n). We store all elements in the Map.

Pros & Cons

Pros: Much faster than brute force. Decouples counting from searching.

Cons: Requires O(n) extra space. Two passes through the data.

Approach 3: One-Pass Hash Map (Optimal)

Search and insert simultaneously. For each number, check if its complement is already in the map. If not, add the current number and keep moving.

Java Implementation

public int[] twoSum(int[] nums, int target) { Map<Integer, Integer> prevMap = new HashMap<>(); for (int i = 0; i < nums.length; i++) { int complement = target - nums[i]; if (prevMap.containsKey(complement)) { return new int[] { prevMap.get(complement), i }; } prevMap.put(nums[i], i); } return new int[] {}; }

Complexity Analysis

Time Complexity: O(n). One single pass through the array.

Space Complexity: O(n). Still requires a map to store seen values.

Key takeaway

This is the industry-standard solution. It is efficient in both time and logic complexity.

Interview Cheat Sheet

Core Pattern

  • • Finding a "match" for current element? Use a Hash Map.
  • • Complement logic: target - nums[i] = complement.
  • • "One-pass" usually means checking the map before inserting.

Follow-up Questions

  • • What if the array is sorted? Use Two Pointers (O(n) time, O(1) space).
  • • Can we do better than O(n) space? Not without sorting.
  • • What if there are multiple pairs? Use a list to store all results.
