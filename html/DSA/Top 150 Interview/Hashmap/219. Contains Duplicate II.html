219. Contains Duplicate II

Finding nearby duplicates within a fixed distance k.

Easy Array Sliding Window

Problem Overview

Given an integer array nums and an integer k, return true if there are two distinct indices i and j such that:

1. nums[i] == nums[j]

2. abs(i - j) <= k

Approach 1: Nested Loops (Brute Force)

Check every pair (i, j). To optimize slightly, we only check j in the range [i+1, i+k].

Java Implementation

public boolean containsNearbyDuplicate(int[] nums, int k) { for (int i = 0; i < nums.length; i++) { // Only check indices within distance k for (int j = i + 1; j <= i + k && j < nums.length; j++) { if (nums[i] == nums[j]) { return true; } } } return false; }

Complexity Analysis

Time Complexity: O(n * k). In the worst case, we check k neighbors for every element.

Space Complexity: O(1). No extra space used.

Verdict

Too slow if k is large (near n), resulting in O(n²) behavior. We need a way to "remember" previous indices.

Approach 2: Hash Map (Value → Last Index)

Traverse the array once. Store the last seen index of each value. If we see a value again, check the distance from its previous index.

Java Implementation

public boolean containsNearbyDuplicate(int[] nums, int k) { Map<Integer, Integer> lastSeenIndex = new HashMap<>(); for (int currentIndex = 0; currentIndex < nums.length; currentIndex++) { int num = nums[currentIndex]; if (lastSeenIndex.containsKey(num)) { int prevIndex = lastSeenIndex.get(num); if (currentIndex - prevIndex <= k) { return true; } } // Always update to the most recent index for future checks lastSeenIndex.put(num, currentIndex); } return false; }

Complexity Analysis

Time Complexity: O(n). Single pass, Map operations are O(1).

Space Complexity: O(n) to store up to n elements in the Map.

Key Detail

We always update the index in the map. Why? Because a later occurrence is more likely to be within distance k of the *next* occurrence.

Approach 3: Sliding Window (Hash Set)

Maintain a "window" of size k using a Set. If a number we're adding is already in the Set, it must be within distance k.

Java Implementation

public boolean containsNearbyDuplicate(int[] nums, int k) { Set<Integer> window = new HashSet<>(); for (int i = 0; i < nums.length; i++) { // If window grows larger than k, remove the oldest element if (i > k) { window.remove(nums[i - k - 1]); } // add() returns false if the element was already present if (!window.add(nums[i])) { return true; } } return false; }

Complexity Analysis

Time Complexity: O(n). Single pass with constant-time Set operations.

Space Complexity: O(min(n, k)). The Set never holds more than k + 1 elements.

Why Sliding Window?

If k is much smaller than n, this approach uses significantly less memory than a full HashMap because it automatically discards old entries.

Comparison Matrix

Approach Time Space Best Use Case
Brute Force O(n * k) O(1) Very small arrays/k
Hash Map O(n) O(n) General purpose
Sliding Window O(n) O(k) Large n, small k

Interview Cheat Sheet

Key Concept

  • • "Duplicate in Window" = Sliding Window + Set.
  • • Hash Map is safer for "Any Distance", but Sliding Window is memory-efficient for "Fixed Distance K".
  • • Pay attention to i > k logic to remove elements from the window.

Edge Cases

  • • k = 0: Always false (distinct indices i, j required).
  • • k >= n: Becomes a standard "contains duplicate" problem.
  • • Empty/Single Array: Always false.
  • • Negative Values: Handled naturally by Maps/Sets.
