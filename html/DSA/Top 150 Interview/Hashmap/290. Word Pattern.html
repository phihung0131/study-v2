205. Word Pattern

Determining if a string s follows a specific character pattern.

Easy Hash Table Bijective Mapping

Problem Overview

Given a pattern and a string s, find if s follows the same pattern.

Rules for "Following":

  • Each character in pattern maps to exactly one non-empty word in s.
  • Each non-empty word in s maps to exactly one character in pattern.
  • The relationship is bijective (one-to-one).

Approach 1: Naive Verification (O(N²))

For every new pair, manually scan all previous pairs to ensure no rules are broken. No extra hash-based data structures used.

Java Implementation

public boolean wordPattern(String pattern, String s) { String[] words = s.split(" "); if (words.length != pattern.length()) return false; for (int current = 0; current < words.length; current++) { char currentChar = pattern.charAt(current); String currentWord = words[current]; // Naive Check: Scan all previous entries for (int prev = 0; prev < current; prev++) { // If characters match, words MUST match if (pattern.charAt(prev) == currentChar && !words[prev].equals(currentWord)) return false; // If words match, characters MUST match if (words[prev].equals(currentWord) && pattern.charAt(prev) != currentChar) return false; } } return true; }

Complexity Analysis

Time Complexity: O(N²). For each pair, we scan all previous pairs.

Space Complexity: O(N) to store the split words array.

Why skip this?

It's too slow for large patterns. In an interview, this is only a starting point to demonstrate basic logic before optimizing with HashMaps.

Approach 2: Double HashMap (O(N))

Use two maps to store the Pattern → Word and Word → Pattern relationships simultaneously for O(1) lookup.

Java Implementation

public boolean wordPattern(String pattern, String s) { String[] words = s.split(" "); if (words.length != pattern.length()) return false; Map<Character, String> charMap = new HashMap<>(); Map<String, Character> wordMap = new HashMap<>(); for (int i = 0; i < words.length; i++) { char c = pattern.charAt(i); String w = words[i]; if (charMap.containsKey(c)) { if (!charMap.get(c).equals(w)) return false; } else { if (wordMap.containsKey(w)) return false; // Bijective rule check charMap.put(c, w); wordMap.put(w, c); } } return true; }

Complexity Analysis

Time Complexity: O(N). We iterate once. HashMap operations are O(1) on average.

Space Complexity: O(M) where M is the number of unique words/chars.

Pros & Cons

Pros: Very fast and highly readable. Standard way to solve mapping problems.

Cons: Slightly higher constant memory usage due to two Maps.

Approach 3: Single Map Indexing (Optimal)

Using a single HashMap<Object, Integer>. Map both the character and the word to the same index where they were last seen. If the returned values differ, the pattern is broken.

Java Implementation

public boolean wordPattern(String pattern, String s) { String[] words = s.split(" "); if (words.length != pattern.length()) return false; // Single map using Object to accept both Character and String Map<Object, Integer> indexMap = new HashMap<>(); for (Integer i = 0; i < words.length; i++) { // put() returns the PREVIOUS value associated with key Object prevCharIdx = indexMap.put(pattern.charAt(i), i); Object prevWordIdx = indexMap.put(words[i], i); // In Java, Map.put returns null if it's the first time seeing the key. // We check if the previous indices match. If they don't, pattern is broken. if (!Objects.equals(prevCharIdx, prevWordIdx)) { return false; } } return true; }

Complexity Analysis

Time Complexity: O(N). Single pass through words.

Space Complexity: O(M). Only one map is used, storing both strings and characters.

Pro Tip

The put() return trick is very clever but requires Integer instead of int to handle null returns from HashMap. Use Objects.equals() for safe comparison.

Interview Cheat Sheet

Core Logic

  • • Step 1: Always split s into a words array first.
  • • Step 2: Check pattern.length() == words.length. Fail if not.
  • • Step 3: Mapping must be Bijective. One char cannot map to two words; one word cannot map to two chars.

Corner Cases

  • • "abba", "dog cat cat fish" (Pattern match fails)
  • • "abba", "dog dog dog dog" (Word match fails - many to one)
  • • "aaaa", "dog cat cat dog" (Word match fails - one to many)
  • • Spaces: Ensure split handles extra spaces if necessary.
