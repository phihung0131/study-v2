205. Isomorphic Strings

Checking for character mapping consistency between two strings.

Easy Hash Table String Pattern

Problem Overview

Two strings s and t are isomorphic if the characters in s can be replaced to get t.

Rules:

  • All occurrences of a character must be replaced with another character.
  • The order of characters must be preserved.
  • No two characters in s may map to the same character in t.
  • A character may map to itself.

Approach 1: Double Character Mapping

Use two maps to ensure a one-to-one relationship. Map 1 handles s → t, and Map 2 handles t → s.

Java Implementation

public boolean isIsomorphic(String s, String t) { Map<Character, Character> sToTMap = new HashMap<>(); Map<Character, Character> tToSMap = new HashMap<>(); for (int i = 0; i < s.length(); i++) { char charS = s.charAt(i); char charT = t.charAt(i); // Rule 1: Check if s character is already mapped to a different t character if (sToTMap.containsKey(charS) && sToTMap.get(charS) != charT) return false; // Rule 2: Check if t character is already mapped to a different s character if (tToSMap.containsKey(charT) && tToSMap.get(charT) != charS) return false; sToTMap.put(charS, charT); tToSMap.put(charT, charS); } return true; }

Complexity Analysis

Time Complexity: O(N) where N is the length of the string. We perform a single pass.

Space Complexity: O(K) where K is the number of unique characters (max 256 for ASCII).

Pros & Cons

Pros: Intuitive logic; directly implements the "one-to-one" mapping rule.

Cons: Higher memory overhead due to HashMap object creation and boxing.

Approach 2: Last Seen Index (Array Optimized)

Instead of characters, map the last seen position of each character. If characters at the same position in both strings weren't seen at the same time last, the mapping is broken.

Java Implementation

public boolean isIsomorphic(String s, String t) { // Use 256 for Extended ASCII range int[] lastSeenS = new int[256]; int[] lastSeenT = new int[256]; for (int i = 0; i < s.length(); i++) { // If current character indices don't match, mapping is inconsistent if (lastSeenS[s.charAt(i)] != lastSeenT[t.charAt(i)]) { return false; } // Update with current index + 1 (avoiding default 0 value collision) lastSeenS[s.charAt(i)] = i + 1; lastSeenT[t.charAt(i)] = i + 1; } return true; }

Complexity Analysis

Time Complexity: O(N). Single pass, O(1) array access.

Space Complexity: O(1). Arrays are fixed size (256) regardless of string length.

Pros & Cons

Pros: Fast, cache-friendly, and very memory-efficient (O(1) space).

Cons: Requires manual handling of the index (using i+1) to distinguish "seen at index 0" from "not seen".

Approach 3: Pattern Normalization

Convert both strings into a "canonical" numerical form. For example, "paper" becomes "0-1-0-2-3". If the generated patterns are identical, the strings are isomorphic.

Java Implementation

public boolean isIsomorphic(String s, String t) { return getPattern(s).equals(getPattern(t)); } private String getPattern(String str) { Map<Character, Integer> charToFirstIndex = new HashMap<>(); StringBuilder patternBuilder = new StringBuilder(); for (int i = 0; i < str.length(); i++) { char currentChar = str.charAt(i); // Record first occurrence if not already present charToFirstIndex.putIfAbsent(currentChar, i); // Append the index of the first occurrence as the "ID" patternBuilder.append(charToFirstIndex.get(currentChar)).append(","); } return patternBuilder.toString(); }

Complexity Analysis

Time Complexity: O(N). We traverse both strings and construct the builder.

Space Complexity: O(N). To store the generated pattern strings.

Pros & Cons

Pros: Decouples the two strings. You can pre-calculate or store patterns in a database for lookups.

Cons: String concatenation and building patterns is slower and uses more memory than raw index checks.

Interview Cheat Sheet

Checklist

  • • Are both strings the same length? (Constraint: 1 ≤ s.length)
  • • Is the mapping 1-to-1? (No collisions)
  • • Is the character range ASCII or Unicode?
  • • If memory is tight, choose int[256].

Edge Cases

  • • "aa", "ab": Fails mapping t to s.
  • • "ab", "aa": Fails mapping s to t.
  • • "a", "a": Trivial pass.
  • • "badc", "baba": Mapping collision 'd' and 'b'.
