383. Ransom Note

Can we build a ransom note using characters from a magazine?

Easy Hash Table String

Problem Summary

Given two strings ransomNote and magazine, return true if ransomNote can be constructed by using the letters from magazine and false otherwise.

Constraint: Each letter in magazine can only be used once in your ransom note.

Approach 1: Search and Delete (Brute Force)

The most intuitive way: for every character in the note, search the magazine. If found, "cross it out" so it can't be used again.

Java Implementation

public boolean canConstruct(String ransomNote, String magazine) { // Use StringBuilder to allow "deleting" characters StringBuilder magazineBuilder = new StringBuilder(magazine); for (char noteChar : ransomNote.toCharArray()) { int indexInMagazine = magazineBuilder.indexOf(String.valueOf(noteChar)); if (indexInMagazine == -1) { return false; // Character not found } // Remove the character so it cannot be reused magazineBuilder.deleteCharAt(indexInMagazine); } return true; }

Complexity Analysis

Time Complexity: O(m * n) where n is ransomNote length and m is magazine length. For each character in n, we potentially scan all of m.

Space Complexity: O(m) to store the StringBuilder copy of the magazine.

Pros & Cons

Pros: Very simple to understand and implement without extra algorithms.

Cons: Extremely inefficient for large strings due to the nested linear search.

Approach 2: Sort and Two Pointers

By sorting both strings, we can use two pointers to move through both strings linearly. If the magazine character is "smaller" than the note character, we skip it.

Java Implementation

public boolean canConstruct(String ransomNote, String magazine) { char[] noteChars = ransomNote.toCharArray(); char[] magChars = magazine.toCharArray(); Arrays.sort(noteChars); Arrays.sort(magChars); int notePtr = 0; int magPtr = 0; while (notePtr < noteChars.length && magPtr < magChars.length) { if (noteChars[notePtr] == magChars[magPtr]) { // Match found, move both pointers notePtr++; magPtr++; } else if (magChars[magPtr] < noteChars[notePtr]) { // Magazine character is too small, skip it magPtr++; } else { // Magazine character is larger than note character // Since sorted, we will never find noteChars[notePtr] further right return false; } } return notePtr == noteChars.length; }

Complexity Analysis

Time Complexity: O(m log m + n log n) dominated by the sorting step.

Space Complexity: O(m + n) or O(log m + log n) depending on the sorting implementation (TimSort uses space).

Pros & Cons

Pros: No extra data structures like Maps are needed, strictly pointers and arrays.

Cons: Sorting overhead makes it slower than counting frequencies for most cases.

Approach 3: Frequency Array (Optimal)

Since the input is limited to lowercase English letters, we can use a fixed-size integer array of 26 to count frequencies. This is effectively a high-performance Hash Map.

Java Implementation

public boolean canConstruct(String ransomNote, String magazine) { // Fixed size array for 26 lowercase English letters int[] alphabetFrequencies = new int[26]; // Increment count for each character in the magazine for (char magazineChar : magazine.toCharArray()) { alphabetFrequencies[magazineChar - 'a']++; } // Decrement count for each character in the ransom note for (char noteChar : ransomNote.toCharArray()) { int index = noteChar - 'a'; alphabetFrequencies[index]--; // If count drops below zero, we ran out of this letter if (alphabetFrequencies[index] < 0) { return false; } } return true; }

Complexity Analysis

Time Complexity: O(m + n). We iterate through the magazine once and the ransomNote once.

Space Complexity: O(1). The array size is constant (26), regardless of input size.

Pros & Cons

Pros: Optimal performance. Extremely fast because array access is O(1) and it uses very little memory.

Cons: Only works if character range is known (e.g., lowercase ASCII). For Unicode, use a HashMap.

Coding Cheat Sheet

Key Takeaways

  • • "Can form word A from word B" usually implies counting frequencies.
  • • In Java, magazineChar - 'a' is a common trick to map 'a'-'z' to 0-25.
  • • Constraints matter: "lowercase English letters" is a huge hint to use an int[26].
  • • If characters were Unicode, you would use HashMap<Character, Integer>.

Edge Cases

  • • ransomNote empty: Should return true (0 characters needed).
  • • magazine empty: Should return false (unless note is also empty).
  • • Note longer than magazine: Can return false immediately.
  • • Single character match/mismatch: Ensure pointers handle length 1.
