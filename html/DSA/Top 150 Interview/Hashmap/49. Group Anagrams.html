49. Group Anagrams

Collecting words that share the same letters into buckets.

Medium Hash Table Sorting

Problem Description

Given an array of strings strs, group the anagrams together. You can return the answer in any order.

Example: strs = ["eat","tea","tan","ate","nat","bat"]
Output: [["bat"],["nat","tan"],["ate","eat","tea"]]

Approach 1: Brute Force Comparison

The direct approach: Maintain a list of groups. For each new word, compare it against the first word of every existing group to see if it's an anagram.

Java Implementation

public List<List<String>> groupAnagrams(String[] strs) { List<List<String>> resultGroups = new ArrayList<>(); for (String currentWord : strs) { boolean addedToGroup = false; // Compare currentWord with existing groups for (List<String> group : resultGroups) { if (isAnagram(currentWord, group.get(0))) { group.add(currentWord); addedToGroup = true; break; } } // If no matching group found, start a new one if (!addedToGroup) { List<String> newGroup = new ArrayList<>(); newGroup.add(currentWord); resultGroups.add(newGroup); } } return resultGroups; }

Complexity

Time: O(N² * K) where N is number of words and K is max length. We do N calls, each potentially comparing against N groups, and comparison takes O(K).

Space: O(N * K) to store the resulting groups.

Pros & Cons

Pros: Very simple logic; works with any string type.

Cons: Extremely slow. Comparing every word against every group representative is redundant.

Approach 2: Categorize by Sorting

All anagrams share a unique property: when their characters are sorted alphabetically, they result in the exact same string. We use this sorted string as a HashMap key.

Java Implementation

public List<List<String>> groupAnagrams(String[] strs) { Map<String, List<String>> groups = new HashMap<>(); for (String currentWord : strs) { char[] characters = currentWord.toCharArray(); Arrays.sort(characters); String sortedKey = new String(characters); // If key doesn't exist, create an empty list groups.computeIfAbsent(sortedKey, k -> new ArrayList<>()); groups.get(sortedKey).add(currentWord); } return new ArrayList<>(groups.values()); }

Complexity

Time: O(N * K log K). We iterate N words, and for each, we sort a string of length K.

Space: O(N * K) to store the contents of the HashMap.

Pros & Cons

Pros: Fast enough for most cases. Very clean and readable implementation.

Cons: String sorting (K log K) can be expensive if words are very long.

Approach 3: Categorize by Frequency (Optimal)

Instead of sorting, we can represent each word by its character counts (e.g., 'a' appears 2 times, 'b' 0 times). This frequency array is converted into a string key like "#2#0#1...".

Java Implementation

public List<List<String>> groupAnagrams(String[] strs) { Map<String, List<String>> groups = new HashMap<>(); for (String s : strs) { int[] counts = new int[26]; for (char c : s.toCharArray()) counts[c - 'a']++; // Build a string key using counts StringBuilder keyBuilder = new StringBuilder(); for (int count : counts) { keyBuilder.append('#'); keyBuilder.append(count); } String key = keyBuilder.toString(); groups.computeIfAbsent(key, k -> new ArrayList<>()); groups.get(key).add(s); } return new ArrayList<>(groups.values()); }

Complexity

Time: O(N * K). We avoid sorting entirely. Character counting and key building are both linear in word length.

Space: O(N * K) to store the HashMap entries.

Efficiency Note

This is the optimal approach for long strings because O(K) is faster than O(K log K). It relies on the constraint that characters are lowercase English letters.

Interview Cheat Sheet

Key Concepts

  • • Anagrams share the same sorted character sequence.
  • • Anagrams share the same character frequency distribution.
  • • Use a HashMap to group items with identical properties.
  • • In Java, Arrays.sort(char[]) is the fastest sorting helper.

Edge Cases

  • • Empty array: Return empty list.
  • • Empty string: "" is a valid string, groups together.
  • • Single word: Results in a single list containing one word.
  • • No Anagrams: Every word gets its own list.
