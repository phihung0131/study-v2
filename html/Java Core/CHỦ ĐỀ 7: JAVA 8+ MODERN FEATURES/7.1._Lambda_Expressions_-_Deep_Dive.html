Lambda Expressions

Chào mừng em. Trước Java 8, chúng ta bị mắc kẹt trong "Verbosisty" (sự rườm rà). Muốn truyền một hành động, ta phải bọc nó vào một Class nặc danh (Anonymous Class). Lambda ra đời để giải phóng tư duy đó.

1 Tại sao cần Lambda?

Cũ: Anonymous Inner Class

button.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("Clicked!");
    }
});
MAGIC!

Mới: Lambda Expression

button.addActionListener(e -> System.out.println("Clicked!"));
Phân tích: Chúng ta đã loại bỏ 5 dòng code boilerplate. Thứ duy nhất quan trọng là System.out.println.

2 Cấu trúc giải phẫu (Anatomy)

(parameters) -> { expression / body }
No Parameters
() -> System.out.print("Hi")
One Parameter
s -> System.out.print(s)
Multi-line Body
(x, y) -> { int sum = x + y; return sum; }

3 Điều kiện cần: Functional Interface

Lambda chỉ có thể được gán cho một interface có duy nhất một phương thức trừu tượng (Single Abstract Method - SAM).

Standard Interfaces

  • • Predicate<T>: Kiểm tra (trả về boolean)
  • • Consumer<T>: Tiêu thụ (không trả về gì)
  • • Supplier<T>: Cung cấp (không nhận input)
  • • Function<T, R>: Chuyển đổi T thành R

Annotation

@FunctionalInterface
public interface MyCalc {
    int operate(int a, int b);
}

4 Under the Hood: invokedynamic

Dành cho Senior: Lambda KHÔNG phải là cú pháp thay thế hoàn toàn cho Anonymous Class.

1. Compile Time: Trình biên dịch tạo ra một phương thức private trong chính class đó chứa logic của Lambda.
2. Runtime: JVM sử dụng lệnh invokedynamic để gọi LambdaMetafactory.
3. Result: Một đối tượng được tạo ra "on-the-fly" mà không cần file .class riêng biệt như Anonymous Class.
Memory Simulation

Lambda reduces footprint by avoiding extra inner class metadata.

5. Variable Capture & Closure

Lambda có thể truy cập các biến bên ngoài, nhưng có một quy tắc sắt đá: Biến đó phải là Final hoặc Effectively Final.

int count = 10; // Biến này không được thay đổi sau đây
Runnable r = () -> System.out.println(count); 

// count = 20; // ERROR: Local variable count defined in an enclosing 
               // scope must be final or effectively final
Giải thích kỹ thuật: Các biến local nằm trên Stack, Lambda đối tượng nằm trên Heap. Nếu Stack Frame của phương thức bị xóa mà Lambda vẫn tồn tại, Java cần copy giá trị biến đó vào Lambda. Để đảm bảo tính nhất quán, giá trị đó không được phép thay đổi.

6. Best Practices & Anti-patterns

Nên: Giữ cho Lambda ngắn gọn

Lý tưởng nhất là 1 dòng code. Nếu nhiều hơn 3 dòng, hãy trích xuất ra một method riêng.

Nên: Sử dụng Method References

list.forEach(System.out::println) thay vì list.forEach(s -> System.out.println(s)).

Không nên: Side effects

Hạn chế thay đổi trạng thái của các đối tượng bên ngoài trong body của lambda.

Không nên: Ép kiểu quá đà

Đừng viết Lambda quá phức tạp khiến trình biên dịch không thể suy luận kiểu dữ liệu (Type Inference).

🎯 Câu hỏi phỏng vấn hóc búa

Q1: Sự khác biệt lớn nhất giữa 'this' trong Anonymous Class và Lambda là gì?

Trong Anonymous Class, 'this' trỏ tới chính đối tượng Anonymous đó. Trong Lambda, 'this' trỏ tới đối tượng của lớp bao quanh (lexical scoping). Lambda không tạo ra một phạm vi mới cho 'this'.

Q2: Tại sao Lambda giúp ứng dụng chạy nhanh hơn và tốn ít bộ nhớ hơn Anonymous Class?

Anonymous class tạo ra một file .class riêng biệt, được classloader tải lên và cấp phát bộ nhớ như một class thông thường. Lambda sử dụng invokedynamic, trì hoãn việc tạo đối tượng đến runtime và thường tái sử dụng các instance nếu Lambda không capture biến nào, giúp giảm áp lực cho Garbage Collector.
