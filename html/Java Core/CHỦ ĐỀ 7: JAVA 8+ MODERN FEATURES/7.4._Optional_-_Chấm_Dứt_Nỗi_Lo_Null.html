2. Ba cách khởi tạo cơ bản

Optional.of(value)

Dùng khi em chắc chắn giá trị không null. Nếu null, nó quăng NPE ngay lập tức.

Optional.ofNullable(val)

An toàn nhất. Nếu val null, nó trả về Optional.empty().

Optional.empty()

Trả về một chiếc hộp rỗng (Singleton instance).

3. Transformation: Tư duy như Stream

Em có thể coi Optional là một Stream có tối đa 1 phần tử.

map

Biến đổi giá trị bên trong nếu nó tồn tại.

opt.map(s -> s.length())
filter

Giữ lại giá trị nếu thỏa mãn điều kiện, nếu không biến thành Empty.

opt.filter(s -> s.startsWith("J"))
flatMap

Dùng khi hàm biến đổi bản thân nó cũng trả về Optional (tránh lồng hộp).

userOpt.flatMap(User::getAddressOpt)

4. Unwrapping: Lấy giá trị ra an toàn

orElse vs orElseGet
// Eager: Luôn gọi phương thức default cho dù opt có giá trị
String res = opt.orElse(getDefault()); 

// Lazy: Chỉ gọi phương thức default khi opt rỗng
String res = opt.orElseGet(() -> getDefault());

Senior Note: Luôn ưu tiên orElseGet nếu việc tạo giá trị mặc định tốn tài nguyên.

Xử lý ném Exception
// Java 10+
User user = opt.orElseThrow(); 

// Tùy chỉnh Exception
User user = opt.orElseThrow(() -> 
    new NotFoundException("User not found"));

# Internals: Hiệu năng và Bộ nhớ

Cost of Optional

Optional là một Object. Việc bao bọc một giá trị vào Optional tốn thêm bộ nhớ trên Heap và thêm một bước truy xuất (indirection).

Đối với Primitive, hãy dùng: OptionalInt, OptionalLong, OptionalDouble để tránh boxing/unboxing lãng phí.

Serialization Issue

Optional KHÔNG thực thi interface Serializable.

Rule: Đừng bao giờ dùng Optional cho các Field trong Entity/POJO. Chỉ dùng làm kiểu trả về cho Method.

6. Những sai lầm phổ biến (Anti-patterns)

1. Gọi .get() mà không kiểm tra

Code: opt.get(); -> Vẫn dính NoSuchElementException. Chẳng khác gì NPE!

2. Dùng Optional cho tham số (Parameters)

Code: public void find(Optional<String> name) -> Ép người gọi phải bọc data vào hộp, làm code rườm rà không cần thiết. Hãy dùng Overloading hoặc check null trực tiếp.

3. Gán null cho biến Optional

Code: Optional<User> user = null; -> Đây là "tội ác". Hãy dùng Optional.empty().

🎯 Senior Interview Challenge

Q1: Tại sao Optional không implements Serializable?

A: Các nhà thiết kế Java dự định Optional chỉ dùng làm kiểu trả về tạm thời (short-lived) cho library APIs, không phải để lưu trữ lâu dài trong database hay gửi qua mạng (RMI).

Q2: ifPresent() và ifPresentOrElse() khác gì nhau?

A: ifPresent chỉ nhận 1 Consumer xử lý khi có data. ifPresentOrElse (Java 9+) nhận thêm 1 Runnable để xử lý trường hợp rỗng, giúp loại bỏ hoàn toàn câu lệnh if-else.

Q3: Sự khác biệt giữa flatMap của Stream và flatMap của Optional?

A: Stream flatMap làm phẳng các Stream bên trong. Optional flatMap làm phẳng (unwrap) các Optional lồng nhau để trả về 1 Optional duy nhất.
