1 Đặc tính Set Interface

Uniqueness

Không cho phép phần tử trùng lặp. Nếu thêm phần tử đã tồn tại, `add()` sẽ trả về `false`.

Null Handling

Cho phép tối đa một phần tử `null`.

Speed

Tốc độ tìm kiếm (`contains`) cực nhanh: $O(1)$.

2 HashSet Internals: Cái bóng của HashMap

"Bí mật lớn nhất của HashSet: Nó không tự quản lý dữ liệu. Nó thuê một `HashMap` làm việc đó." 

// Source code internals (OpenJDK 17)
public class HashSet<E> extends AbstractSet<E> implements Set<E> {
    
    private transient HashMap<E, Object> map;

    // Giá trị giả (Dummy value) để lưu vào Map vì Map cần cặp Key-Value
    private static final Object PRESENT = new Object();

    public HashSet() {
        map = new HashMap<>();
    }

    public boolean add(E e) {
        // Trả về null nếu key chưa tồn tại, trả về giá trị cũ nếu đã có
        return map.put(e, PRESENT) == null;
    }
}

Khi em gọi hashSet.add("Java"), thực chất em đang gọi hashMap.put("Java", PRESENT). Vì `Key` trong `HashMap` là duy nhất, nên `HashSet` nghiễm nhiên trở thành duy nhất.

3 The Mandatory Contract: equals() & hashCode()

Quy tắc "Vàng" của Senior:

"Nếu hai object bằng nhau (equals == true), chúng PHẢI có cùng hashCode.
Nếu hai object có cùng hashCode, chưa chắc chúng đã bằng nhau (Collision)."

Tại sao quy tắc này lại sống còn với HashSet?

  1. Step 1: HashSet dùng `hashCode()` để tìm "thùng" (bucket) chứa dữ liệu.
  2. Step 2: Nếu bucket đó có sẵn dữ liệu (Collision), nó dùng `equals()` để kiểm tra xem object mới có thực sự trùng với object cũ không.

⚠️ Nếu em override `equals` mà quên `hashCode`, Set sẽ chứa các phần tử trùng lặp "vô lý" vì chúng rơi vào các bucket khác nhau.

// Anti-pattern!
class Student {
    String id;
    @Override
    public boolean equals(Object o) { ... }
    // QUÊN OVERRIDE hashCode() -> BUG!
}

Mentor Tip: Load Factor & Initial Capacity

Giống như ArrayList, HashSet cũng có cơ chế resize. Mặc định là khi đạt 75% capacity (Load Factor = 0.75), nó sẽ tăng gấp đôi kích thước. Nếu em biết trước mình sẽ lưu 10,000 phần tử, hãy khởi tạo new HashSet<>(13334) để tránh resize.

7 Câu hỏi Phỏng vấn Senior

Câu 1: Chuyện gì xảy ra nếu hai phần tử khác nhau có cùng hashCode? ▼

Đây gọi là Hash Collision. Cả hai phần tử sẽ rơi vào cùng một bucket. Trong Java 8+, nếu bucket đó vượt quá 8 phần tử, nó sẽ chuyển từ LinkedList sang Self-balancing BST (Red-Black Tree) để giữ hiệu năng tìm kiếm là $O(\log n)$ thay vì $O(n)$.

Câu 2: Tại sao phương thức add() của HashSet lại trả về boolean? ▼

Vì `HashSet` dựa trên `HashMap.put(K, V)`. Phương thức `put` của Map trả về giá trị cũ của Key. Trong HashSet, giá trị luôn là `PRESENT`. Nếu `put` trả về `null`, nghĩa là Key đó chưa từng tồn tại -> `add` thành công (true). Nếu trả về `PRESENT`, nghĩa là đã có rồi -> `add` thất bại (false).
