Chi tiết: ThreadPoolExecutor

Mọi Thread Pool đều xoay quanh class ThreadPoolExecutor với các tham số quan trọng mà một Senior phải thuộc lòng:

corePoolSize

Số lượng thread tối thiểu luôn được giữ trong pool, kể cả khi chúng rảnh rỗi.

maximumPoolSize

Số lượng thread tối đa pool có thể mở rộng khi Queue đã đầy.

workQueue

Nơi chứa các task đang chờ xử lý (Thường dùng LinkedBlockingQueue).

RejectedExecutionHandler

Chiến lược xử lý khi Pool quá tải (Abort, CallerRuns, Discard...).

Triển khai thực tế

ThreadService.java Best Practice 
ExecutorService executor = Executors.newFixedThreadPool(10);

// Gửi task không cần kết quả trả về
executor.execute(() -> {
    System.out.println("Processing async task: " + Thread.currentThread().getName());
});

// Gửi task có lấy kết quả (Future)
Future<String> future = executor.submit(() -> {
    Thread.sleep(2000);
    return "Task Completed!";
});

// Luôn nhớ đóng pool khi ứng dụng tắt
executor.shutdown();

4 Loại Pool phổ biến

1. FixedThreadPool

Số lượng thread cố định. Phù hợp cho server có tài nguyên xác định trước.

2. CachedThreadPool

Tạo thread mới nếu cần, hủy thread nếu rảnh quá 60s. Phù hợp cho nhiều task ngắn hạn.

3. SingleThreadExecutor

Chỉ 1 thread duy nhất. Đảm bảo các task chạy tuần tự (Sequential).

4. ScheduledThreadPool

Dùng để lập lịch chạy task sau một khoảng thời gian hoặc định kỳ.

Mentor's Golden Rules

⚠️

Không dùng Executors.newCachedThreadPool() bừa bãi!

Nó có maximumPoolSize = Integer.MAX_VALUE. Nếu hệ thống bị spike request, nó sẽ tạo hàng triệu thread và gây sập hệ thống (OOM).

💡

Đặt tên cho Thread (Naming Policy)

Sử dụng ThreadFactory để đặt tên thread (ví dụ: "inventory-worker-1"). Điều này cực kỳ quan trọng khi bạn cần phân tích Thread Dump lúc gặp lỗi.

Câu hỏi Mentor (Interview Level)

Q1: Sự khác biệt giữa submit() và execute()?

Trả lời: execute dùng cho các task "fire and forget". submit trả về một Future object giúp bạn lấy kết quả trả về hoặc bắt Exception xảy ra bên trong thread.

Q2: Làm sao để định lượng số thread tối ưu (Tuning)?

Trả lời: Công thức chung:
- CPU-Intensive task: số core + 1.
- I/O-Intensive task (DB, API): số core * (1 + thời gian chờ / thời gian xử lý).
