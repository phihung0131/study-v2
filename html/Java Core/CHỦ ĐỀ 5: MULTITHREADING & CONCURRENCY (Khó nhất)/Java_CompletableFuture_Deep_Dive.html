CompletableFuture:
The Power of Asynchrony

Tại sao Future thôi là chưa đủ?

Interface Future chỉ cho phép bạn kiểm tra xem task xong chưa. Nó không cho phép bạn nói: "Khi task này xong, hãy tự động làm việc tiếp theo".

CompletableFuture giải quyết điều này bằng mô hình Callback-driven. Bạn định nghĩa pipeline, và data sẽ tự động chảy qua đó.

1. Khởi tạo tác vụ bất đồng bộ

No Result

runAsync()

Dùng cho các task không trả về kết quả (Runnable).

Has Result

supplyAsync()

Dùng cho các task trả về giá trị (Supplier).

// Tự động chạy trên ForkJoinPool.commonPool()
CompletableFuture<String> cf = CompletableFuture.supplyAsync(() -> {
    simulateSlowNetwork();
    return "Dữ liệu từ Server";
});

2. Nghệ thuật nối chuỗi (Chaining)

thenApply(Function)

Biến đổi kết quả từ kiểu A sang kiểu B (Giống map trong Stream).

thenAccept(Consumer)

Sử dụng kết quả cuối cùng (thường để in log hoặc lưu DB) và không trả về gì nữa.

thenCompose vs thenCombine

Compose: Khi task 2 phụ thuộc vào kết quả task 1 (FlatMap).

Combine: Chạy 2 task độc lập và kết hợp kết quả khi cả hai đều xong.

CompletableFuture.supplyAsync(() -> fetchUser(id))
    .thenApply(user -> user.getName().toUpperCase()) // Biến đổi
    .thenCompose(name -> fetchOrders(name))          // Nối tiếp task khác
    .thenAccept(orders -> System.out.println(orders)); // Kết thúc

⚠️ Xử lý lỗi trong thế giới Async

Trong đa luồng, Exception xảy ra ở thread con sẽ không làm crash thread chính. Bạn cần bắt nó chủ động.

exceptionally()

Cung cấp giá trị mặc định nếu có lỗi xảy ra ở bất kỳ mắt xích nào phía trước.

handle()

Mạnh mẽ hơn, cho phép xử lý cả trường hợp thành công và thất bại trong một hàm.

Deep Dive: Threading Model

Mặc định: ForkJoinPool.commonPool()

Mặc định, CF sử dụng Pool chung của toàn JVM.

Nguy hiểm: Nếu bạn chạy các task I/O (gọi API chậm) trên commonPool, bạn có thể làm nghẽn toàn bộ ứng dụng (kể cả Parallel Streams).

Best Practice: Luôn truyền một Custom Executor (Thread Pool riêng) cho các phương thức có hậu tố *Async. 

Executor myPool = Executors.newFixedThreadPool(50);
CompletableFuture.supplyAsync(() -> task(), myPool);

Mentor's Interview Tips

Q1: Sự khác biệt giữa thenApply và thenApplyAsync?

Trả lời: thenApply chạy callback trên cùng thread của task trước đó. thenApplyAsync sẽ đẩy callback vào một thread khác từ Pool để thực thi, giúp giải phóng thread hiện tại nhanh hơn.

Q2: Làm sao để chờ NHIỀU CompletableFuture cùng lúc?

Trả lời: Dùng CompletableFuture.allOf(cf1, cf2, ...) để đợi tất cả xong, hoặc anyOf(...) để lấy kết quả từ thằng nhanh nhất.
