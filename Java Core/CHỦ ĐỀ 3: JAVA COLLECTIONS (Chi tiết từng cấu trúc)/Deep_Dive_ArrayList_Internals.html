ArrayList: Master the Internals

1 Tổng quan & Phân cấp (Hierarchy)

ArrayList là gì?

ArrayList là một Mảng động (Dynamic Array). Khác với mảng thông thường (`[]`), ArrayList có thể tự động tăng kích thước khi cần thiết.

  • Cho phép phần tử trùng lặp.
  • Bảo toàn thứ tự thêm vào (Insertion order).
  • Triển khai interface RandomAccess (Truy cập cực nhanh qua index).

2 Cấu trúc Internals (Under the hood)

"ArrayList thực chất chỉ là một Wrapper bao quanh một mảng Object thuần túy."

// Cấu trúc cốt lõi trong OpenJDK 17
public class ArrayList<E> extends AbstractList<E> implements List<E>, RandomAccess, Cloneable, java.io.Serializable {
    
    // Mảng lưu trữ thực sự
    transient Object[] elementData; 

    // Số lượng phần tử hiện tại
    private int size;

    // Dung lượng mặc định ban đầu (Nếu không chỉ định)
    private static final int DEFAULT_CAPACITY = 10;
    
    // ...
}

Lưu ý quan trọng về `transient`:

Tại sao `elementData` lại là transient? Vì ArrayList tự triển khai cơ chế Serialization riêng (qua `writeObject` / `readObject`) để tránh serialize những ô trống (null) trong mảng nhằm tiết kiệm băng thông mạng/dung lượng đĩa.

3 Cơ chế Resizing (Growth Strategy)

Khi số lượng phần tử vượt quá `capacity`, ArrayList sẽ gọi phương thức `grow()`:

int oldCapacity = elementData.length;
int newCapacity = oldCapacity + (oldCapacity >> 1); // 1.5 lần

Dịch bit >> 1 tương đương chia 2. Công thức: $New = Old + Old/2$.

4 Phân tích Hiệu năng (Time Complexity)

Thao tác Time Complexity Lý do
Add (vào cuối) O(1) * Amortized (khấu hao), thỉnh thoảng tốn O(n) khi resize.
Get/Set (theo index) O(1) Nhờ truy cập mảng qua offset bộ nhớ trực tiếp.
Add (giữa/đầu) O(n) Phải dịch chuyển các phần tử phía sau sang phải.
Remove O(n) Phải dịch chuyển các phần tử phía sau sang trái.

5 Fail-Fast & modCount

Đã bao giờ em gặp ConcurrentModificationException chưa? Đó chính là cơ chế **Fail-Fast**.

List<String> list = new ArrayList<>();
list.add("Java");
list.add("Kotlin");

for (String s : list) {
    if (s.equals("Java")) {
        list.remove(s); // BOOM! Tung Exception
    }
}

Cơ chế bảo vệ:

ArrayList duy trì một biến `modCount` (số lần mảng bị thay đổi cấu trúc). Khi khởi tạo một Iterator, nó sẽ copy giá trị này vào `expectedModCount`. Mỗi lần loop, nó kiểm tra:

if (modCount != expectedModCount)
  throw new ConcurrentModificationException();

6 Best Practices (Senior Level)

Luôn chỉ định Initial Capacity nếu biết trước kích thước

Tránh việc CPU phải resize liên tục 1.5x (copy mảng cực tốn kém).

List<Item> items = new ArrayList<>(1000);

Ưu tiên `ArrayList` thay vì `LinkedList` trong hầu hết các case thực tế

LinkedList có chi phí bộ nhớ cực lớn cho mỗi node (Next/Prev pointers) và hiệu năng cache (spatial locality) kém xa ArrayList.

7 Thử thách Tư duy (Interview Prep)

Câu 1: ArrayList có Thread-safe không? Làm sao để nó Thread-safe? ▼

Không. ArrayList không đồng bộ (non-synchronized). Để nó thread-safe, em có 2 cách:

  1. Dùng Collections.synchronizedList(new ArrayList<>()).
  2. Dùng CopyOnWriteArrayList (Phù hợp cho case Read nhiều Write ít).
Câu 2: Tại sao ArrayList xóa phần tử ở đầu lại chậm hơn ở cuối? ▼

Vì khi xóa ở đầu (index 0), Java phải gọi System.arraycopy() để dịch chuyển tất cả n-1 phần tử còn lại lên phía trước 1 vị trí để lấp đầy khoảng trống. Xóa ở cuối thì chỉ cần set `null` và giảm `size` (O(1)).
