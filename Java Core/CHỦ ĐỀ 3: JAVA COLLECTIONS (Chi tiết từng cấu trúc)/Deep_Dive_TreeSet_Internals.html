2 TreeSet Internals: Lại là TreeMap!

"Giống như HashSet dựa vào HashMap, TreeSet thực chất là một lớp vỏ bao quanh `TreeMap`." 

// Source code (OpenJDK 17)
public class TreeSet<E> extends AbstractSet<E> implements NavigableSet<E> {
    
    // Đối tượng TreeMap thực hiện mọi công việc nặng nhọc
    private transient NavigableMap<E,Object> m;

    // Giá trị giả
    private static final Object PRESENT = new Object();

    public TreeSet() {
        this(new TreeMap<>());
    }

    public boolean add(E e) {
        return m.put(e, PRESENT) == null;
    }
}

TreeMap sử dụng cấu trúc Red-Black Tree. Đây là một loại cây tìm kiếm nhị phân tự cân bằng, đảm bảo độ phức tạp cho các thao tác tìm kiếm, thêm, xóa luôn là O(log n).

3 Ordering: Ai đứng trước?

1. Natural Ordering

Các phần tử phải triển khai interface Comparable.

class Student implements Comparable<Student> {
    int score;
    public int compareTo(Student other) {
        return this.score - other.score;
    }
}

2. Custom Ordering

Sử dụng Comparator khi khởi tạo TreeSet.

// Sắp xếp ngược
TreeSet<Integer> set = new TreeSet<>(
    (a, b) -> b - a
);

5 Các hàm NavigableSet "đỉnh cao"

NavigableSet cung cấp các phương thức tìm kiếm cực kỳ linh hoạt:

lower(e): Lấy phần tử < e lớn nhất.
higher(e): Lấy phần tử > e nhỏ nhất.
floor(e): Lấy phần tử ≤ e lớn nhất.
ceiling(e): Lấy phần tử ≥ e nhỏ nhất.
descendingSet(): Trả về một Set được đảo ngược thứ tự (O(1)).

Mentor Note: Uniqueness in TreeSet

Khác với HashSet dùng equals(), TreeSet sử dụng compareTo() (hoặc compare()) để xác định sự trùng lặp.
Nếu compareTo() trả về 0, TreeSet coi đó là phần tử trùng và sẽ không thêm vào, bất kể equals() có trả về gì đi nữa. Đây là điểm hay gây bug nếu em code không nhất quán.

7 Câu hỏi Phỏng vấn Senior

Q1: Tại sao TreeSet lại chậm hơn HashSet?

Mentor Answer: Vì HashSet chỉ tính mã hash và truy cập mảng ($O(1)$). Trong khi TreeSet phải duyệt cây, thực hiện các phép so sánh và thỉnh thoảng phải "xoay cây" (rotate) để tái cân bằng sau mỗi lần thêm/xóa ($O(\log n)$).

Q2: Tôi có thể lưu String trong TreeSet không? Tại sao?

Mentor Answer: Có, vì class `String` đã implement Comparable<String> sẵn (sắp xếp theo thứ tự từ điển).
