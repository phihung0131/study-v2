LinkedList: The Chain of Nodes

Khám phá sức mạnh của Doubly Linked List và lý do tại sao nó lại là một Deque tuyệt vời.

1 Đặc điểm & Phân cấp (Hierarchy)

LinkedList là gì?

LinkedList trong Java là một triển khai của Doubly Linked List (Danh sách liên kết đôi). Mỗi phần tử là một "Node" riêng biệt nằm rải rác trong bộ nhớ Heap.

✨ Điểm đặc biệt: Nó vừa là một List, vừa là một Deque (Double Ended Queue).

Nhờ implement Deque, LinkedList có thể dùng làm Stack (LIFO) hoặc Queue (FIFO).

2 Cấu trúc Node Đôi (Internals)

Mọi thứ trong LinkedList xoay quanh một class nội tĩnh (static inner class) tên là Node.

// Source code internals (Java 17)
public class LinkedList<E> extends AbstractSequentialList<E> implements List<E>, Deque<E> {
    
    transient int size = 0;
    transient Node<E> first; // Con trỏ tới Node đầu tiên
    transient Node<E> last;  // Con trỏ tới Node cuối cùng

    private static class Node<E> {
        E item;        // Dữ liệu thực tế
        Node<E> next;  // Con trỏ tới Node tiếp theo
        Node<E> prev;  // Con trỏ tới Node phía trước

        Node(Node<E> prev, E element, Node<E> next) {
            this.item = element;
            this.next = next;
            this.prev = prev;
        }
    }
}

Tư duy Senior: Vì là liên kết đôi, việc thêm/xóa ở cả hai đầu (Head/Tail) chỉ tốn O(1) vì ta luôn giữ tham chiếu tới `first` và `last`.

5 Cạm bẫy Bộ nhớ (Memory Overhead)

Tại sao LinkedList lại tốn memory?

Trong một JVM 64-bit:

  • ArrayList: Chỉ tốn 4-8 bytes cho mỗi reference trong mảng.
  • LinkedList Node:
    • Object Header: 12-16 bytes
    • Item Ref: 4-8 bytes
    • Next Ref: 4-8 bytes
    • Prev Ref: 4-8 bytes
    Tổng cộng: ~40 bytes cho mỗi phần tử!

Spatial Locality (Tính cục bộ không gian)

ArrayList nằm liên tục trong bộ nhớ, tận dụng được **CPU Cache**. LinkedList nằm rải rác, khiến CPU phải nhảy liên tục (Cache Miss), dẫn đến tốc độ duyệt (iteration) thực tế của LinkedList thường chậm hơn ArrayList rất nhiều.

6 Best Practices: Khi nào nên dùng?

✅ Use LinkedList when:

  • Bạn cần một Queue hoặc Stack (dùng push/pop, offer/poll).
  • Bạn thêm/xóa phần tử ở đầu danh sách cực kỳ thường xuyên.
  • Bạn đang dùng ListIterator để duyệt và sửa đổi danh sách tại vị trí hiện tại.

❌ Avoid LinkedList when:

  • Bạn cần truy cập ngẫu nhiên list.get(index).
  • Bộ nhớ hạn chế (vì overhead của Node quá lớn).
  • Ưu tiên hiệu năng duyệt nhanh (ArrayList nhanh hơn do Cache Locality).

7 Thử thách Mentor (Interview Questions)

Q1: LinkedList tìm kiếm phần tử theo index như thế nào để tối ưu?

Mentor Answer: Java không duyệt từ đầu đến cuối một cách mù quáng. Nó kiểm tra nếu index < (size >> 1) (tức là index nằm ở nửa đầu), nó sẽ duyệt từ first. Ngược lại, nó sẽ duyệt ngược từ last. Tuy tối ưu gấp đôi nhưng độ phức tạp vẫn là O(n/2) ≈ O(n).

Q2: Tại sao LinkedList lại implement RandomAccess?

Mentor Answer: Đánh bẫy rồi! LinkedList **KHÔNG** implement RandomAccess. Chỉ có ArrayList và Vector mới implement interface đánh dấu này. Đây là cách để các thuật toán (như Collections.binarySearch) biết rằng việc truy cập theo index là nhanh hay chậm để chọn chiến thuật phù hợp.
