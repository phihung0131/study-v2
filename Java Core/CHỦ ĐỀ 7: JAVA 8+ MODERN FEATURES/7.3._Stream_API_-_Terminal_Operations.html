PHẦN KẾT CỦA STREAM

Terminal Operations

Nếu Intermediate Ops định nghĩa cách thức xử lý dữ liệu, thì Terminal Ops định nghĩa kết quả cuối cùng. Khi một Terminal Op được gọi: Pipeline được thực thi, dữ liệu được tiêu thụ, và Stream sẽ bị đóng lại.

! Đặc điểm cốt lõi

Eager Execution

Kích hoạt toàn bộ Pipeline ngay lập tức.

Single Use

Sau khi gọi, Stream không thể tái sử dụng.

Return Value

Trả về List, Map, Optional, Primitive hoặc Void.

2. Mutable Reduction: collect()

Đây là phương thức quan trọng nhất. Nó biến đổi Stream thành một Collection hoặc một cấu trúc dữ liệu tùy chỉnh.

// Gom vào List
List<String> list = stream.collect(Collectors.toList());

// Gom vào Map (Key: ID, Value: Name)
Map<Integer, String> map = users.stream()
    .collect(Collectors.toMap(User::getId, User::getName));

4. Immutable Reduction: reduce()

Dùng khi em muốn kết hợp các phần tử thành **một giá trị duy nhất** (như tính tổng, tích, hoặc tìm chuỗi dài nhất).

Cú pháp 2 tham số:
int sum = numbers.stream()
    .reduce(0, (a, b) -> a + b);

0 là giá trị khởi tạo (Identity).

Cú pháp 1 tham số:
Optional<Integer> max = numbers.stream()
    .reduce(Math::max);

Trả về Optional vì stream có thể rỗng.

5. Short-circuiting Operations

Matching
  • • anyMatch: Trả về true nếu ít nhất 1 cái đúng.
  • • allMatch: Trả về true nếu tất cả đều đúng.
  • • noneMatch: Trả về true nếu không cái nào đúng.
Finding
  • • findFirst: Lấy phần tử đầu tiên gặp được.
  • • findAny: Lấy bất kỳ phần tử nào (tốt cho Parallel).
Senior Insight: Các hàm này không duyệt hết Stream. Ngay khi có kết quả, nó sẽ dừng Pipeline -> Cực kỳ tối ưu hiệu năng.

6. Deep Dive: Collector Interface

Để tùy chỉnh cách collect() hoạt động, em cần hiểu 4 thành phần của interface Collector<T, A, R>:

Supplier
Tạo container rỗng (new ArrayList)
Accumulator
Thêm phần tử vào container
Combiner
Gộp 2 container (cho Parallel)
Finisher
Biến đổi container thành kết quả cuối

🎯 Senior Interview Prep

Q1: Phân biệt reduce() và collect()? Khi nào dùng cái nào? ↓
- collect() thực hiện Mutable Reduction. Nó thay đổi một container có sẵn (như thêm vào List). Rất hiệu quả về bộ nhớ khi làm việc với Collections.
- reduce() thực hiện Immutable Reduction. Nó luôn tạo ra một giá trị mới từ hai giá trị cũ. Thích hợp cho các kiểu dữ liệu primitive (int, double) hoặc các Immutable Objects (String, BigInteger).
- Rule: Nếu muốn gom vào Collection, hãy dùng collect. Nếu muốn tính toán ra một giá trị duy nhất, hãy dùng reduce.
Q2: Tại sao collect(Collectors.toList()) đôi khi tốt hơn collect(Collectors.toUnmodifiableList())? ↓
toList() mặc định (trong Java 16+) có thể cho phép sửa đổi (tùy implementation). toUnmodifiableList() đảm bảo tính an toàn dữ liệu nhưng sẽ ném Exception nếu em cố ý add() thêm phần tử sau này. Trong kiến trúc Microservices, việc dùng Unmodifiable giúp tránh các lỗi logic khó debug do thay đổi trạng thái bất ngờ.
