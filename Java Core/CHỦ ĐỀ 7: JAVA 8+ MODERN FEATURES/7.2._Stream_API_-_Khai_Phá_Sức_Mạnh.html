Intermediate Operations (Lazy)

Trả về một Stream mới. Không thực thi cho đến khi Terminal Op được gọi.

  • filter(Predicate): Lọc phần tử.
  • map(Function): Ánh xạ biến đổi giá trị.
  • flatMap(Function): Làm phẳng (flatten) stream các stream.
  • sorted(): Sắp xếp (Stateful operation).

Terminal Operations (Eager)

Kích hoạt toàn bộ Pipeline và trả về kết quả cuối cùng (List, Object, Void).

  • collect(Collectors): Gom kết quả vào Collection.
  • forEach(Consumer): Duyệt qua từng kết quả.
  • reduce(BinaryOperator): Kết hợp các phần tử thành một.
  • anyMatch/findFirst: Short-circuiting operations.

5. Internal Working: Sự thật về vòng lặp

Nhiều người nghĩ Stream chạy hết filter cho cả list rồi mới chạy tới map. **SAI!**

Cơ chế: One-Pass & Lazy Evaluation

Java Stream sử dụng một cấu trúc dữ liệu gọi là Pipeline/Stage. Khi Terminal Op được gọi, nó duyệt qua nguồn dữ liệu **một lần duy nhất**. Mỗi phần tử sẽ được đẩy qua toàn bộ pipeline:

Phần tử 1 -> Filter -> Map -> Collect.
Phần tử 2 -> Filter (Nếu sai, bỏ qua luôn, không chạy Map).

Note: Short-circuiting (như limit(1)) sẽ dừng ngay lập tức khi tìm được kết quả, giúp hiệu năng vượt trội so với vòng lặp truyền thống khi xử lý dữ liệu lớn.

6. Parallel Streams - Con dao hai lưỡi

Khi nào dùng?
  • ✅ Dữ liệu cực lớn (hàng triệu bản ghi).
  • ✅ Các tác vụ xử lý độc lập, không phụ thuộc trạng thái.
  • ✅ Tận dụng Multi-core (sử dụng Common ForkJoinPool).
Nguy cơ?
  • ❌ Overhead khi chia nhỏ và gộp dữ liệu (Splitting/Merging cost).
  • ❌ Race condition nếu body của lambda thay đổi biến ngoài.
  • ❌ Hiệu năng có thể TỆ HƠN nếu dữ liệu nhỏ.

7. Real-world Production Code

public List<String> getHighValueOrderCodes(List<Order> orders) {
    return orders.stream()
        .filter(o -> o.getStatus() == OrderStatus.COMPLETED) // Lọc
        .filter(o -> o.getAmount() > 1000)                  // Lọc tiếp
        .map(Order::getTrackingCode)                       // Biến đổi
        .distinct()                                        // Loại trùng
        .limit(10)                                         // Lấy 10 thằng đầu
        .collect(Collectors.toList());                     // Đóng gói
}

🎯 Interview Checklist

Q1: Stream có thể được tái sử dụng (reuse) không?

A: Không. Một stream chỉ được duyệt một lần. Nếu em gọi Terminal Op lần thứ hai, nó sẽ quăng IllegalStateException.

Q2: flatMap() khác gì map()?

A: map biến đổi 1 object thành 1 object. flatMap biến đổi 1 object thành 1 Stream các object và "trải phẳng" chúng ra thành một stream duy nhất.

Q3: stream().peek() thường dùng làm gì?

A: Thường dùng để debug (in log) mà không làm thay đổi luồng dữ liệu.
