Product of Array Except Self

LeetCode #238 | Difficulty: Medium | Array, Prefix Sum Pattern

📝 Problem Statement

Given an integer array nums, return an array answer such that answer[i] is equal to the product of all the elements of nums except nums[i].

Critical Constraint:

You must write an algorithm that runs in O(n) time and without using the division operation.

Example: [1, 2, 3, 4] → [24, 12, 8, 6]

Approach 1: Nested Loops (Brute Force)

Intuition

The most direct way is to iterate through each index i, and for every i, use another loop to multiply all elements j where j != i. Simple to implement, but very slow for large arrays.

Java Implementation

public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] answer = new int[n];

    for (int outerIndex = 0; outerIndex < n; outerIndex++) {
        int currentProduct = 1;
        for (int innerIndex = 0; innerIndex < n; innerIndex++) {
            // Skip the element at the current outer index
            if (outerIndex != innerIndex) {
                currentProduct *= nums[innerIndex];
            }
        }
        answer[outerIndex] = currentProduct;
    }
    return answer;
}

Complexity

Time: O(n²). Two nested loops iterate over the array.

Space: O(1) (excluding output array).

Verdict

Violates the O(n) time requirement. Fails for large inputs.

Approach 2: Prefix & Suffix Products ($O(n)$ Space)

Intuition

Any answer[i] is the product of (everything to the left of i) multiplied by (everything to the right of i). We can pre-calculate these using two auxiliary arrays:
1. Prefix: prefix[i] stores product of nums[0...i-1].
2. Suffix: suffix[i] stores product of nums[i+1...n-1].

Java Implementation

public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] prefixProducts = new int[n];
    int[] suffixProducts = new int[n];
    int[] result = new int[n];

    // 1. Construct Prefix Array
    prefixProducts[0] = 1;
    for (int i = 1; i < n; i++) {
        prefixProducts[i] = prefixProducts[i - 1] * nums[i - 1];
    }

    // 2. Construct Suffix Array
    suffixProducts[n - 1] = 1;
    for (int i = n - 2; i >= 0; i--) {
        suffixProducts[i] = suffixProducts[i + 1] * nums[i + 1];
    }

    // 3. Combine both
    for (int i = 0; i < n; i++) {
        result[i] = prefixProducts[i] * suffixProducts[i];
    }
    return result;
}

Complexity

Time: O(n). Three linear passes.

Space: O(n). Uses two extra arrays to store products.

Efficiency

Satisfies the time requirement! However, can we improve the space?

Approach 3: Space-Optimized Prefix Product ($O(1)$ Space)

Intuition: "The Two-Pass Trick"

We can reuse the answer array to store prefix products first. Then, instead of building a full suffix array, we use a single variable to keep track of the running suffix product as we iterate backwards, multiplying it into our answer array in real-time.

Java Implementation (Optimal)

public int[] productExceptSelf(int[] nums) {
    int length = nums.length;
    int[] answer = new int[length];

    // 1. Use answer array for Prefix Products
    answer[0] = 1;
    for (int i = 1; i < length; i++) {
        answer[i] = answer[i - 1] * nums[i - 1];
    }

    // 2. Use a single variable for Suffix Product while scanning backwards
    int rightSuffixProduct = 1;
    for (int i = length - 1; i >= 0; i--) {
        // Result = (Existing Prefix) * (Running Suffix)
        answer[i] = answer[i] * rightSuffixProduct;
        // Update Suffix for the next element to the left
        rightSuffixProduct *= nums[i];
    }

    return answer;
}

Complexity Analysis

Time: O(n). Two passes (one forward, one backward).

Space: O(1). No extra space used besides the output array (as per problem rules).

Interview Tip

This is the Gold Standard answer. Always start with the Prefix/Suffix intuition and then explain how to "crush" them together to save space.

⚡ Fast Recall

  • • Pattern: Left-to-Right pass, then Right-to-Left pass.
  • • Formula: res[i] = PrefixProd[i] * SuffixProd[i].
  • • Key Insight: Division is forbidden because an array might contain zeros. Total / 0 is undefined.
  • • Zero Handling: If one zero exists, all other indices are 0 except the index of the zero. If two zeros exist, the entire result is 0.

🧩 Edge Case: Zeros

Consider [1, 0, 3, 4].
• Index 0: 0 * 3 * 4 = 0
• Index 1: 1 * 3 * 4 = 12
• Index 2: 1 * 0 * 4 = 0

The Prefix/Suffix product naturally handles zeros without any special "if" checks!
