Gas Station

LeetCode #134 | Difficulty: Medium | Greedy Algorithm

⛽ Problem Context

There are n gas stations along a circular route, where the amount of gas at the i-th station is gas[i]. You have a car with an unlimited gas tank and it costs cost[i] of gas to travel from the i-th station to its next (i+1)-th station.

Goal: Find the starting gas station's index if you can travel around the circuit once in the clockwise direction, otherwise return -1.

Key Observation: If the total sum of gas is less than the total cost, a solution is mathematically impossible.

Approach 1: Try Every Start (Brute Force)

Intuition

We attempt to start at every index i from 0 to n-1. For each start, we simulate the journey. If our tank ever drops below zero, that starting point is invalid.

Java Implementation

public int canCompleteCircuit(int[] gas, int[] cost) {
    int n = gas.length;
    for (int start = 0; start < n; start++) {
        int tank = 0;
        boolean possible = true;
        for (int i = 0; i < n; i++) {
            int currentStation = (start + i) % n;
            tank += gas[currentStation] - cost[currentStation];
            if (tank < 0) {
                possible = false;
                break;
            }
        }
        if (possible) return start;
    }
    return -1;
}

Complexity

Time: O(n²). We check up to n stations for each of the n possible starts.

Space: O(1).

Verdict

Inefficient. It repeats calculations and doesn't utilize the greedy property of the circuit.

Approach 2: Greedy Pruning (Optimal)

Intuition: "The Pruning Logic"

If station A cannot reach station B, then any station between A and B also cannot reach station B.

Why? Because if we reached an intermediate station from A, we would have at least 0 gas (or more) in the tank. If starting with extra gas didn't help us reach B, starting with 0 gas at the intermediate station definitely won't help.

Java Implementation (Optimal)

public int canCompleteCircuit(int[] gas, int[] cost) {
    int totalNetGas = 0;
    int currentTank = 0;
    int startStation = 0;

    for (int i = 0; i < gas.length; i++) {
        int netAtStation = gas[i] - cost[i];
        totalNetGas += netAtStation;
        currentTank += netAtStation;

        // If currentTank drops below 0, station 'i' cannot be the start,
        // and no station between 'startStation' and 'i' can be the start.
        if (currentTank < 0) {
            startStation = i + 1;
            currentTank = 0;
        }
    }

    // If total gas is less than total cost, circuit is impossible
    return (totalNetGas >= 0) ? startStation : -1;
}

Complexity Analysis

Time: O(n). Single pass through the arrays.

Space: O(1). Constant extra variables.

Aha! Moment

If you run out of gas at station 'i', don't just try starting at 'start + 1'. Jump all the way to 'i + 1'. This greedy pruning reduces the search space significantly.

Efficiency Comparison

Approach Time Space Logic
Brute Force O(n²) O(1) Restart from start + 1
Greedy (Optimal) O(n) O(1) Jump over failures

⚡ Recall Cheat Sheet

  • • Pattern: Greedy Pruning.
  • • Pruning Rule: If you fail to reach station $j$ from station $i$, then any station $k$ between $i$ and $j$ will also fail to reach $j$.
  • • Total Sum Property: If $\sum gas \geq \sum cost$, a solution must exist.
  • • Greedy Choice: Just find the first station that can finish the rest of the array without going negative.

🧩 Edge Cases

  • Single Station: gas=[2], cost=[1]. Start at 0, net is 1. Success.
  • Impossible Circuit: Total gas < Total cost. Immediate -1.
  • Start at end: The algorithm naturally wraps around via the total sum check.
