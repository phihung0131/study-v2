Remove Element

LeetCode #27 | Difficulty: Easy | In-Place Modification

📝 Problem Summary

Given an integer array nums and an integer val, remove all occurrences of val in nums in-place. The relative order of the elements may be changed.

Return Value: Return k, where k is the number of elements in nums which are not equal to val. The first k elements of nums should contain the elements which are not equal to val.

Example: nums = [3, 2, 2, 3], val = 3 → Result: k = 2, nums = [2, 2, _, _]

Approach 1: Shifting Elements (Brute Force)

Intuition

The most intuitive way is to iterate through the array, and every time we find val, we "delete" it by shifting all subsequent elements one position to the left. While easy to understand, this is inefficient because a single removal can trigger O(n) shifts.

Java Implementation

public int removeElement(int[] nums, int val) {
    int currentSize = nums.length;
    int index = 0;
    
    while (index < currentSize) {
        if (nums[index] == val) {
            // Shift everything left
            for (int j = index; j < currentSize - 1; j++) {
                nums[j] = nums[j + 1];
            }
            currentSize--; // Array effectively shrinks
            // Don't increment index, need to check the NEW element at this position
        } else {
            index++;
        }
    }
    return currentSize;
}

Complexity Analysis

  • ⏱️ Time Complexity: O(n²). In the worst case (all elements are val), we perform n shifts for n elements.
  • 📦 Space Complexity: O(1). We modify the original array.

Pros & Cons

Cons:

Very slow for large arrays. Repeated data movement is redundant.

Approach 2: Two Pointers (Optimal)

Intuition

Instead of shifting elements every time, we use two pointers:
1. reader: Scans through every element in the array.
2. writer: Keeps track of where the next "valid" element should be placed. If the reader finds a value not equal to val, it gives it to the writer to "keep" it.

Java Implementation

public int removeElement(int[] nums, int val) {
    // The writePointer tracks the position for the next valid element
    int writePointer = 0;
    
    // The readPointer explores the entire array
    for (int readPointer = 0; readPointer < nums.length; readPointer++) {
        // If the current element is NOT the one we want to remove
        if (nums[readPointer] != val) {
            // Place it at the writePointer position
            nums[writePointer] = nums[readPointer];
            // Increment writePointer for the next valid element
            writePointer++;
        }
    }
    
    // writePointer now equals the count of elements that aren't 'val'
    return writePointer;
}

Complexity Analysis

  • ⏱️ Time Complexity: O(n). We traverse the array exactly once.
  • 📦 Space Complexity: O(1). Only two integer variables used for pointers.

Pros & Cons

Pros:

Maximum efficiency. Elements are only moved once if necessary.

Strategy Comparison

Approach Complexity Data Movement Code Complexity
Shifting (Brute) O(n²) High (Redundant) Medium (Nested Loop)
Two Pointers (Optimal) O(n) Minimal (Once per val) Low (Single Pass)

💡 Takeaways

  • • Two pointers (Fast/Slow) is the standard for In-place Array Modification.
  • • The Slow Pointer (Writer) defines the boundary of the "answer" array.
  • • The Fast Pointer (Reader) filters data from the source.

🧩 Interview Note

If the elements to remove are rare, you can optimize further by swapping val with the last element in the array and decreasing length. This avoids copying valid elements repeatedly.
