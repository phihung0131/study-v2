Candy Problem

LeetCode #135 | Difficulty: Hard | Greedy, Array

🍬 Problem Rules

There are n children standing in a line. Each child is assigned a rating value given in the integer array ratings. You are giving candies to these children subjected to the following requirements:

  • Each child must have at least one candy.
  • Children with a higher rating than their neighbors must get more candies than their neighbors.

Goal: Return the minimum number of candies you need to have to distribute the candies.

Approach 1: Iterative Correction (Brute Force)

Intuition: "Keep Fixing until Perfect"

Start by giving every child 1 candy. Then, keep scanning the line from left to right. If you find a child whose rating is higher than a neighbor but they don't have more candies, increase their candy count. Repeat this until no more changes are needed.

Initializing: All children get 1 candy.

Java Implementation

public int candy(int[] ratings) {
    int n = ratings.length;
    int[] candies = new int[n];
    Arrays.fill(candies, 1);

    boolean hasChanged = true;
    while (hasChanged) {
        hasChanged = false;
        for (int i = 0; i < n; i++) {
            // Check left neighbor
            if (i > 0 && ratings[i] > ratings[i-1] && candies[i] <= candies[i-1]) {
                candies[i] = candies[i-1] + 1;
                hasChanged = true;
            }
            // Check right neighbor
            if (i < n - 1 && ratings[i] > ratings[i+1] && candies[i] <= candies[i+1]) {
                candies[i] = candies[i+1] + 1;
                hasChanged = true;
            }
        }
    }
    
    int totalCandies = 0;
    for (int c : candies) totalCandies += c;
    return totalCandies;
}

Complexity

Time: O(n²). In the worst case (ratings [5,4,3,2,1]), we might need n full passes.

Space: O(n). To store the candies array.

Why it's slow?

When you update a child's candy count, it might cause their other neighbor relationship to break, forcing another loop pass. This "ripple effect" is inefficient.

Approach 2: Two-Pass Greedy (Optimal)

Intuition: "Divide and Conquer Neighbors"

Instead of checking both neighbors simultaneously, we handle one side at a time:
1. Left-to-Right Pass: Ensure child i has more candies than i-1 if their rating is higher.
2. Right-to-Left Pass: Ensure child i has more candies than i+1 if their rating is higher.
By taking the max of these two conditions, both rules will be satisfied in just two passes.

Ready to perform two independent greedy passes.

Java Implementation (The Best Interview Answer)

public int candy(int[] ratings) {
    int n = ratings.length;
    int[] leftToRight = new int[n];
    int[] rightToLeft = new int[n];
    
    Arrays.fill(leftToRight, 1);
    Arrays.fill(rightToLeft, 1);

    // Pass 1: Satisfaction from the left
    for (int i = 1; i < n; i++) {
        if (ratings[i] > ratings[i - 1]) {
            leftToRight[i] = leftToRight[i - 1] + 1;
        }
    }

    // Pass 2: Satisfaction from the right
    for (int i = n - 2; i >= 0; i--) {
        if (ratings[i] > ratings[i + 1]) {
            rightToLeft[i] = rightToLeft[i + 1] + 1;
        }
    }

    int total = 0;
    for (int i = 0; i < n; i++) {
        // We must satisfy BOTH constraints
        total += Math.max(leftToRight[i], rightToLeft[i]);
    }
    return total;
}

Complexity

Time: O(n). Three linear traversals.

Space: O(n). Storing two auxiliary arrays.

Key Insight

By splitting the bidirectional constraint into two unidirectional ones, we turn a global problem into two simple local ones. The max() function acts as our "merge" logic.

Approach 3: Single-Pass Peak/Valley ($O(1)$ Space)

Intuition: "Slope Mathematics"

Imagine the ratings as hills. An up-slope of length 3 needs 1+2+3 candies. A down-slope of length 2 needs 2+1 candies. The only tricky part is the Peak: its candy count must be the maximum of the up-slope and down-slope lengths.

Logic Breakdown

UP SLOPE

Count elements increasing. Add to total.

DOWN SLOPE

Count elements decreasing. Add to total.

PEAK ADJUST

If down-slope > up-slope, peak must grow.

Java Implementation (Space Optimized)

public int candy(int[] ratings) {
    if (ratings.length <= 1) return ratings.length;
    
    int candies = 1;
    int up = 0;
    int down = 0;
    int oldUp = 0;

    for (int i = 1; i < ratings.length; i++) {
        if (ratings[i] > ratings[i - 1]) {
            up++; down = 0;
            candies += 1 + up;
            oldUp = up;
        } else if (ratings[i] == ratings[i - 1]) {
            up = 0; down = 0; oldUp = 0;
            candies += 1;
        } else {
            down++; up = 0;
            candies += down;
            // If down-slope exceeds previous up-slope, 
            // the peak needs an extra candy
            if (down > oldUp) candies++;
        }
    }
    return candies;
}

Complexity

Time: O(n). Single pass.

Space: O(1). No auxiliary arrays.

Usage Note

This approach is mathematically beautiful but much harder to implement correctly in a high-pressure interview. Always start with the Two-Pass method first!

Efficiency Comparison

⚡ Algorithm Cheat Sheet

Core Strategy

When faced with "neighbor constraints," try splitting the problem into One-Directional requirements.

  • 1. If rat[i] > rat[i-1], then can[i] = can[i-1] + 1.
  • 2. If rat[i] > rat[i+1], then can[i] = can[i+1] + 1.
  • 3. Final answer is max(1, 2).

Visual Memory Hook

Think of a V-shape ratings graph. The bottom of the V gets 1 candy. As you go up either side, the candies increase linearly. The point where two slopes meet takes the maximum of both.
