Insert Delete GetRandom O(1)

LeetCode #380 | Difficulty: Medium | Design, Array, Hash Table

🎯 Problem Requirements

Implement the RandomizedSet class:

  • insert(val): Inserts an item val into the set if not present. Returns true if the item was not present, false otherwise.
  • remove(val): Removes an item val from the set if present. Returns true if the item was present, false otherwise.
  • getRandom(): Returns a random element from the current set of elements. Each element must have the same probability of being returned.

Strict Constraint:

You must implement the functions such that each function works in average O(1) time complexity.

Approach 1: Naive ArrayList

Intuition

A simple ArrayList handles insert and getRandom efficiently. However, remove is problematic because finding the element takes O(N), and deleting it from the middle requires shifting elements, which is also O(N).

Select an operation to simulate the naive approach.

ArrayList Data

Java Implementation (Naive)

class RandomizedSet {
    List<Integer> list = new ArrayList<>();

    public boolean insert(int val) {
        if (list.contains(val)) return false; // O(N) linear search
        list.add(val); // O(1) append
        return true;
    }

    public boolean remove(int val) {
        int index = list.indexOf(val); // O(N) linear search
        if (index == -1) return false;
        list.remove(index); // O(N) element shifting
        return true;
    }

    public int getRandom() {
        int randomIndex = new Random().nextInt(list.size());
        return list.get(randomIndex); // O(1) index access
    }
}

Complexity Analysis

Insert: O(N) (due to contains() check).

Remove: O(N) (due to finding index and shifting).

GetRandom: O(1).

Why it fails

The requirement asks for average O(1) for ALL three. The linear search and shifting are the bottlenecks here.

Approach 2: HashMap + ArrayList (Optimal)

Intuition: "The Swap & Pop Trick"

To achieve O(1), we need to solve two problems:
1. Lookup: Use a HashMap to store Value -> Index. This makes finding an element's location O(1).
2. Deletion: Removing from the middle is O(N), but removing the last element is O(1).
The Trick: Copy the last element into the position of the element to be deleted, update its index in the map, and then pop the last element.

Try inserting or removing values to see the Map/List interaction.

Java Implementation (Optimal)

class RandomizedSet {
    Map<Integer, Integer> valToIndexMap = new HashMap<>();
    List<Integer> valuesList = new ArrayList<>();
    Random rand = new Random();

    public boolean insert(int val) {
        if (valToIndexMap.containsKey(val)) return false;
        
        // Add to Map with its upcoming index in the List
        valToIndexMap.put(val, valuesList.size());
        valuesList.add(val);
        return true;
    }

    public boolean remove(int val) {
        if (!valToIndexMap.containsKey(val)) return false;
        
        int indexToRemove = valToIndexMap.get(val);
        int lastElement = valuesList.get(valuesList.size() - 1);
        
        // 1. Move last element to the position of element to remove
        valuesList.set(indexToRemove, lastElement);
        // 2. Update the last element's index in the Map
        valToIndexMap.put(lastElement, indexToRemove);
        
        // 3. Remove last element from List and current value from Map
        valuesList.remove(valuesList.size() - 1);
        valToIndexMap.remove(val);
        
        return true;
    }

    public int getRandom() {
        return valuesList.get(rand.nextInt(valuesList.size()));
    }
}

Complexity Analysis

  • Insert: O(1) (Average Map put + List append).
  • Remove: O(1) (Map lookup + List swap + List pop).
  • GetRandom: O(1) (List index access).

Key Concept

The ArrayList provides fast random access, while the HashMap provides fast lookup. The "Swap with Last" technique keeps the list compact without shifting costs.

Efficiency Comparison

Approach Insert Remove GetRandom
Naive ArrayList O(N) O(N) O(1)
Map + List (Optimal) O(1) O(1) O(1)

⚡ Recall Cheat Sheet

  • • HashMap alone: $O(1)$ insert/delete, but cannot $O(1)$ random access (no index).
  • • ArrayList alone: $O(1)$ random access, but $O(N)$ lookup/delete.
  • • The Hybrid: Map stores index of value in list.
  • • The Removal Trick: Replace target with last element in the list to avoid shifting.

🧩 Edge Cases

  • Remove Last Element: The logic works naturally. IndexToRemove equals LastIndex. Swap does nothing, Pop removes it.
  • Insert Duplicate: Check Map first (returns false).
  • Remove Non-existent: Check Map first (returns false).
  • Empty Structure: Handle random selection carefully (size 0).
